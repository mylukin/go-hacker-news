## Separating signal from noise in coding evaluations

- 原文链接: [Separating signal from noise in coding evaluations](https://openai.com/index/separating-signal-from-noise-coding-evaluations/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48837396)

This article talks about how OpenAI checks if its AI models are good at coding by using special tests, but many of these tests have problems. The main idea is that if tests are broken or unfair, they do not show the real skills of the AI, which can lead to wrong choices about using these models.

OpenAI used to use a test set called SWE-bench Verified, but found it had big problems, like letting in outside information by mistake. They then switched to a new test set, SWE-Bench Pro, which was supposed to be better and more realistic. SWE-Bench Pro asks the AI to make changes to real code, making sure new features work and old ones don’t break. The best AI improved a lot on this test in less than a year.

However, OpenAI did a deep review of SWE-Bench Pro and found that about 30% of the tasks were broken. Problems included tests that were too strict, prompts that did not give all the needed details, tests that missed important things, and confusing instructions. To check for these problems, OpenAI used both computer programs and real software engineers to look at the tasks. The computer program flagged about 27% as broken, while the human engineers found even more, about 34%.

The review process had two parts: one used AI agents with access to the code, while the other used human engineers who checked each task carefully. Sometimes the computer and humans disagreed, but they both found many of the same problems. Humans often found even more issues, especially with tests that did not check enough.

OpenAI says these problems are hard to avoid because the tests come from real open-source project changes, which are often messy and made for people, not AI. Tests can be too strict because they focus on a specific fix, not any correct answer. But, as AI gets better, it can also help find these broken tests faster.

Now, OpenAI says that the coding tests should be made by expert software developers just for testing AI. They no longer recommend using SWE-Bench Pro and want better tests that are fair, hard to cheat, and really show if the AI is good at coding.

In the comments, many people agree that coding benchmarks are often flawed. Some say it’s very hard to make tests that are both fair and realistic, because real-world code is complicated. Others point out that strict tests often punish creative or better solutions that do not match the “expected” answer.

Some developers share stories about their own code passing all tests but still being rejected because it looked different from the example solution. Others say that benchmarks made from open-source projects are messy because real code changes involve a lot of communication and hidden rules.

A few people think using human reviewers helps, but it’s slow and expensive. Some like the idea of using AI to help check the tests, but warn that AI can also miss things. There are also comments about how important it is to have honest benchmarks, because companies make big decisions based on these scores.

One person suggests that making good benchmarks is its own hard problem, just like making good AI models. Another says that no test is perfect, but we should always try to make them better. Some think that sharing more details about the broken tasks would help everyone learn.

Overall, the comments show that people know benchmarks are very important, but also very hard to get right. Many agree with OpenAI that better tests, made by experts for AI, are needed.

## Chatto is now open source

- 原文链接: [Chatto is now open source](https://www.hmans.dev/blog/chatto-is-open-source)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48833116)

Chatto is a new open-source group chat app that you can now host on your own server, for free. The creator says Chatto is fast, easy to use, and focuses on privacy, with all data fully encrypted and no tracking or ads.

You can install Chatto quickly using Homebrew or by downloading binaries for Linux, macOS, and Windows. The app is simple to set up—just run one executable, and it serves both the backend and frontend. Each Chatto server hosts one community, and there’s no data sharing or federation between servers. If you want to join multiple communities, you just connect to each server directly. If you want to host more than one, you just start more Chatto processes.

Chatto supports group voice and video calls, including screen sharing. Calls are end-to-end encrypted, so only the people in the call can see or hear what’s shared. The app is designed to use very little system resources, making it fast and lightweight. The roadmap includes more features like content moderation and better multi-server support. The developer plans to reach version 1.0 within a year, but there may be some breaking changes before then.

If you don’t want to self-host, a hosted version called Chatto Cloud is coming soon. This will be paid, but focused only on hosting, with no ads or extra features. It will use European infrastructure, offer automatic backups, and let you move your data in or out at any time.

In the Hacker News comments, many people are excited about a new open-source chat tool, especially one that is easy to set up and focused on privacy. Some users ask about how Chatto compares to existing tools like Matrix, Mattermost, or Discord, especially regarding federation and features. Others point out that not having federation means you can’t message users on other Chatto servers, which is both a plus for privacy and a minus for connecting communities.

A few commenters worry about adoption, since many groups already use popular tools like Slack or Discord, but some like the idea of a simpler, no-tracking option. There are questions about how secure the encryption is, and how safe it is to trust new software for private chats. Some developers praise the simple self-hosting process and clean design, while others wish for more moderation tools and mobile apps. Several people say they want to try self-hosting Chatto, and a few offer ideas or bug reports to help improve it. Overall, the comments show both excitement and caution, with lots of helpful feedback for the creator.

## Decoding the obfuscated bash script on a Uniqlo t-shirt

- 原文链接: [Decoding the obfuscated bash script on a Uniqlo t-shirt](https://tris.sherliker.net/blog/obfuscated-self-evaluating-bash-script-by-cdn-akamai-being-supplied-to-consumers-via-retail-stores/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48829312)

A new Uniqlo t-shirt features an unusual design: a chunk of obfuscated Bash script on the back, created by Akamai for their “Peace for All” campaign. The article author describes how he noticed the code, realized it was a self-evaluating, base64-encoded Bash script, and got curious about what it did.

The author’s wife showed him the shirt, which has a heart on the front and a big block of code on the back. Looking close, he spotted a shebang (`#!/bin/bash`) and recognized it was a special type of script where the real code is hidden by base64 encoding. This is a common trick used to hide what the code does until it runs. The script is made to be copy-pasted into a terminal, decoded, and run, showing a fun “Easter egg” message.

To figure out what the script did, the author had to use OCR (optical character recognition) to read the code from the t-shirt. He used different tools (Android’s circle-to-search, Tesseract, Claude AI) to transcribe the code perfectly, since base64 encoding doesn’t allow for errors. Some other shirts in the same campaign had incomplete code, which wouldn’t run.

After decoding, the author found the script prints a looping, colorful “PEACE FOR ALL” message in a sine wave shape across the terminal. The script hides the cursor, uses terminal sizes, and picks colors for each letter, making a nice animated effect. The author also discussed the font used for the print, first thinking it was Consolas, but later learning (from Hacker News comments) that it was Roboto Mono.

Akamai said in a press release that the shirt is a tribute to the early days of the internet, with references to Linux and the old “beige box” computers. The article notes that he wasn’t the first to notice this code—other nerds online have also decoded it and posted about it.

In the Hacker News comments, many people found the shirt and code idea clever and fun. Some said it’s a great way to mix tech culture with fashion. A few joked that putting obfuscated code on clothing is a nerd badge of honor or a subtle way to find like-minded people in public. Others discussed the technical side, such as how base64 encoding works, why it’s often used for hiding scripts, and the effort required to transcribe code without errors.

Some commenters worried that using `eval` on base64 code is similar to how malware is hidden, and joked about the dangers of running unknown scripts. A few discussed the choice of font, correcting the article and pointing out that the font is Roboto Mono, not Consolas. There were also comments appreciating the attention to detail and the playful message, while some wondered if more companies will use similar “hidden code” designs in the future. A few people shared links to other shirts or items with real code printed on them and swapped stories about nerdy Easter eggs they’ve found in the wild.

## Mistral's Robostral Navigate: a state of the art robotics navigation model

- 原文链接: [Mistral's Robostral Navigate: a state of the art robotics navigation model](https://mistral.ai/news/robostral-navigate/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48832212)

Mistral AI announced a new AI model called Robostral Navigate, which lets robots move through spaces using just one regular camera and plain-language instructions. The model works in offices, homes, and even outdoors, and it does better than other robots that need extra sensors like LiDAR or many cameras.

Robostral Navigate is an 8-billion-parameter model trained only with simulated data. It understands both images and language, so you can tell the robot to do things like “walk through the corridor and stop at the second shelf,” and it will figure out how to move there. The robot looks through its camera, gets the command, and then points to the spot in its view where it should go next. If the place isn’t in view, it uses simple directions like “move forward two meters.” This pointing method is strong because it doesn’t depend on perfect camera settings or exact measurements.

Training the model was made faster by a special method called prefix-caching, which lets the AI learn from whole episodes at once, not just one moment at a time. This made training much quicker and used fewer computing resources. First, the team trained Robostral with this efficient method, and then they improved it further by letting it learn from its own mistakes in simulated worlds. This extra training with trial and error boosted its success by over 3%.

Robostral Navigate has been shown to succeed at following instructions in new places it hasn’t seen before, reaching a 76.6% success rate, better than other robots with more sensors. It works with many robot shapes and sizes, including wheeled, legged, and flying robots, and can handle unexpected obstacles or camera differences.

## Show HN: Microsoft releases Flint, a visualization language for AI agents

- 原文链接: [Show HN: Microsoft releases Flint, a visualization language for AI agents](https://microsoft.github.io/flint-chart/#/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48834924)

Microsoft just launched Flint, a new tool that helps make charts for AI agents easier and better. Flint lets you write a simple chart description, and then it creates a nice chart for you, without needing to set all the tiny details.

Flint is like a language for making charts. You only need to give it your data, say what each column means (like “year”, “region”, “number of users”), and pick a chart type (like line chart or heatmap). Flint’s “compiler” then figures out the best way to show your data. It fills in all the low-level details like how to draw the axes, which colors to use, or how to space things out. For example, if you want to show monthly active users by region, you just say so in the Flint spec, and Flint creates a faceted line chart that splits the data by region and shows the trend over time.

Flint supports 46 different chart types, including bar charts, heatmaps, and even special ones like sunburst charts. You can use Flint with popular chart libraries like Vega-Lite, ECharts, and Chart.js. If you want to switch from one chart type to another (say, from a bar chart to a pyramid chart), you just change a word in your chart description, and Flint takes care of the rest.

One cool feature is Flint’s use of “semantic types.” This means you tell Flint if your data column is, for example, a date, a category, or a number. Flint uses this information to pick the right colors, axis labels, and even how to format numbers. Another nice thing is that Flint automatically arranges your chart for you. If you add more bars or categories, Flint stretches or shrinks the chart to fit everything nicely, so you don’t have to adjust anything by hand.

Flint is open source, and you can install it with npm for JavaScript or TypeScript projects. There’s also a gallery online where you can see 83 chart examples.

In the Hacker News comments, some users are excited and say Flint could save a lot of time, especially when building dashboards or making quick data visualizations. People like how Flint hides the messy details and lets you focus on what you want to show, not how to show it. Some think it could work well with AI, since AI tools often struggle with detailed chart settings.

Other users point out that there are already other tools for making charts, like Vega-Lite or Observable Plot. They wonder if Flint is really different, or just another layer on top of those. Some worry that hiding the details might make it harder to fix problems if the automatic settings don’t look right. There are also questions about how easy it is to add custom features or special chart types that Flint might not support yet.

A few users are happy that Flint is open source and say they want to try it in their own projects. Some are curious if Flint will be kept updated or if it will become another forgotten tool. Others mention that having a single easy format for many chart backends could make it easier to switch between chart libraries in the future. Overall, most people think Flint is a helpful step for making data visualization easier, but they want to see how it works in real projects.

## Turning a pile of documents into a searchable useable knowledge base

- 原文链接: [Turning a pile of documents into a searchable useable knowledge base](https://github.com/linuxrebel/DocuBrowser)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48837110)

This article is about DocuBrowse, a tool that lets you search and manage a big pile of documents on your computer using AI. DocuBrowse can handle many file types like PDFs, Word docs, eBooks, and more, making these files easy to find and use.

The main idea is that DocuBrowse builds a special index of your files, so you can search not just by keywords but also by meaning. For example, if you look for “that contract about the lease renewal,” it can find the right document even if those exact words don’t appear. Each search result shows an AI summary of the file, so you can see what’s inside before opening it. All of this happens on your own computer with local AI models, so your data never leaves your machine, and you don’t need an internet connection or account.

DocuBrowse uses both keyword search (fast, like regular search) and semantic search (AI looks for meaning in your words). By default, it mixes both methods for better results. The AI models used are run locally through a tool called Ollama. DocuBrowse can read and index many formats and also scans for private info (like Social Security numbers or credit card numbers), removing and blacklisting files that have sensitive data.

The user interface is simple, with dark and light themes, paged results, and easy ways to open, delete, or get a summary of each file. It can handle multiple document folders, and you can ignore certain folders if you want. Search is fast—usually less than 150 milliseconds. There are many command-line options for advanced users, like scanning, embedding, removing duplicates, and managing ignored or blacklisted files. The system is secure: it only works on your computer, uses CSRF tokens, and checks every request to make sure nothing from the web can access your files.

DocuBrowse is easy to install on Linux, Windows, and Mac, with clear instructions and native installers. If you run into problems, there are troubleshooting tips in the docs. There are some limits: scanned PDFs need OCR to be searchable, and some eBooks with DRM can’t be fully indexed. Right now, it only works in English, and there’s no authentication, since it’s designed for single-user, local use.

In the comments, many people like that DocuBrowse runs fully offline and keeps data private. Some say it’s great to avoid sending documents to a cloud service. Others like the support for many file formats and the fast, AI-powered search. Some users mention that setup could be tricky, especially with many dependencies like Python, Ollama, and Calibre. A few worry about how much memory and CPU the AI models use, but others say the performance is good for normal use. Some developers are excited about the open-source license and want to add features like OCR for scanned files, multi-language support, or Docker deployment. There are also comments about security—people like that the tool is hardened against outside attacks, but some wish for optional authentication for shared computers. A few users talk about using DocuBrowse for work or personal document management, saying it solves a real problem. Others compare it to other tools but appreciate its strong privacy focus and flexibility.

- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48834405)

OpenAI just launched GPT-Live, a new AI voice model for ChatGPT. This update makes talking with ChatGPT feel much more like chatting with a real person.

GPT-Live uses a full-duplex system, so it can listen and talk at the same time. It pays attention with small phrases like “mhmm,” lets you interrupt, and keeps the conversation flowing. If you pause, it waits instead of talking over you. For harder questions, it quietly sends them to a stronger AI (GPT-5.5) and keeps chatting while it finds the answer. There are two versions: GPT-Live-1 (for paid users) and GPT-Live-1 mini (for free users). The system can now show visuals for things like weather or stocks while you talk. You can also pick how fast or smart you want it to respond—quicker answers or deeper thinking. The voices are more natural, and ChatGPT can better ignore background noise. Safety is a big focus: GPT-Live has new rules to avoid unsafe content, and special protections for teens. It won’t copy real voices and will warn or stop if things get risky in a conversation. Right now, GPT-Live works on iOS, Android, and chatgpt.com, and more features (like video) are coming soon.

People on Hacker News have mixed feelings about this release. Some are excited and impressed by how natural the conversations feel now. Others worry about privacy—wondering if OpenAI will save and use their voice data. Some mention concerns about people becoming too emotionally attached to AI, especially teens. A few users talk about the safety controls and wonder if they will block useful conversations by mistake. There are also worries about the AI misunderstanding accents or not working well in every language. Several developers are eager to try the new API, but want details on pricing and limits. Some point out that while the voice is more human, there’s still a “robotic” feel at times. Others suggest that GPT-Live could change how we use phones or help those who need hands-free help. Overall, many agree it’s a big step for AI voice, but see new problems to solve along the way.
