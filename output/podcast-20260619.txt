For students aged 17 to 19, the government wants them to use AI more, so they are ready for life and work.

The main idea is to protect young children from problems that AI might bring, but also to teach older students how to use AI in a smart way. The government says students should learn good skills first, before using new technology.

Next, there is an article explaining how ATProto, the protocol used by Bluesky, is different from Mastodon or ActivityPub. People often ask, “Where are all the Bluesky instances?” but in ATProto, there are no “instances.” The protocol separates hosting—where your data lives—and apps—how you view and interact with data. It is a bit like blogs and RSS readers: blogs live on many servers, and readers collect posts from everywhere.

Mastodon and ActivityPub try to decentralize by letting anyone run an “instance,” which is a mix of hosting, app, and community. But your identity is tied to your instance, so if your instance goes down, your identity disappears. ATProto does it differently. Your data lives on a server you choose, called a PDS, and any app can read from any server. You can swap your host without losing your identity and use new or different apps any time. Apps do not own your data.

In the comments, some people agree the article explains the difference well. Others think the article is too hard on ActivityPub and that both systems have trade-offs. Some say you still have to pick a host, which feels similar to picking an instance. Others think the article is too technical and does not answer practical questions about moderation, censorship, or data control.

There is debate about running parts of the network. Some think running your own server is too expensive, so most people will still use the big Bluesky servers, which feels centralized. Others say costs are dropping and servers are for developers, not normal users. People also discuss moderation: Mastodon lets admins block other instances, while in ATProto, apps can choose their own moderation. If you get banned from one app, you can move to another, but if the big app bans you, you lose access to a large audience. Some like that ATProto lets you move your data; others worry about central points of failure. In the end, many agree that both models have strengths and weaknesses, and real decentralization is hard to solve.

Next, we have a story about making espresso with sound waves instead of hot water, which could cut energy use by 75%. Researchers found that using ultrasound at room temperature can make espresso as strong and tasty as the traditional method. High-frequency sound waves shake coffee grounds in water, creating bubbles that help pull out flavor and caffeine fast. The process uses a metal device that vibrates the filter basket in a regular espresso machine.

The researchers compared their “ultrasonic espresso” to normal espresso with about 100 regular coffee drinkers in a blind test. People could not tell the difference in taste, smell, or bitterness. For filter coffee, the ultrasound version was even liked more. The main benefit is energy saving, which matters most for companies making bottled or ready-to-drink coffee.

In the comments, some people are excited about the technology, especially for big coffee makers. Others are skeptical about the energy savings, pointing out that the research may not include all energy costs. Some coffee fans say espresso should be hot, but others note that for cold or instant drinks, temperature does not matter. There are questions about using ultrasound with milk, and about how the process changes taste. Home coffee lovers debate if this method is useful for them, and some talk about the noise of ultrasound machines. Overall, many say saving energy is more important for industry than for home coffee.

Turning to programming news, Project Valhalla is a big change coming to Java in JDK 28. It brings “value classes,” which let you write classes that work like primitives for better performance and memory use. With value classes, you can make new types like Point or Money that act like regular classes, but the JVM can store and use them like plain numbers. Normally, Java uses references for everything except basic types like int or double, which can be slow. Value classes let the JVM store data directly, side by side, making things faster and using less memory.

The Valhalla project has been in the works for over ten years. Value classes are declared with a new “value” keyword, have no identity, and all fields are final. Two value objects with the same data are “the same” with ==. You can’t synchronize on them. Value classes can still be null for now, but non-nullable types are planned for the future.

Key optimizations are scalarization and heap flattening. This is best for small objects, but if it’s too big or can be null, the JVM may still use regular objects. In JDK 28, this is a preview feature, not enabled by default.

Generics are not fully supported yet—Java still uses type erasure, so you can’t get flat storage with generics like List<Point> for now. That will come later.

People in the comments are excited about the speed and memory improvements, but worry about new complexity, especially how == now checks all fields. Some compare Java’s approach to .NET and C#. Some think Java is just catching up to C#, but others say making this work in Java is harder because of backwards compatibility. Some complain about the slow pace, since non-nullable types and full generic support are still coming. Others defend Java’s careful approach. Some wish for more human writing in the article. Most agree that, once finished, Valhalla will make Java faster and more flexible for high-performance tasks.

Others debate the value of knowing rare words, or the link between vocabulary and intelligence. Most find the quiz fun, but see room for improvement in the user experience and accuracy.

Moving to history, we have a story about how a small team at the University of Massachusetts Amherst built a wind turbine in 1976, starting the U.S. wind industry. The project was led by William Heronemus, who wanted wind power to help solve energy problems after the 1970s oil crisis. They built the first turbine using old car parts and handmade blades, and used it to heat a small house during cold winters.

At that time, most people thought wind power was only for small tasks, but Heronemus believed it could provide a lot of electricity for many homes. He gave talks and wrote papers about wind farms and even floating turbines in the ocean. He inspired many students who later helped start big wind farms, especially in California. Even today, many wind experts started their careers with Heronemus.

In the comments, one person shared a story about wind turbine research in Romania, where their dad saw a big vertical turbine fail because the brakes did not work. The commenter said they did not find these stories interesting as a child, but now see how important and exciting this work was. Other readers enjoyed the memory and talked about the challenges and failures of early wind projects, and how learning from mistakes is important in engineering.

Next, a new bill called the JAWBONE Act was introduced in the U.S. Senate to stop government officials from pushing tech companies to silence legal online speech. The bill lets people sue the government if officials try to make companies take down posts or accounts that are protected by the First Amendment.

"Jawboning" is when the government pressures companies to censor speech without the public knowing. The bill would give users a way to fight back in court. For example, the EFF is helping the creator of an app called ICEBlock, who was pressured by officials to remove it from the App Store. The EFF is also suing to find out more about how the government spoke with Apple and other companies.

The article says not every message from the government to a tech company is bad, but there is a risk if the government goes too far. The EFF points out that social media companies also have their own free speech rights and can choose how to moderate content. The Supreme Court has agreed that companies can decide what is allowed on their sites. The EFF hopes this new law will protect users’ rights while letting companies run their platforms as they see fit.

In the comments, people joked about the creative name of the bill. Some point out that the bill is bipartisan. Others worry if the sponsors really care about free speech in all cases, since politicians sometimes support speech when it helps their side. Some say both sides try to limit speech when it suits them. Others debate if more laws are needed, or if the First Amendment is enough. There are examples from other countries, and debate about government pressure to remove harmful misinformation. Some say companies should be able to manage their own platforms, while others worry about too much government power. Many agree the issue is hard, and the new law could have both good and bad effects.

Now, Google Workspace is warning some users on Firefox that they should use Chrome to access their work account. The message says users may soon lose access unless they use Chrome, even if their browser is up-to-date. The warning comes from a Workspace security page saying to "secure your device" by using Chrome. The author tried to get help from Google, but got no clear answer. The main worry is that this move could force teams to use only Chrome, even if they prefer Firefox or need to test their apps on different browsers.

Many Hacker News readers say this warning is set by company IT teams using a feature called Context-Aware Access, not by Google itself. Some say blame the company IT, but others say Google makes it much easier to manage Chrome than Firefox, pushing teams toward Chrome. IT people explain that Chrome is easier to control and monitor, so it is safer for company data. Supporting more browsers costs time and resources.

Other readers feel this could hurt browser choice and help Google’s monopoly. Some point out that Chrome blocks ad blockers, which can make browsing less safe. There is worry that forcing only Chrome will make web standards weaker. People also debate if Chrome is really more secure than Firefox, with some saying Chrome has better business controls, but others argue that Firefox can be managed too. Some say companies have the right to pick what software workers use, but many dislike that the message uses unclear words like "secure browser" when it really just means "Chrome." Some see this as a sneaky way for Google to push more people onto Chrome, which could hurt competition and innovation.
