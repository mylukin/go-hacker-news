First, let’s talk about Myna, a new monospace font made for programmers. Myna is special because it gives extra care to symbols, making them as clear and important as letters. The idea came from the author’s wish for better-looking arrows, dollar signs, and other symbols that often look odd in regular coding fonts. Myna aligns multi-character symbols like "->" and "::", so code looks neat and balanced. It also fixes common problems like confusing "1", "l", "I", or "0", "O", and "o". Right now there is one style, but more may come if people ask. It works on Linux, macOS, and Windows, and you can use it in most editors and terminals. Myna started as a tweak of Source Code Pro, but now takes ideas from other fonts like Fira Mono and Inconsolata. The font is free under the SIL Open Font License, supports some non-ASCII symbols, and the author welcomes bug reports and requests.

In the comments, many users liked the focus on clear symbols, saying code with lots of operators often looks messy in other fonts. People praised the distinct shapes, especially for languages like Haskell and Perl. Some wanted more weights or italics to fit their workflow. There was debate about real ligatures, with some users liking them and others preferring a plain ASCII look. Some said the font is modern and clean, while others think it could use more polish. Many agreed that font choice is personal, and shared their own favorites. Some reported small issues with spacing or rendering and suggested improvements. Overall, people were happy to see a font that cares so much about coding symbols.

Next, we have a story from a Ruby developer who found out that Ruby already has a built-in way to compare app versions, which solved a problem he was working on. He had written his own AppVersion class to compare strings like "1.2.3" in his Hotwire Native app. But during a community Zoom call, another developer told him about Ruby’s built-in Gem::Version class, which does this job even better. Gem::Version can handle version strings, including pre-release tags like "beta", and is well tested. The author quickly switched to using it and wondered what other standard features he might be missing in Ruby or Rails. He also talked about how important it is to join community events and learn from others.

In the comments, many people said they too have written code only to later find a built-in solution. Some praised the author’s openness to learning, while others said reading the docs and talking to other developers can save a lot of time and prevent bugs. There were stories about reinventing the wheel, but most agreed it is fine for learning, while real projects should use trusted libraries. People shared links to lesser-known Ruby features and encouraged everyone to explore the standard library. The main lesson: don’t be afraid to ask questions, and stay connected to the community.

Now, let’s talk about traceroute and how your computer’s data travels across the Internet. The author built a website that shows the path your data takes using a custom traceroute tool called ktr. Traceroute works by sending packets with a time to live (TTL) value. Each router along the way reduces TTL by one, and when it hits zero, the router sends back an error. By increasing TTL each time, the tool learns which routers the data passes through. The site runs traceroute from the server to your computer, then tries to show what the reverse path might look like, but warns that the real return path can be different. The tool also finds which autonomous system (AS) owns each IP address using public databases, but this is tricky because the data is messy. At the edge of each AS, routers use BGP, a protocol that helps decide the best path for your data. The article’s goal is to help people understand how the Internet works, share open source code, and encourage others to explore.

In the comments, most people liked the website and found it fun and educational. Some remembered learning about networking with traceroute. There were technical discussions about why reverse paths can be different and why parsing WHOIS data is so messy. Many praised the site’s clever use of CSS to update results live without JavaScript. Some users discussed privacy and the difference between running traceroute from server or client. People liked the history lesson about BGP, and some wanted even more detail about the politics and business behind the Internet. Overall, the feedback was very positive, saying the site makes a complex topic easy to understand.

Next up is Ribir, a new GUI framework for Rust and WebAssembly. Ribir lets you write your app’s logic once and run it on desktop systems—Linux, Windows, and macOS—as well as on the web, with mobile support planned. The framework is “non-intrusive,” so you don’t need to change your data or logic much. It only updates parts of the UI that need changing. You can write your UI with normal Rust code, and use macros for a more declarative style, but this is optional. Ribir uses a widget system, a theme system, and turns your views into GPU-drawn 2D shapes. There is support for event bubbling and capturing, text input, and basic widgets, but the docs say things are still rough and may change. The layout system is inspired by Flutter. Ribir is open source and MIT-licensed.

In the comments, people were excited to see another Rust GUI framework and liked the non-intrusive idea. Some wondered why there are so many Rust GUI projects and if any will become popular. Users liked the WASM support and said this is good for cross-platform apps. There were questions about stability, and comparisons with other Rust GUI projects like Dioxus and Iced. Some pointed out that missing mobile support and rough widgets could be problems for now, but praised the clear docs and simple examples. Overall, people were curious and supportive, but cautious about using Ribir in production yet.

Now, let’s move to a story about YouTube removing videos that show how to bypass Windows 11 requirements. The CyberCPU Tech channel had posted tutorials on installing Windows 11 on unsupported hardware or with a local account. YouTube took them down, saying they could cause “physical harm or death,” and gave the channel strikes. The creator appealed, but both appeals were denied very quickly. At first, the creator thought it was just an AI moderation mistake, but later wondered if Microsoft was involved. YouTube later restored the videos, saying it was not automation. The article explains that automated moderation often cannot tell safe content from dangerous, especially for tech topics. Human review is needed, but big platforms rely too much on AI.

Hacker News commenters were upset, saying teaching people to use their own computers is not a real risk. Some thought YouTube is just avoiding any legal trouble. Many said automated moderation often gets things wrong, and appeals are useless if no real human checks them. Some worried that this kind of moderation makes it hard to share tech knowledge. Others joked about what “physical harm” could come from installing Windows 11 on old hardware. Most agreed that the takedown was too strict and that platforms need better ways to handle tech content.

Next is an article about a colorful Venn diagram made for seven sets. Each set gets its own color, chosen to be even around the color wheel. The author wanted every area of the diagram to look the same size and for the diagram to be playful and balanced. The diagram is interactive, so you can spin it and see different sides, almost like a 3D object. Mixing the seven colors in all ways gives 128 different color regions. The shape is based on a famous earlier 7-set Venn diagram, but the author wanted it to feel more fun and easy to explore.

In the comments, people were impressed by how hard it is to make a 7-set Venn diagram clear and balanced. Some remembered how even 3 or 4 sets were hard to draw in school. The colors make it easier to read, but some worried about color blindness. There were discussions about the math behind Venn diagrams and how each extra set doubles the regions. Some people wanted to see the code or math used to build the diagram, while others just enjoyed the playful look. Many said interactive diagrams like this are a good way to get students interested in math and logic.

Now let’s talk about why some programmers love OCaml, even though it is not very popular. The article’s author started with Haskell, liked its strong types and functional style, but found it complex and hard to use with slow compile times. Then they tried Go, which was simple and fast, but too simple—missing features like pattern matching and sum types. After this, the author listed what they want: fast compile, simple runtime, strong types, functional features, good speed, and good docs. They found OCaml gives all these things, with fast compile times, strong types, type inference, single binary output, and good tools. OCaml is simple but expressive, though it has some old or confusing parts.

In the comments, many agreed that OCaml is powerful with strong types, good speed, and nice features, but wished it was more popular. Some said OCaml’s syntax is unusual and the French influence can be confusing. Others pointed out the smaller ecosystem and fewer job opportunities. Some said build tools and package management have improved, and a few wanted a more modern syntax. People who switched from Haskell said OCaml is simpler for daily work. Projects like ReasonML and ReScript help bring OCaml ideas to new users. Some warned the object-oriented features in OCaml are best avoided. Overall, people see OCaml as a hidden gem, but it still faces challenges growing its user base.

Next, we remember James Watson, the scientist who helped discover the structure of DNA, who has died. Watson and Francis Crick found DNA’s double helix shape in the 1950s, using Rosalind Franklin’s X-ray images. This discovery changed biology and medicine, and won Watson, Crick, and Wilkins the Nobel Prize in 1962. Later, Watson worked on genetics, led the Human Genome Project for a while, and wrote books about DNA. But he also made statements in later years that many found offensive and racist, losing honors and jobs because of this.

In the comments, many people thanked Watson for his work, saying the double helix discovery changed science. Some pointed out that Rosalind Franklin did not get enough credit. There was debate about how to judge Watson’s legacy, with many saying it’s important to remember both the good and bad parts of his life. Some felt sad about his death, but said it is right to talk about everything he did. Many agreed his discovery was huge, but his personal behavior made his story complicated.

In the comments, many people thank Soumith for his work, saying PyTorch changed their lives and research. Some worry about the project’s future, but others point out that open-source projects often keep growing with new leaders. People praised the friendly and open tone of Soumith’s post, and hope Meta keeps supporting PyTorch. Some wonder what Soumith will do next, and wish him luck. The overall feeling is thankful and hopeful.

Finally, Jean-Baptiste Kempf won the European SFS Award 2025 for his work on VLC Media Player. VLC started as a small student project in 1996, first as a network streaming tool, then as a media player used by billions. Kempf joined in 2000, brought in new developers, and made sure VLC stayed alive and free. He started the non-profit VideoLAN, and refused big offers to sell VLC or put in ads, because he cares about user freedom. In 2018, France made him a Chevalier de l’Ordre National du Mérite.

In the comments, many said VLC is one of the first things they install on any computer. People like that it is simple, fast, and free of ads or bloat. Some shared stories about using VLC on old computers or for strange video files. There was praise for VLC’s support for many formats, thanks in part to FFmpeg. Some said the traffic cone logo makes VLC friendly and memorable. People wish more software projects stayed open and community-driven like VLC. Many thanked Kempf for years of work and agreed he deserves the award.
