Next, we talk about a new way to connect layers in Transformer models, called Attention Residuals, or AttnRes. Normally, Transformers use “residual connections,” where each layer adds its output to the previous layer. This works, but in deep models, the effect of each layer gets weaker, and the numbers can get too big. AttnRes lets each layer use attention to pick which earlier information is most important, instead of treating all past outputs the same.

There are two main types: Full AttnRes lets each layer look at all previous layers, but uses a lot of memory. Block AttnRes groups layers into blocks, using normal connections inside each block and attention between blocks, to save memory.

Block AttnRes is shown in PyTorch code, and results show models with AttnRes do better on benchmarks like MMLU, TriviaQA, and HumanEval. The biggest gains are for tasks needing more reasoning or coding. Training is also more stable.

On Hacker News, people are excited about the idea, saying it could help build bigger, smarter models. Some like that it’s a “drop-in replacement.” Others ask if it will slow down training or make debugging harder, since now every layer can see many earlier outputs. Some want to know how much memory Block AttnRes really saves, and if it works for very large models. A few say other research has tried similar ideas, but this one looks simple and practical. There is hope it will become a new standard, but also warnings that benchmarks don’t always mean real-world gains. The community is interested and ready to test the method.

Now, let’s talk about the Los Angeles Aqueduct, a huge water system built to bring water from the Sierra Nevada mountains to LA, over 300 miles away. The aqueduct starts at the Owens River, takes water into a canal, and flows downhill all the way to LA using only gravity. Building the canal required a constant gentle slope, which was hard in rough land. The first part is open, but after the Alabama Gates it’s lined with concrete to stop leaks. The Alabama Gates also allow workers to send extra water back to the river if needed.

Taking water from Owens Valley caused big problems for local farmers and ranchers, who lost water for their land. Some even broke parts of the aqueduct in protest. The loss of water dried up Owens Lake, creating dust storms. The aqueduct crosses hills and valleys, sometimes going underground in big pipes called inverted siphons, and has several reservoirs along the way. There are now two main aqueducts, one from the early 1900s and a second from the 1970s.

Water is used to generate electricity as it enters LA, and is cleaned and stored in the Los Angeles Reservoir, which is covered with black plastic balls to keep it safe. LA also pulled water from Mono Lake, causing more lawsuits and new rules to protect the environment.

The article says the aqueduct was an amazing engineering project, but also did a lot of damage. Now, climate change makes water supply less certain. The writer says future engineers should learn from this, and think about nature, people, and long-term effects.

In the comments, people are amazed by the size and use of gravity, and say it shows how bold early engineers were. Others talk about the harm to Owens Valley, and some mention the movie “Chinatown,” based on these events. There are technical discussions about keeping such a long canal working, and the clever use of inverted siphons and hydro power. Some worry about the future as climate change affects water supply. Many say future water projects must be more careful with nature and local people. The comments show both respect for the engineering and sadness for the damage done.

Next, NumKong is a new open-source library with over 2,000 kernels for fast math on many types of numbers, from tiny mixed-precision to high-precision. The author stopped work on SimSIMD and now focuses on NumKong, which works in seven programming languages and supports many hardware types like Intel AMX, Arm SME, and RISC-V. It can run on CPUs, GPUs, and even browsers with WebAssembly. NumKong supports many number formats, and is tested for both speed and accuracy.

For normal tasks like matrix multiplication, it’s not always the fastest, but it often gives more accurate results, especially with unusual or small number types. NumKong provides special tricks for integer dot products, reductions, and more. It can do geospatial math, align 3D shapes, and handle complex AI math faster than older libraries. The code is modular, and you control memory and threading. There are examples in Python, C++, and Rust for common AI tasks.

On Hacker News, users like the huge scope and multi-language support, and are impressed that NumKong supports new number types and many platforms. Finally, we look at “Parallel Perl” or pperl, a new Perl 5 interpreter written in Rust, with help from AI tools and agents. The author, Richard Jelinek, used Perl and AI to build smart, off-grid homes, then created pperl to add new features to Perl.

Pperl aims for high compatibility with Perl 5.42 but is written in Rust for safety and speed. It uses Rust’s Rayon library to run loops in parallel and has just-in-time compilation for much faster code. It can call C libraries directly, supports pre-compiled bytecode cache files, and can run as a daemon for instant script startup. Benchmarks show big speed-ups—sometimes over 400 times faster than Perl 5 for some tasks.

Many on Hacker News are excited to see Perl get a modern, fast interpreter. Some discuss the challenge of full compatibility with Perl 5, known for edge cases. Others wonder if pperl will help bring Perl back to life, or if it is too late. There’s interest in the technical details, and some Perl fans want to try pperl for their own scripts. Some worry that Perl’s reliance on CPAN modules with C extensions may be hard to support. People compare pperl to projects like PyPy for Python, and hope it can make Perl scripts useful again for automation and data tasks. Many thank the author for pushing Perl forward.
