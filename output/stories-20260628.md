## GLM 5.2 beats Claude in our benchmarks

- 原文链接: [GLM 5.2 beats Claude in our benchmarks](https://semgrep.dev/blog/2026/we-have-mythos-at-home-glm-52-beats-claude-in-our-cyber-benchmarks/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48709670)

GLM 5.2 is a new open-weight AI model from Zhipu AI, and it just beat the popular Claude model in a test for finding security bugs in code. The test checked how well different AI models could spot a problem called IDOR, where a user can see or change another user’s data by guessing an ID number.

The Semgrep team wanted to see how much finding these bugs depends on the AI model itself versus the tools (“harness”) wrapped around it. Their own tool, Semgrep Multimodal, uses a special harness to help the AI look in the right places, and with this setup, it got the best results. But when all models were given just a simple prompt and code (with no extra help), GLM 5.2 did much better than expected, even beating the advanced Claude Code model.

GLM 5.2 is special because anyone can download and run it, and it costs much less than closed models like Claude. It uses a Mixture-of-Experts setup, meaning it’s big but uses only part of itself for each answer, making it cheaper and faster. It also can look at huge codebases—up to a million tokens at once—which is good for big security jobs. On common coding tests, it scores as well as top models but is much cheaper to use.

The main test looked at how well each model could find real IDOR problems (recall), avoid false alarms (precision), and do both at the same time (F1 score). GLM 5.2 got a 39% F1, beating Claude’s 32%. It also cost only about $0.17 per real bug found, which is important for running checks across lots of code. Other open models like MiniMax and Kimi did worse, showing that GLM 5.2 is a standout, not the norm.

The results show that the harness or tool around the model matters a lot—Semgrep’s pipeline, with its harness, did best. But it’s a big deal that an open and cheap model like GLM 5.2 can beat a top closed model on its own, with no extra help.

In the comments, some people are excited about open-weight models getting so good, especially since they can be used privately and at low cost. Others warn that this is just one kind of security bug, and results might change with other problems or data. Some users ask about the difference between “open-weight” and “open-source,” and the article’s authors reply that open-weight means you get the trained model but not the full training data or process.

Many users think the test was fair, but a few want to see broader tests, more types of security flaws, or more difficult codebases. Some are surprised at how much the harness boosts results and suggest focusing more on better tools rather than bigger models. A few people worry about “reward hacking,” where models cheat during training, but others say this just shows the need for better testing and guards. Overall, the community agrees this is an important moment for open models, but they want to see more experiments before making big claims.

The author ends up unsure—AI and doctor both sound sure of themselves, but only one can be right. This article shows how the price of memory, like DRAM and storage, has changed from 1960 to 2026. It uses charts and data to help people see the trends for different types of memory.

The main chart shows the cost per gigabyte over time for DRAM, NAND flash, and HBM memory. Prices have dropped a lot over the years, especially as new memory types and generations came out. For DRAM, the article breaks down prices by generation, like DDR, DDR2, DDR3, DDR4, and DDR5, with older data being less exact.

The article also talks about the cost to build AI accelerators, such as those from Nvidia, AMD, Google, and Amazon. It shows how much of the total cost comes from memory (HBM), logic chips, packaging, and other parts. HBM prices are special because they are sold only to big companies, not in stores, so the price numbers are just industry guesses.

The data comes from old and new sources. For older DRAM prices, the article uses the McCallum dataset, which tracks memory prices since 1957. For newer prices, it uses Amazon price history from Keepa. For SSD prices, it tracks the cheapest NVMe SSDs since 2016, but skips strange prices that seem like mistakes.

HBM prices are only estimates, since there is no public market. The article tells readers to be careful: the numbers show the cheapest prices found, not what most people pay, and sometimes they are for old products being cleared out. The article also points out that the price data is not adjusted for inflation.

The data is updated every month for DRAM and NAND, and every quarter for HBM. Anyone can download the full dataset as a CSV file. The article is maintained by David Shim at Stanford.

## TOP500 at ISC’26: We have a New Number 1 Supercomputer

- 原文链接: [TOP500 at ISC’26: We have a New Number 1 Supercomputer](https://chipsandcheese.com/p/top500-at-isc26-we-have-a-new-number)

The article talks about the new number one supercomputer in the world, called LineShine, revealed at the ISC 2026 event in Germany. LineShine, built in Shenzhen, China, is the first Chinese supercomputer to be added to the TOP500 list in nine years.

LineShine uses a special CPU called the LX2, which follows the Armv9 standard and supports new instructions for faster calculations. Each LX2 chip has 304 cores running at 1.55 GHz, with 32 GB of fast memory on the chip and 256 GB of regular DDR5 memory. The system is very big, with over 22,000 nodes and more than 13 million CPU cores in total. It can reach 2.2 exaflops of real performance, which means it can do about two billion billion calculations per second. The supercomputer uses 42 megawatts of power, making it one of the most powerful CPU-only systems ever built. It is also the best in another test called the HPCG benchmark, which checks how well the computer can handle real scientific workloads.

There is also a new number six supercomputer, called HPC7, from Italy’s Eni company. This system uses AMD chips and is based on the same technology as another famous supercomputer, El Capitan, just smaller. Italy now has more computing power on the TOP500 list than any other country in Europe. The old number one, Japan’s Fugaku, is now number nine but still does very well in some tests.

The article notes that the Green500 list, which tracks the most power-efficient supercomputers, has not changed in its top 10 for the first time ever. However, because some old systems were retired, the overall field has become more efficient.

The writer is surprised to see China submit a new top system, as they have been quiet for years even though they likely had powerful computers already. There are questions about whether China will now add even more of its systems to the list. The writer wonders if this will lead the US government to spend more money on new, bigger supercomputers.

Another interesting point is that an oil company, Eni, now has two of the top ten supercomputers. The author also asks why big AI companies do not submit their huge GPU systems to the TOP500 list, even though these are some of the most powerful computers in the world.

Finally, the TOP500 list will soon be managed by a new group, ACM SIGHPC, which will make it easier to reference past lists.

Some readers question why China waited so long to submit results and discuss possible political reasons or export restrictions. Others talk about the focus on CPU-only systems versus GPU-based ones, noting that GPUs are usually better for many tasks, but LineShine shows CPUs can still be very strong. Several users wonder about the real-world use of such big systems and if AI clusters should be listed as supercomputers, since they have huge computing power but are not tested in the same way. ## Librepods: AirPods liberated

- 原文链接: [Librepods: AirPods liberated](https://github.com/librepods-org/librepods)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48710232)

This project is called LibrePods, and it helps users get AirPods features on devices that are not made by Apple, like Android phones and Linux computers. The main idea is to let people use cool AirPods functions, such as noise control, ear detection, battery status, and head gestures, without needing an iPhone or Mac.

LibrePods works by copying the special way AirPods talk with Apple devices. Features like changing listening mode, seeing battery status, and renaming AirPods are already working on both Linux and Android. Some features, like head gestures and noise cancellation with just one AirPod, are only on Android right now. Others, like high-quality two-way audio and heart rate monitoring, are not ready yet or may need more work, sometimes even root access on Android.

There are also plans to allow “Find My” network use and better spatial audio, but these may be tricky and might need extra permissions. The project explains how to “spoof” the device’s identity to make AirPods think your device is an Apple product, which unlocks more features. This is done by changing settings on Linux or using the Xposed module on Android.

Most of the code is written by hand, but some parts—like the head gesture logic—were made using AI. The team also thanks people who have helped with reverse engineering the AirPods protocol. If you use Windows or other systems, there are some other apps suggested, like MagicPods.

The code is open source and uses the GNU General Public License. The project is not linked to Apple, and users are warned not to trust websites pretending to be the official LibrePods page.

Hacker News comments show a lot of interest and excitement. Many users like that AirPods can now be used fully on Android and Linux. Some people are impressed by the reverse engineering work, calling it clever and useful for breaking tech “walled gardens.” Others ask about privacy and security, wondering if spoofing the device or using root is safe. There are also questions about which AirPods models work best and how stable the features are.

A few users say they have tried the app and it works well for basic things, but advanced features sometimes have bugs. Some share tips about setting up VendorID spoofing. Others want to know if this will help with other Apple accessories, like AirPods Max or Beats headphones. One commenter warns about legal risks, since copying Apple’s protocol might upset Apple.

Overall, most people think LibrePods is a great tool for giving users more freedom and choice with their devices. Some hope the project will keep growing, while others are just happy to see real progress in making Apple’s hardware work outside Apple’s world.

This article is about running OpenBSD on the Lemote Yeeloong laptop, a rare Chinese netbook known for supporting free software without hidden code. The Yeeloong uses the Loongson (Godson) MIPS processor, which was developed in China to create local technology and reduce dependence on foreign CPUs.

The article explains the history of the Loongson chips, starting from the 1980s when China began investing in its own technology. The first Godson chips were MIPS-compatible but not fully licensed, leading to some legal issues. Over time, the chips improved, with the Godson-2F being used in the Yeeloong laptop. The Yeeloong was designed to be cheap and easy for normal people to buy, with simple hardware, low power use, and an open bootloader called PMON. This made it popular with people who care about free software, like Richard Stallman.

The writer bought a second-hand Yeeloong and describes its hardware in detail: it is small, light, and has some design flaws, like a noisy fan and a strange keyboard. The author takes the laptop apart to check the cooling and shows how to use the PMON bootloader to explore the system and load different operating systems.

The article goes step-by-step through installing OpenBSD on the Yeeloong. It explains some problems, like needing to use both an internal SSD and an SD card because of space limits, and having to build software from source because there are no binary packages for this CPU. The author also talks about hardware bugs in the Loongson-2F, like issues with branch prediction that can freeze the system, and how open source developers worked around these problems. Some Wi-Fi adapters did not work with OpenBSD, but a Ralink USB dongle did.

The process involves partitioning the SD card, installing OpenBSD, copying the kernel to the right place, and fixing boot settings. The author highlights that updates and upgrades must be done carefully, and sometimes it is easier to wait for a new OpenBSD release than to fix everything yourself. The Yeeloong’s performance is not great compared to other old computers, but it is unique because of its open design and history.

## Show HN: NanoEuler – GPT-2 scale model in pure C/CUDA from scratch

- 原文链接: [Show HN: NanoEuler – GPT-2 scale model in pure C/CUDA from scratch](https://github.com/JustVugg/nanoeuler)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48710778)

This project is about building a GPT-2-like language model from scratch using only C and CUDA—no machine learning libraries or frameworks. The creator wrote everything by hand: forward and backward passes, tokenizer, training process, and even custom CUDA kernels for GPU speed.

The model can run and train on a CPU or a single consumer GPU (like an RTX 4070). It uses a hand-made byte-level BPE tokenizer, pretrains on books and web text, and then fine-tunes to become a simple chatbot. The model has around 116 million parameters, so it can write English sentences that look good, but its answers do not have real knowledge. The chat model is just to show the pipeline works, not to build a smart assistant. 

The project is named “NanoEuler” because it uses a numerical method from mathematics (the Euler method) in its neural network design. Each layer in the network is like a small step in solving a math equation, inspired by how deep learning and math connect.

The code is very transparent: gradients (for learning) are checked by comparing hand-written math to a simple numerical method, so it’s clear there are no mistakes. The model includes common features from modern transformers: RMSNorm, rotary embeddings, grouped-query attention, and a special attention method called FlashAttention for speed. There are no biases in the network, and the tokenizer works at the byte level like GPT-2.

You can train small models on your CPU for practice or run the full pipeline on GPU. The CUDA part is from scratch too—no PyTorch or TensorFlow. All parts are tested to make sure the math matches between CPU and GPU. Training data comes from Project Gutenberg books and a clean web dataset; there are scripts to download everything.

The whole project is meant for learning and research, not production. The author wants to “own every piece”—meaning, they understand and wrote every part, with no black boxes.

In the Hacker News comments, people praised the project’s clarity and completeness. Many were impressed by the hand-written backpropagation and the effort to check gradients so carefully. Some commented that understanding every part of a big model is rare today, so this project is a great learning tool. Others said that building everything from scratch is a huge task, and thanked the author for sharing the code and ideas.

A few readers pointed out that while the model can write fluent text, it lacks real-world knowledge—this is expected at this small scale. Some asked about training times and hardware requirements; others wondered if such a project could help them learn about transformers and CUDA programming.
