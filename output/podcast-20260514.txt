First, let’s talk about car privacy. A Toyota RAV4 Hybrid owner decided to remove the modem and GPS from their 2024 car to stop it from sending data to the company. The article explains that modern cars collect a lot of personal information—like your location, speed, and even video or driver details. This data can be sold or shared, causing concerns about privacy and safety. The writer gives examples: hackers unlocking cars remotely, insurance rates going up based on driving data, and companies sharing private footage. To stop this, the writer took out the modem (called the DCM) and GPS. Most car features still work, but cloud services like crash alerts and emergency calls stop working. The car’s microphone also stops, but a bypass kit can fix that. There’s a warning: if you use Bluetooth, the car can still send data through your phone’s internet. Using a USB cable for CarPlay avoids this. The guide lists all tools and steps clearly, with tips for safe removal and testing. In the comments, many people agree with the privacy concerns and praise the clear guide, but some worry about losing safety features or breaking the warranty. A few suggest buying older, less connected cars, while others want stronger privacy laws so such work isn’t needed.

Next, Amazonbot will start following robots.txt rules for web crawling, starting June 15, 2026. The author got an official email from Amazon confirming this change. Before, Amazonbot often ignored robots.txt, scraping sites even when blocked. Now, website owners can control Amazonbot with standard robots.txt files. The email explains how to allow or block the bot at different levels, and notes that if no robots.txt is set up, Amazonbot follows normal crawling rules. The author jokes about the email and shares that they built their own tool, Anubis, because of Amazon’s old behavior. They plan to update Anubis for the new rules. Hacker News commenters have mixed feelings. Some are happy Amazon is finally respecting the standard; others say it’s strange it took so long. Some worry Amazon might still ignore the rules if it wants data badly enough, or point out that robots.txt is not a real law. People discuss the pros and cons for small website owners, and some joke about whether other bots will ever follow the rules like Google or now Amazon.

Now for some security news. A research team found the first public macOS kernel memory corruption exploit on Apple’s new M5 chip, even with Apple’s Memory Integrity Enforcement, or MIE, turned on. MIE is a special hardware system meant to block memory attacks. The team found the bug by accident, using both human experts and an AI tool called Mythos Preview. Their exploit starts as a normal user and ends with full system control, showing that MIE is strong but not perfect. They will share full technical details after Apple fixes the bug. Many commenters are impressed by how quickly a small team, with AI, could beat Apple’s best protections. Some worry that this means companies will have a harder time keeping systems safe as AI helps attackers. There’s debate about Apple’s hardware-based security, with some saying it slows users down. Others are thankful the team reported the bug responsibly, and some joke about the big difference between Apple’s fancy offices and hacker workspaces.

In another creative tech project, someone connected a desktop RTX 5090 GPU to an M4 MacBook Air using Thunderbolt eGPU docks, Linux VMs, and custom software. Macs don’t support NVIDIA GPUs natively, so the author set up a Linux virtual machine, passed the GPU through, and worked around hardware limits like DMA mapping caps. They patched the Linux NVIDIA driver to fix errors. Performance is mixed—at low resolutions, Apple’s built-in GPU is sometimes faster because of overhead from emulation and virtualization. At 4K, the eGPU helps a lot, making unplayable games playable, though still much slower than a gaming PC. AI tasks work much better, with big speedups for tasks like local LLM inference. Hacker News commenters are impressed by the technical effort but see it as a fun experiment, not practical for most people. Some hope for better native eGPU support on Macs in the future, while others point out the cost and complexity.

Let’s move to server security. “Nginx-Rift” is a new exploit for a serious bug, CVE-2026-42945, in Nginx’s rewrite module. This bug lets attackers run code on web servers if certain settings are used, and has existed since 2008. The problem comes from a memory miscalculation when using “rewrite” and “set” commands, leading to a buffer overflow and remote code execution. The exploit works on Nginx versions from 0.6.27 to 1.30.0 and Nginx Plus R32 to R36. The researchers used automated tools to find the bug, and shared steps to reproduce it. Many commenters are worried because Nginx is so widely used. Some are surprised the bug was missed for 18 years, others debate if all sites are at risk or just those with certain settings. Some note that patching is easy if servers are up to date, but many real-world servers run old software. There’s discussion about whether open source is safer, since code is open but bugs can still hide for years.

Now to another security story. Experts found a way to bypass firmware downgrade protection on the Tesla Wall Connector. Tesla’s charger uses two firmware slots, and new updates use a “ratchet” number to block downgrades. The updater checks the ratchet, but the bootloader does not. The trick is to update to new firmware, then erase the slot and write an older, signed firmware—after reboot, the bootloader runs the old version because it only checks the signature. The exploit takes about 30 minutes. The article suggests better protections, like checking the ratchet in the bootloader or forcing a reboot after updates. Tesla has now fixed the problem with an automatic update. In the comments, people praise the deep technical analysis and clever attack, but worry about risks if chargers are hacked. Some say this is a common type of bug when only part of a system checks security. There’s discussion about whether regular users could do this, and how to design updates better in the future.

OpenAI has added Codex management to the ChatGPT mobile app. Now you can manage Codex tasks on your phone, not just your computer. You can approve changes, answer questions, and watch live updates like code diffs and test results, all from your phone. Codex uses a relay system to keep your files and secrets safe—they don’t leave your secure machine. For businesses, Codex now supports remote environments with SSH, access tokens for automations, hooks to customize actions, and HIPAA compliance for healthcare. The new features are rolling out to iOS and Android, with Windows coming soon. In the comments, readers are excited about managing coding tasks on the go, but some worry about always being connected. There are questions about security and how smooth mobile code reviews will be. Some are excited about the enterprise features, while others wait to try Windows support. Most agree this is a big step for AI-assisted coding, but want to see how well it works in practice.

The job is about showing developers the price impact of their code before money is spent, using tools that put cost info into workflows like code editors and pull requests. The role involves creating content, building a community, and working with users. Benefits include remote work, good equity, and lots of paid time off. In the comments, some think cloud cost surprises are a big problem and like Infracost’s approach. Others debate if Developer Advocates really drive adoption, or if it’s just more marketing. Some worry that extra checks can slow coding, and others compare Infracost to similar tools. There are stories of surprise cloud bills, and debate about how to make engineers care about costs. Most agree that cloud cost control is important, but there’s no perfect solution yet.

Runo is a new web scraper API that claims to be 6-7 times more efficient than others. It turns any webpage into clean, typed JSON by sending a simple schema. No need to write selectors or parse HTML yourself. Runo handles both simple and JavaScript-heavy sites, even solving CAPTCHAs on higher plans. Pricing starts with a free plan, and you can buy more requests as needed. In the comments, users like the simple schema and clean results, as well as handling hard sites and bot protections. Some worry about API blocking or legal risks, and others wonder if “semantic extraction” will really work on all sites. There’s discussion about latency, rate limits, and the risks of third-party services. Overall, people see Runo as helpful for many scraping jobs, with the usual risks.

Finally, Start9 launched the RISC-V Router, aiming to make advanced networking and security easy for everyone. The hardware is open-source, using a RISC-V processor, open boot tools, Linux, and open schematics. The only closed parts are the WiFi firmware and two early boot binaries, but the team is working to open those too. The software, StartWRT, is a fork of OpenWrt with a modern, simple GUI. A key feature is Security Profiles, letting you control what each device can do, and multi-password WiFi for different access levels. You can chain VPNs for privacy and set up dynamic DNS easily. In the comments, people are excited about the open-source approach and using RISC-V in real products. Some mention the $300 price, but others say it’s worth it for privacy and openness. There are questions about hardware specs, software updates, and shipping timelines. Some are cautious about the non-refundable pre-order policy, while others understand the risks of open hardware projects. Overall, there’s excitement, careful optimism, and practical concerns about price and delivery.
