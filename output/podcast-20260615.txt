First, let’s talk about a recent scam targeting software engineers. A developer got a LinkedIn message from someone claiming to be a recruiter for a crypto startup. The recruiter asked him to check a GitHub codebase and look at some problems with outdated Node modules. The engineer was suspicious, so he opened the code in a safe, temporary server and only read the files. Quickly, he found a hidden file that quietly built a web address and ran any code sent from a remote server. This backdoor was hidden between many comments and looked like a normal test file. The trick was that running the usual npm install would execute the harmful code. The GitHub repo used the name of a real developer, and the recruiter used a real LinkedIn profile from someone with no tech background. When the engineer asked for help, the recruiter suddenly acted like a Node.js expert, which was strange. The engineer reported the fake repo and profile, but they were still online at the time of his article. In the comments, many people said these scams are getting more common and harder to spot. Some shared similar stories about fake job offers, and others discussed ways to stay safe, like using read-only tools or virtual machines. People also said it is important to check commit histories and not trust code from strangers, especially if asked by someone you don’t know. Some worried that reporting scams is too slow, and both GitHub and LinkedIn should do more. Many thanked the engineer for sharing his story as a warning to others.

Next, Iroh 1.0 is a new networking tool that connects devices using keys instead of IP addresses. The team believes this is a safer and easier way for devices to find each other, even behind firewalls or changing networks. Iroh uses keys to set up connections, manage identity, and handle permissions. It’s open source and already works on millions of devices, used for things like streaming, secure chats, file sharing, and games. Iroh 1.0 is the first stable version and supports modern networking features like QUIC multipath and NAT traversal. You can use it with many languages including Rust, Python, Node.js, Swift, and Kotlin. The team promises version 1.0 will stay stable and supports older versions with public relays. On Hacker News, many users liked the idea of using keys, saying it could fix old problems with networks. Some were excited for peer-to-peer apps, and others liked the many language supports. There were questions about security and privacy, and how well it works with strict firewalls. Some compared it to Tor and ZeroTier, and many hope it will help make the internet more private and less centralized. Overall, people are positive and eager to try Iroh in their projects.

For a bit of fun, TinyWind is a browser sailing game where you control a pirate ship using real wind physics. Players have already sailed more than 380,000 kilometers. You adjust sails and steer, watch the changing wind, collect coins, and rescue sailors. The game is simple and colorful, with retro graphics. It’s easy to start but hard to master, especially with realistic wind mechanics like tacking. On Hacker News, people liked the relaxing but challenging gameplay and the simple design. Some shared sailing tips and memories, and others wished for more features or mobile support. Overall, it’s a fun way to learn about sailing.

The developer runs services in Docker and used to spend hours updating and testing. Now, with AI help, they can read summaries and add health checks quickly. OpenCode runs on a VM with its own keys, so the AI cannot push changes directly. Instead, it creates pull requests, which the developer reviews. Finally, a new article explains a way to break weak RSA keys called “short-sleeve” keys using polynomials. These weak keys appear when the random numbers are not truly random, making the key numbers mostly zeros in a pattern. The authors found hundreds of such keys in the wild, often due to a software bug in CompleteFTP from 2016 to 2023. The bug caused keys to have too many zeros, making them easy to factor. The CompleteFTP team fixed the bug and made tools to help users replace bad keys. In the comments, people liked the clear explanation and the new attack idea. Some are worried about old servers still using weak keys. Others say this shows why using standard crypto libraries is safer than writing your own. There are also technical discussions about the attack, and some share stories of finding similar bugs before. The main lesson is that small mistakes in cryptography can create big risks, and it’s important to keep checking for these problems.
