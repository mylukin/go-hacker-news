Next, we look at a technical project: building a medieval island map with hex tiles and the Wave Function Collapse algorithm. The map is computer-generated, not hand-drawn, and includes roads, rivers, forests, and villages. WFC works by making sure each tile’s edges match its neighbors, like a puzzle. Hex tiles are harder than squares, with six edges and many possible rotations and heights. The author divides the big map into smaller grids and uses tricks like backtracking and “local WFC” to fix stuck spots. Sometimes, a mountain is used to hide a problem, since mountains can fit anywhere.

The map also uses Perlin noise to place trees and buildings in groups, because WFC alone doesn’t do that well. Water is hard to make look good, so the author uses special effects for waves and sparkles. The hex grid math uses “cube coordinates” to handle neighbors easily. The 3D tiles are made in Blender, and everything is drawn efficiently to run smoothly even on a phone.

Commenters are impressed by the graphics and technical details. Some share their own experiences with WFC and agree that hex tiles are much harder than squares. Others discuss possible improvements, like better water effects or more natural terrain. Many like how real the map feels, and some are reminded of games like Carcassonne. The code and demo are seen as great for learning, and most users just enjoy exploring the map.

Now to the browser: JSLinux has released support for running full 64-bit x86 Linux systems right in your web browser. This is a big step up for the project, which was already known for running older or smaller systems. Now, you can boot Alpine Linux with support for new CPU features like AVX-512, just by clicking a link. Other systems like Windows 2000, FreeDOS, and RISC-V Linux are also available. You don’t need to install anything, and some systems let you save and load files.

Many in the comments are amazed, calling it “magic” or “science fiction.” People say it’s great for testing and teaching, even if it’s slower than real hardware. There’s talk about how browser tech has improved, how AVX-512 support works, and the possibilities for running other systems or even games. Some remember using older JSLinux versions and are happy to see the project still growing. A few mention security worries, but most see it as a great demo and learning tool.

Now to open source: there is a debate about whether it is enough for something to be legal, or if it also needs to be fair. The story is about the Python library chardet, which was remade using AI. The new version was built by giving only the old API and tests to an AI, not the source code itself. The result is much faster and different, and the maintainer changed the license from LGPL to MIT. Some big open source names support the relicensing, saying it’s legal if you don’t copy code directly. Others, including the original author, say this breaks the rules and spirit of copyleft.

The article points out that copyleft says changes must stay open, while permissive licenses allow code to be closed. The main worry is that if AI makes it easy to get around copyleft, open source may lose trust. Some say new rules are needed to protect not just code, but also APIs and tests.

In the comments, people are divided. Some feel using AI to bypass copyleft feels wrong, others think the law is clear and developers should adapt. There are calls for stronger licenses, and worries that fewer people will want to contribute to open source if their work can be taken so easily. Some say open source is about sharing and making things more open, but many agree that the line between what is legal and what is fair is more important than ever.

Next, we look at Mog, a new programming language for AI agents to write and run code safely. Mog is similar to Lua but has static types, and is designed for speed and security, especially where AI writes or changes code. Mog is compiled, has clear types, no operator precedence, and no implicit conversions. All system actions are controlled by the host program, using a capability system. Scripts only get the permissions they need, making it safer than letting AIs use Bash or Python.

Mog supports modern features like async/await, pattern matching, and native tensors. You write Mog code in `.mog` files, compile them with a Rust-based compiler, and run them as standalone programs or plugins. The syntax is simple and designed to be easy for LLMs to handle. Error handling is strict, with no exceptions—only Results you must match on.

In the comments, people like the idea of a strict and AI-friendly language for plugins. Some say it prevents bugs, especially when code is written by AIs. Others wonder if we really need another language, since Lua and WASM already exist, but note that Lua is hard to sandbox and WASM is heavier. Some ask about real-world security and want more built-in libraries, but see the small core as a feature. Overall, Mog is seen as an interesting experiment, and people are waiting to see if it will catch on in real AI agent systems.

Now let’s travel back in time with the story of Thomas Selfridge, the first person to die in a powered airplane crash. In 1908, Selfridge, a young officer and aviation supporter, flew as a passenger with Orville Wright during a demonstration for the Army. The plane crashed after a propeller broke, and Selfridge died from a head injury—pilots did not wear helmets then. The Army later required helmets, and the Wright brothers improved their designs. Selfridge is remembered not just for his death, but for his work as an engineer and early flight supporter.

Hacker News users reflect on the risks of early flight, how safety rules often come after accidents, and how innovators like Selfridge helped make flight safer for everyone. Some say history should remember achievements, not just how people died. Others are surprised at how quickly aviation moved from risky experiments to reliable transport. The story reminds us that every new technology has its first accidents, but those lessons make things safer in the end.
