- 原文链接: [Windows 9x Subsystem for Linux](https://codeberg.org/hails/wsl9x)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48120162)

This article is about WSL9x, a project that lets you run a modern Linux kernel inside Windows 9x without rebooting. It mixes Windows and Linux systems, so you can use programs from both at the same time, using features like memory protection and multitasking.

WSL9x has three main parts: a special Linux kernel, a driver for Windows 9x, and a small DOS program called `wsl.com`. The driver loads the Linux kernel into memory, sets up some technical things like threads and stacks, and helps Linux and Windows talk to each other. When you run a Linux command, the driver checks if Windows can handle it; if not, it sends it to Linux. The Linux kernel has been changed to work with Windows, using Windows system calls instead of normal Linux ones. The `wsl.com` program lets you use the normal DOS window as a Linux terminal, so you can type Linux commands right inside Windows.

To build and use WSL9x, you need special tools to build Linux and Windows code. You need a Windows 9x disk image ready, then you build the kernel, copy files, and run a command at the DOS prompt to start using Linux. If you want colorful text, you can add a DOS driver for ANSI colors.

Some asked why you would want to do this, since Windows 9x is old and not very safe. Others said it’s a fun and clever technical challenge, and they liked seeing old and new tech work together. A few people talked about the hard parts of making Windows and Linux share things like memory and interrupts. Some wondered if it’s just a joke, but others pointed out the real technical skill involved—especially rewriting parts of Linux to use Windows APIs and making the two systems work together without crashing. ## SANA-WM, a 2.6B open-source world model for 1-minute 720p video

- 原文链接: [SANA-WM, a 2.6B open-source world model for 1-minute 720p video](https://nvlabs.github.io/Sana/WM/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48159445)

SANA-WM is a new open-source AI model from NVIDIA that can turn just one image and a camera path into a full minute of 720p video, all on a single GPU. The model has 2.6 billion parameters and is designed to make long, detailed videos that follow exact camera moves, and it does this faster and more efficiently than many older, bigger models.

The main ideas behind SANA-WM are: first, it uses a special kind of “hybrid linear attention” to make longer videos without using too much memory. Second, it controls the camera’s position in 3D space very carefully, so the viewer can move through the scene in any direction. Third, it has a two-step process: the first part makes the basic video, and the second “refiner” makes it look sharper and more detailed. The team trained the model using about 213,000 public video clips with accurate camera movement labels, finishing the whole training in just 15 days with 64 powerful GPUs. Once trained, the model can make a minute-long video in about half a minute on just one good GPU.

The model can create many different scenes: mountain trails, forests, caves, deserts, underwater temples, and even spaceships and alien landscapes. All these videos start from a single still image, and SANA-WM fills in the motion and details, like snow falling, fog drifting, or fireflies flying. The videos are very consistent and natural, even over a whole minute. The model can even follow complex camera paths exactly, letting users control how the scene changes over time. SANA-WM’s output quality is close to large, closed models, but it uses less computing power and is fully open-source.

From the Hacker News comments, many people are impressed by the speed and quality, noting that making a one-minute, 720p video on a single GPU is a big step forward. Some users say that the detailed, stable scenes are much better than what earlier open-source models could do, especially for long videos without flicker or weird changes. Others discuss how SANA-WM could help with animation, filmmaking, or game design, since you can get a whole minute of video from just a single image and prompt.

Some commenters ask about the real-world uses—will this replace human animators, or just be a tool? A few are curious about the training data: is 213k clips enough, and does this model generalize well to new scenes? There are questions about how SANA-WM compares to Google and OpenAI’s bigger, closed models—some think open-source is catching up fast. Others bring up technical points, like how the hybrid attention works and how the two-stage process helps keep the video good from start to finish.

A few users worry about hardware access, since training still needs many expensive GPUs, but most people like that running the model is much easier. Overall, users see SANA-WM as a big step for open video AI, making long, high-quality, controllable videos possible for more people.

## A molecule with half-Möbius topology

- 原文链接: [A molecule with half-Möbius topology](https://www.science.org/doi/10.1126/science.aea3321)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48109756)

This article talks about a new molecule that has a “half-Möbius” shape, which is very unusual in chemistry. The scientists made a ring of 13 carbon atoms (with 2 chlorine atoms) that has a twist, but not a full Möbius twist—only half.

Normally, a Möbius strip is a band with a single twist, making it have only one side. In chemistry, molecules can sometimes have similar twists in their bonds and orbitals, which changes their properties. Here, the molecule does not twist 180°, like a full Möbius, but 90°, so the special features only repeat after going around the ring four times, not two. This is why they call it a "half-Möbius" topology.

The team built this molecule by removing chlorine atoms from a bigger molecule, one atom at a time, using a very sharp tip on a cold surface. They used special microscopes to look at the atoms and the way the electrons move around the ring. They found that the electrons form helical (spiral-shaped) patterns, which fit the half-Möbius idea.

They could also switch the molecule between different states: two chiral (mirror-image) forms, each with the half-Möbius twist in the opposite direction, and a flat “normal” state without any twist. This switching was done by touching the molecule with the microscope tip. The team also ran big computer calculations, some even on quantum computers, to show how the electrons and atoms change during this process.

This discovery matters because molecules with strange topologies can have new kinds of stability and different ways for electrons to move, which could help with things like molecular electronics or quantum computing. Usual chemistry rules for rings (like Hückel or Möbius rules about aromaticity and stability) do not apply here, so this is new ground.

In the comment section, many users are excited about the idea of a half-Möbius molecule and praise the clever experiments. Some people are interested in how this might change our understanding of molecular electronics, especially for future computers. Others ask if these molecules could be made in larger amounts, or only one at a time with special equipment. A few users wonder about the real-life uses: will these shapes make better materials, or are they just a scientific curiosity?

Some comments point out that it’s hard to see how these effects would scale up beyond single molecules. Others are hopeful that this will push chemistry to find more strange and useful shapes. There are questions about the quantum computer calculations—some users want to know if quantum computers are really necessary, or if normal computers could do the job. Overall, the discussion is positive, with most people agreeing this is a creative and important step in molecular science.

## Halt and Catch Fire

- 原文链接: [Halt and Catch Fire](https://unstack.io/halt-and-catch-fire)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48162468)

This article talks about the term “Halt and Catch Fire” (HCF), which started as a joke in the computer world but has real history. HCF is an old phrase for a computer instruction that makes the CPU stop working, sometimes in a way that only turning the power off can fix. The “catch fire” part is a joke, but in some cases, like with old IBM machines, bad instructions really did make parts overheat.

The article explains that HCF is used for instructions that make the processor freeze or break, whether by accident or on purpose. Programmers liked to give these instructions three-letter names, and HCF joined other funny ones like “EPI” (Execute Programmer Immediately) and “CRN” (Convert to Roman Numerals). The story focuses on the Motorola 6800 chip, which had extra, undocumented instructions. If you ran the wrong code, the chip would go into a strange state, reading every part of its memory over and over. On some hardware, this could even change what you saw on the screen.

A 1977 article in BYTE magazine described two dangerous codes for the 6800, which the author called HCF. The chip would get stuck, moving through memory like a counter, and nothing could stop this except a reset. Other engineers called these codes things like “Drop Dead” because of how they locked up the system. Sometimes, engineers used them on purpose because the signals made it easy to check things with special tools.

Later, Motorola engineers explained that this odd behavior was even kept on purpose because it helped test the memory quickly. Newer tests have shown exactly how the chip acts when these codes are run: it takes a short pause, then starts quickly running through addresses, and there are even more unknown codes with similar effects.

The article also says other processors had these problems, like the 6502 and the famous Pentium F00F bug, where special instructions could crash or freeze the computer. Today, people still use random data to test for bugs in chips and software—an idea called fuzzing.

In the end, the writer enjoyed exploring this piece of history and reminds us that even modern software depends on hardware, which can still go wrong. “Halt and Catch Fire” is a catchy phrase, and the writer jokes it might be a good name for a future project.

In the comments, many readers share stories about old computers and how they broke or acted strangely when weird instructions ran. Some remember the TV show “Halt and Catch Fire” and how it made these technical ideas popular. Others say that while the phrase is funny, real hardware bugs can be serious and hard to fix, especially when you cannot recover except by cutting the power. A few people bring up similar bugs in other chips, like the 6502 or early Intel CPUs, and talk about their own experiments with illegal opcodes. Some users note that fuzzing for hardware bugs is still important for finding new security problems. Others point out that most modern software developers never see these kinds of bugs because they work so far away from the hardware. A few readers joke about making their own “HCF” commands for fun, while others think the phrase is overused or misunderstood. Some people are impressed by how much old hardware could be “broken” just by running the wrong code. Finally, there’s agreement that “Halt and Catch Fire” is a perfect example of old-school computer humor that still makes people smile.

## Δ-Mem: Efficient Online Memory for Large Language Models

- 原文链接: [Δ-Mem: Efficient Online Memory for Large Language Models](https://arxiv.org/abs/2605.12357)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48158506)

This article talks about δ-mem, a new way to help large language models remember and use information from past conversations. Usually, making these models remember more means giving them a bigger context window, but that is expensive and not always useful.

The authors suggest a different idea: instead of making the memory bigger, they add a small, smart memory system to the model. This memory is very compact, using only an 8x8 matrix. The system learns using something called the “delta rule,” which means it updates its memory just a little bit at a time as new information comes in. When the language model needs to generate text, it uses this memory to make small changes (called “low-rank corrections”) to how it pays attention to past words.

This new memory system does not need to retrain or change the main model. It does not need to make the context window larger, either. Even with such a small memory, the model does better on tasks where remembering past information is important. For example, it scores about 1.1 times better than the original model and about 1.15 times better than other memory methods. On some hard memory tests, it can even do 1.3 times better. Importantly, this method does not make the model worse at other tasks.

In the Hacker News comments, some people are excited about the practical use of δ-mem. They like that it helps models work better without needing more computer power. Others ask about the delta rule—some remember it from older AI ideas and are happy to see it used in a new way. A few users wonder if this small memory is enough for real-world use, or if it will hit limits soon. Some point out that not needing to retrain the whole model could save lots of money and time.

There are questions about how easy it is to add δ-mem to different models. A few users want to see open-source code or a demo. Others worry about security: could this memory system leak private data? Some think this approach is a good step toward making long-term AI assistants. But a few are unsure if it will work as well when the model faces very complex or very long tasks. Still, most agree that this is a smart and simple way to make language models remember better.

## Fame! A Misunderstanding: A new translation of Albert Camus's complete notebooks

- 原文链接: [Fame! A Misunderstanding: A new translation of Albert Camus's complete notebooks](https://lareviewofbooks.org/article/albert-camus-complete-notebooks-ryan-bloom-existentialism-absurd/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48129593)

The article is about a new translation of Albert Camus’s complete notebooks, which helps correct misunderstandings about his ideas and reputation. For a long time, people have thought of Camus as an “existentialist” or a philosopher of the “absurd,” but this new translation shows he saw himself more as an artist and writer.

The translation by Ryan Bloom brings together all of Camus’s notebooks from 1933 to 1959, including the important Oran Notebook, which was just found in 1988. This notebook shows Camus’s thoughts while writing his famous works, _The Stranger_ and _The Myth of Sisyphus_. The article explains that Camus did not want his books to be “thesis-novels” that just teach a philosophy. He wanted to show human experience, not only ideas.

Camus thought that mixing philosophy and fiction too much makes stories boring and artificial. He first liked the idea of writing philosophy in novels, but after reading Sartre and Kafka, he changed his mind. He believed artists should describe and feel life, while philosophers try to explain and solve things, which can take away the real experience.

The notebooks reveal that Camus wanted to show the importance of the body, experience, and the present moment, not just abstract thinking. He argued against violence and political murder, saying that turning people into “ideas” or “enemies” is dangerous. He also wrote about his own struggles with depression and how this changed his way of thinking.

Many people, including other writers like Sartre, misunderstood Camus and labeled him as an existentialist, even though he clearly said he was not and explained why. This mistake became popular because of book marketing, journalism, and academic habits. Even after Camus said he was not an existentialist, publishers and critics kept calling him one, and this idea became so common that it was hard to change.

Others say that labels like “existentialist” are often used by marketing and media to sell books, not to explain ideas. A few point out that Camus himself struggled with fame and did not want to be known just for a label. Some commenters discuss how hard it is to change public opinion once a wrong idea becomes popular.

Several users share how reading Camus’s notebooks or essays changed their understanding of his work, making them see him as a writer focused on life and art, not just abstract ideas. Others debate if it really matters what label we use, as long as people read and enjoy the books. Some warn that academic or media simplifications can hurt the real meaning of an author’s work. A few highlight the importance of new translations in helping us see classic writers in a new way. Overall, the comments show excitement about the new translation and hope that it will help readers understand Camus better.
