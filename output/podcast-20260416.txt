First, OpenAI released a major update to Codex, their AI tool for developers. Codex is now much more than a coding assistant. It can use your computer almost like a person, moving the mouse, clicking, and typing. It works with more apps, can review pull requests, open files and terminals, connect to remote computers, and even use its own web browser. Codex can also make images and designs with the new gpt-image-1.5 model, which is great for frontend and game design.

Codex now supports over 90 plugins, so it can work with tools like Jira, GitLab, Slack, Gmail, and Notion. It can open and preview files like PDFs and spreadsheets, track its own actions, and summarize plans. Codex remembers your past actions and settings, so you can pick up work where you left off. It can even schedule and finish tasks that last days or weeks, and suggests what to do next by looking at open comments and messages.

These updates are now available for Mac users and will come to more people soon. Codex aims to help developers not just write code, but also understand systems, debug, and manage projects. OpenAI says this is a step toward making AI more helpful for everyone.

There are questions about how well Codex understands complex projects. Some have used Codex for small, repeatable jobs and found it helpful, but others say it needs better controls before they trust it with important work.

Next, Anthropic released Claude Opus 4.7, an upgrade to Opus 4.6. The focus is on better coding skills, improved image vision, and higher reliability for hard tasks. Opus 4.7 handles long and complex coding jobs with less help, follows instructions more closely, and checks its own work before finishing. It now supports higher-resolution images, making it better at reading screenshots and diagrams.

Testers found it much better at software engineering, solving harder problems and finding more bugs. In finance and legal work, it gave more accurate answers and understood tricky requests. Some teams said it worked for hours without losing focus and could build complex tools, like a Rust text-to-speech engine, on its own.

Opus 4.7 is also safer, with new safeguards against bad cybersecurity use. Security pros can join a special program to use it for good purposes. The model is not as strong in cyber tasks as Anthropic’s Mythos Preview, but it’s a big step forward.

Other new features include control over "effort levels," a new review command for code checking, and better memory for long tasks. Prices stay the same, but the new tokenizer may use more tokens for the same input. Anthropic offers tips to control costs and a guide for moving from 4.6 to 4.7.

Top comments say Opus 4.7 is a strong competitor to GPT-4 and Gemini, especially for coding. Users like Anthropic’s honest reporting of limits and the way the model pushes back in technical discussions. Some note that old prompts may need to change due to tighter instruction-following. There’s excitement about better vision and handling of complex workflows. Some worry that strict safeguards might block useful cybersecurity research, while others are glad Anthropic is careful. Users compare Opus 4.7 to other models and note that higher token counts could raise costs. The new “ultrareview” command and better memory are welcome, but some say creative tasks still need work. One user points out that real-world use matters more than benchmarks. Overall, the community sees Opus 4.7 as a big upgrade and will watch how it performs as more people use it.

Now, some good news for R developers. Tree-sitter, a fast code parser written in C, now works well with R thanks to a new grammar file. Tree-sitter helps computers understand code structure, which is useful for formatting, finding functions, and giving help in editors. Before, R had its own ways to parse code, but Tree-sitter can parse many languages and is faster, especially for live editing.

With the new grammar file made by Davis Vaughan and others, tools can use Tree-sitter for R. This powers tools like Air (for reformatting code), Jarl (for linting and fixing), and the Positron IDE (for autocomplete and help). GitHub now uses Tree-sitter for better search and navigation for R code. Tree-sitter-based tools can also help with searching, finding dependencies, refactoring, and mutation testing. Many of these tools are built in Rust, making them fast and easy to use with big codebases.

In the comments, people are happy to see Tree-sitter used for R and share how it has improved other editors like VS Code and Neovim. Some talk about the hard work needed to write grammar files and thank the team for making one for R. There are questions about keeping grammar files up to date, but Tree-sitter has an active group of maintainers. Users also like that Rust makes these tools fast, and they say Tree-sitter is already good with tricky R code but can get even better. Overall, the community is excited about Tree-sitter’s future in R and open tools that make coding easier and more fun.

Next, there’s a new problem in AI: not enough powerful computer chips for everyone. Renting Nvidia’s newest AI chips is now almost 50% more expensive than two months ago, and companies like CoreWeave want customers to sign long contracts. Big companies like OpenAI say they cannot do all their projects because of limited compute. Some top AI models are only available to about forty other companies, not everyone. This is a big change from when new AI models were easier to access.

The article lists five signs of this “scarcity era.” First, the best AI models will only be sold to top customers. Second, using the best AI will become very expensive. Third, even if you pay, you might have to wait because computers are busy. Fourth, prices will keep going up as demand grows. Fifth, developers will need to use smaller models or build their own systems until more chips and data centers arrive.

Many commenters agree that chip shortages are making AI work harder, especially for small companies. Some say this could slow AI progress and make it less fair. Others think the shortage will push people to build better or more efficient models. Some worry that open-source projects will suffer because they can’t pay for chips. Others compare it to the old days of fighting for cloud servers. Some think this problem will not last forever, as more data centers and new chips are built. There are also business ideas for offering smarter or cheaper chip solutions. A few warn that if AI stays expensive, it could slow down progress in all of tech. Overall, people are worried about fairness, speed, and new business ideas in this age of AI scarcity.

Let’s talk about Google’s new Android CLI. This tool helps developers build Android apps much faster, with or without Android Studio, and works well with AI agents. Android CLI is a command-line tool to set up environments, create projects, and manage devices. You can use commands like “android sdk install” and “android create” to quickly start projects from official templates.

Another feature is “Android skills”—small instructions for common tasks like navigation or moving from XML to Compose. These help AI agents follow Google’s latest advice and avoid old or bad patterns. Developers can add, use, or share these skills easily. The Android Knowledge Base is a resource that gives AI agents the newest official documentation and tips, even if their training data is old.

You can start a project with the CLI and later move it into Android Studio, which now has built-in AI agents to help with every step. There’s a new AI-powered project flow to help prototype ideas quickly, and these tools support all Android devices.

In the comments, developers are excited about faster builds and easier automation, especially for CI pipelines. People like being able to use any AI agent, not just Google’s, for more flexibility. Some are happy to see tools that keep up with Google’s best practices and cut down on confusing steps. Others worry that relying on AI agents may hide what’s happening under the hood, making it harder for beginners to learn. There are questions about real build speeds for large apps and how well it works with custom setups. Some miss the old, simpler days of Android builds and hope these changes won’t add too much complexity. Overall, the feeling is positive, but people want to see real-world results and hope Google keeps the tools open and stable.

Now, a fun hardware story. A hacker built a DIY robot arm called AutoProber that uses AI to help with hardware hacking. The robot is made from cheap parts like duct tape, an old camera, and a CNC machine. It scans electronic boards, finds parts like pins and chips, and probes them safely. The robot uses a microscope and a custom probe to look at the board, take many small photos, and join them into a big map. It marks pins and shows them on a web dashboard, so you can choose which ones to test. If you approve, the robot will touch those pins and report the results.

AutoProber has strong safety steps. It checks a special signal to avoid moving if something is wrong, and stops if there’s a problem. All control can be done from a web dashboard, Python scripts, or the AI agent. The project is open source for noncommercial use, with full code and setup instructions on GitHub. The creator warns to use it only in labs where you have permission.

Hacker News users are impressed by the creativity and the use of cheap parts. They like that the code and setup are open and easy to follow. Some say automatic probing is tricky and can damage things, so safety features are key. Others share their own CNC stories and wonder if AI really helps or if most of the work is still manual. People like the idea of making hardware hacking easier for beginners. There are questions about using the tool in factories, but some worry about the business license. Overall, users agree that this DIY tool can help more people learn electronics and security.

Next, the Qwen team released Qwen3.6-35B-A3B, a new open-source AI model for coding and reasoning. It uses a “mixture-of-experts” method, with 35 billion parameters in total, but only 3 billion active at once. This design means the model works faster and uses less memory than older models, but still does well in coding and reasoning tasks.

Qwen3.6-35B-A3B beats earlier Qwen models and matches or passes popular models like Gemma4-31B, especially in coding and multimodal tasks. It is free to download or use via API and works with tools like Qwen Code, OpenClaw, and Claude Code. It scores high in coding and reasoning tests, and is also good at understanding images and videos. The open weights mean anyone can self-host or build new tools. The team wants feedback and plans to release more models.

In the comments, users are excited about the strong performance with few active parameters. Many think the mixture-of-experts idea is clever and efficient. Some users share good test results, but others note the model is still slower than smaller ones on weak hardware. There are questions about the license and if it is truly open for commercial use. Developers are happy about support for open-source coding agents, and some wonder how well the model’s vision works in real projects. Others like the easy setup and code examples. Many are glad to see strong open models from outside the US and hope this will encourage more global AI development.

Finally, Cloudflare announced a new AI platform that lets developers use many AI models from one place. The platform makes it easy to build smart agents that can switch between models and providers. Developers get a single API to call models from over 12 providers, like OpenAI, Google, and Alibaba Cloud. Switching models just takes one line of code. Developers can track all AI costs in one dashboard, add custom tags, and soon bring their own models using a tool called Cog.

The system sends responses as soon as possible, making it fast. Cloudflare’s many data centers mean responses are quick worldwide. If a provider goes down, Cloudflare can switch to another automatically. Streaming responses are saved, so if a connection drops, the agent can continue without starting over.

In the comments, users like having one API for many models, saying it saves time and helps try new models. Some are happy about tracking costs from different providers. Others like that you can bring your own models for privacy. Some worry about lock-in and what happens if Cloudflare raises prices or changes things. There are questions about how failover works and if the extra layer slows things down, but some think worldwide data centers make it faster. Some developers are excited to try the new features, while others want to see real-world results. Overall, there is both excitement and careful questions about how the platform works and what it means for building smart, reliable AI agents.
