## Show HN: Moongate – Ultima Online server emulator in .NET 10 with Lua scripting

- 原文链接: [Show HN: Moongate – Ultima Online server emulator in .NET 10 with Lua scripting](https://github.com/moongate-community/moongatev2)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=47275236)

This article shares Moongate v2, a new Ultima Online server emulator made in C# using .NET 10. The project aims to give a high-performance, modern server with Lua scripting while keeping the classic Ultima Online feeling.

The server is built from scratch, not copied from other server projects, though it borrows ideas from older projects like POLServer and ModernUO. Moongate uses a modular structure, clear network and game code separation, and focuses on correctness and easy updates. It uses AOT (Ahead-of-Time) compilation for better speed and lower memory use. Packet handling is typed and source-generated to avoid slow reflection, which helps with fast startup and smooth AOT builds. The game world is divided into sectors (chunks), loaded only as needed, making it efficient for big worlds with many players. The server uses Lua scripting, so custom actions and behaviors can be added easily. Lua scripts can control NPCs, handle events like player speech, and create UI elements (gumps). The system supports data-driven templates for items and characters, loaded from JSON files. Persistence uses MessagePack for fast, small binary saves, with snapshot and journal files to keep the world state. There’s a built-in email system for account actions like password recovery. The project has good test coverage, benchmarks for all hot paths, and a friendly console and web admin UI. Docker support and monitoring with Prometheus and Grafana are included for easy hosting.

In the Hacker News comments, many users express nostalgia for Ultima Online and excitement to see new projects for such an old game. Some praise the technical choices, like chunk-based world loading, AOT compilation, and using Lua, saying these make the project modern and scalable. Others ask about compatibility with original clients, and some share their own stories of running UO shards or writing emulators. One person wonders if there could be legal risks with EA, the owner of Ultima Online, but others point out that many emulators exist and are usually safe if not used for profit. Some are surprised to see a C# game server for such an old game, but agree it’s a good language for this. A few people discuss how hard it is to fully support all game features, and how scripting helps, but is also a challenge. Others ask about the current development status and how they can help or contribute, since the project is looking for more developers. There is also mention of the project’s clean code, active development, and good documentation, which encourages people to try it. Finally, some users talk about the fun and community around running private UO servers, and hope that projects like Moongate can help keep the old MMO spirit alive.

## Hardening Firefox with Anthropic's Red Team

- 原文链接: [Hardening Firefox with Anthropic's Red Team](https://www.anthropic.com/news/mozilla-firefox-security)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=47273854)

Anthropic worked with Mozilla to use their AI, Claude, to find security problems in Firefox. In just two weeks, Claude found 22 bugs, and 14 of them were serious—making up almost one-fifth of all high-severity Firefox issues fixed that year.

The team first tested Claude on old Firefox bugs to check its skill, but then asked it to find new problems in the latest version. Claude was able to spot a tricky bug in the JavaScript engine within twenty minutes. The team helped Mozilla by sending detailed bug reports, and Mozilla quickly fixed most of these problems for all Firefox users.

While Claude is very good at finding bugs, it is not as good at turning them into working attacks (called exploits). The AI only managed to make two simple exploits out of hundreds of tries, and they only worked in a special test setup without some real browser protections. This means that while AI is powerful at finding issues, it is still harder for it to actually break software security.

Anthropic shared tips for using AI in security work. They say it helps to use “task verifiers”—tools that check if the AI's bug fixes really work and do not break anything else. They suggest that when sending bug reports, researchers should include clear test cases, proof-of-concept steps, and suggested fixes. This helps software makers trust and use AI-found bugs.

The company warns that AI may soon get better at making real attacks, not just finding bugs, so there is a short time for defenders to fix software before attackers catch up. They plan to keep working with open-source projects and share their results.

Hacker News users had mixed reactions. Some were impressed with how fast and well the AI found real problems in such a major project. People liked seeing AI used to help with defense, not just attack. Others were worried that soon, AI could make hacking much easier for bad actors, and this could speed up the arms race between attackers and defenders.

Some commenters questioned if AI bug-finding would make software safer or just flood teams with too many reports, including false alarms. A few thought that while AI can help, human experts are still needed, especially to judge which bugs matter most. There were also practical worries: if AI can make exploits, should the public have access to these tools? Others pointed out that open-source projects may not have the resources to handle a big jump in bug reports.

A few users shared that more automation could help busy maintainers, but only if the reports are high-quality and easy to verify. Some liked Anthropic’s idea to share clear, reproducible steps with each report. Others hoped AI would soon help with fixing bugs, not just finding them. Finally, some people said this work is a sign of the future—AI will be a key tool for both making and breaking software security.

- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=47219462)

Open Camera is a free, open source camera app for Android phones and tablets. It gives users more control over taking photos and videos compared to most default camera apps.

The app has many features: it can auto-level photos, supports different scene modes, color effects, and manual controls like ISO and white balance. You can use a timer, take photos by making a noise, and change what the volume buttons do. It has special tools for photographers, like grids, crop guides, histograms, and focus peaking. Open Camera can add GPS location, date, time, and even custom text to your pictures. It supports HDR, panorama mode (even with the front camera), RAW files, slow motion, and noise reduction for low light. You can also remove photo metadata for privacy. The app works with the newer Camera2 API for advanced controls, if your device supports it. There are no ads in the app, and it is completely free. The source code is under GPL v3, so anyone can look at it or help improve it.

Open Camera needs Android 5.0 or higher, but older versions support Android 4.0.3. Some features may not work on all devices, because not every phone has the same camera hardware or software support. The developer, Mark Harman, says he cannot test the app on every phone, so users should test it before using it for special events. The app uses icons from Google’s Material Design and other sources, all with proper open licenses. There is a clear privacy policy, and users can choose if they want to share any data.

In the Hacker News comments, many users praise Open Camera for being free and open source. People like that it does not show ads or track users, unlike many other camera apps. Some users say they use Open Camera all the time, especially on older or cheaper phones where the stock camera app is bad. Others mention that Open Camera is great for privacy, as it can strip metadata from photos. A few people say they wish it had even more manual controls, or that the interface was easier to use. Some users report bugs on certain devices, but most agree it is reliable and powerful for a free app. There are also comments about how rare good open source Android apps are, and how important it is to have choices outside of Google or phone makers’ software. Some suggest donating to the developer or helping with code. A few users wonder if similar open source camera apps exist for iOS, but most agree Android is more open for this kind of app. Many believe Open Camera is a great example of a useful, high-quality free app that respects users.

## Payphone Go

- 原文链接: [Payphone Go](https://walzr.com/payphone-go/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=47220411)

Payphone Go is a game where people search for working payphones in California and use them to score points. Players create an account, find a payphone using an online map, and then call a special toll-free number to check in and earn points.

The first person to use a payphone gets the most points, while later callers get fewer. Each payphone can only give full points once per player, but you can still call again for fun or to be added to the log. The leaderboard shows who has found the most payphones and scored the most points.

Payphone Go uses the phone number of each payphone to track where calls come from. The creator got the payphone list from official records, but some payphones on the list might not exist anymore. If you find a mistake, you can email the creator to fix it. The game is meant to celebrate payphones, which are rare and disappearing.

People on Hacker News found the idea creative and fun. Some remembered using payphones as kids and liked the “treasure hunt” style of the game. Others wondered how many working payphones are left and shared stories about finding broken or missing ones. A few users said the point system makes it feel like a real-life video game, while some asked if there could be versions for other states or countries. A couple of comments talked about the technical side—how caller ID works and how the database matches numbers. Some worried the list might be out of date, but others liked that you can help update it. Several people saw it as a way to preserve memories of old tech, and many praised the project as a cool mix of nostalgia, hacking, and real-world adventure.
