## Bitwarden CLI compromised in ongoing Checkmarx supply chain campaign

- 原文链接: [Bitwarden CLI compromised in ongoing Checkmarx supply chain campaign](https://socket.dev/blog/bitwarden-cli-compromised)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=47876043)

Bitwarden CLI, a command-line tool for the Bitwarden password manager, was hacked as part of a bigger supply chain attack linked to Checkmarx. The hack happened in the npm package version `@bitwarden/cli2026.4.0` and used a file called `bw1.js` with dangerous code. Attackers got into Bitwarden’s build process by using a broken GitHub Action, similar to other attacks in the same campaign.

The bad code in `bw1.js` did many harmful things. It tried to steal secrets like GitHub tokens, AWS keys, and other important files by searching the memory and environment of the system. The code also sent stolen data to a server called `audit.checkmarx.cx` and even made fake GitHub repositories with Dune-themed names. Sometimes, the code used the GitHub API or npm registry to send out stolen information.

The attack used tricks to stay hidden. It would not run if the computer’s language was Russian. It made sure only one copy was running by creating a lock file. It also changed the user’s shell files to run itself again later. The malware tried to look like it was part of a big mission, using strange names and messages about "resistance against machines."

If you use Bitwarden CLI, the article says you should check your systems right away. Remove the bad package, change all your secrets, and look for odd files or GitHub activity. Watch for new repositories with Dune-related names, strange workflows, or files like `/tmp/tmp.987654321.lock` and changes in `~/.bashrc` or `~/.zshrc`. Also, check if your tokens or cloud keys were used in odd ways.

Hacker News users had strong reactions. Some were shocked that a popular tool like Bitwarden could be hit, saying it shows open-source supply chains are very risky. Others pointed out how attackers now target CI/CD pipelines because they hold so many secrets. A few were upset about how late the warning came, wondering why such a big attack was not caught sooner. Some users shared ideas for safer workflows, like using short-lived tokens and tighter GitHub Action settings. People also discussed the challenge of trusting npm packages, with some saying there’s no easy fix except to always be careful. A few commenters joked about the Dune references, but most agreed the attack was serious and showed how creative hackers have become. Some worried about what else might be compromised and called for better tools to catch these problems early.

The author says today’s internet is filled with social media, ads, algorithms, and lots of boring or fake content, often made by AI, which makes it less interesting and less human.

Back in 1999, most people used the internet for real conversations, personal websites, and direct access to information. There was no endless scrolling or recommendation feeds. The author suggests using tools and protocols from that time, like RSS feeds for news, IRC or XMPP for chat, and email for talking to people. He also mentions old and simple protocols like Gopher and Gemini, but says the real problem is not with HTTP, the main web protocol, but with how we use it. To improve your web experience, he suggests blocking ads and bad sites, using text-only browsers, and turning off JavaScript. Most of all, he wants people to focus on real human connections, not on automated or fake content.

In the comments, many readers agree that the old internet felt more personal and creative. Some people share their own methods, like using RSS readers, self-hosted chat, and bookmarking or archiving what they like. Others say they miss personal blogs and forums, and dislike today’s ad-driven, algorithmic sites. Some think it is hard to go back because friends and family only use modern platforms. A few believe tools like RSS are too hard for most people to set up. There are also warnings that even old protocols had problems, like spam on email and IRC. Some readers think the internet is always changing, and we cannot really go back, but we can try to make better choices for ourselves. Others suggest using both new and old tools, depending on their needs. Many agree with the author that blocking ads and being careful with what you read and watch is important. ## An update on recent Claude Code quality reports

- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=47878905)

This article talks about problems with Claude Code, a tool from Anthropic, where users noticed it was giving worse answers. The company found three separate changes caused the trouble, but they have now fixed all of them.

First, they changed the “reasoning effort” setting from “high” to “medium” to make things faster, but this made answers less smart. Users did not like this, so Anthropic switched back to the higher setting. Second, there was a bug in how old conversation history was cleared after a user returned to a session. This bug made Claude forget what it was doing and repeat itself, which confused people and used more of their quotas. Third, they tried to make Claude less wordy by telling it to keep answers short, but this accidentally made coding answers worse, so they removed this change too.

Each of these changes affected only some users, so the problems seemed random and hard to find. Their own tests did not catch the bugs, and it took a while to see what was wrong. Anthropic says they now have new rules: more staff will use the public version, code reviews will be better, and every change to how Claude talks will be tested more carefully. They are also letting users know about changes on Twitter and GitHub, and as a thank you for patience, all usage limits are reset.

From the Hacker News comments, some users praised Anthropic for the clear and honest report, saying it is rare for companies to admit mistakes in detail. Others said these issues show why AI products need more careful testing, especially when changes can confuse users or break workflows. A few people liked that Anthropic is listening to feedback and fixing things quickly, while others wondered if the problems show that AI models are too complex or fragile. Some developers shared their own stories about bugs caused by small changes in AI prompts or settings. Others asked why the bugs were not noticed sooner, and suggested more public testing or open source tools. A few said that usage limits and performance changes are a big deal for paying customers, and companies should be more careful. Overall, people seemed happy with the fixes but want Anthropic and other AI companies to be more transparent and careful in the future.

## Girl, 10, finds rare Mexican axolotl under Welsh bridge

A 10-year-old girl named Evie found a rare Mexican axolotl under a bridge in Wales while playing near the River Ogmore. This is the first recorded case of an axolotl found in the wild in the UK, and experts say there are only 50 to 1,000 left in the world.

Axolotls are a type of salamander that never leave the water as adults. They keep their gills and can even regrow body parts like legs and eyes. Axolotls have become popular pets, especially after appearing in games like Minecraft and Roblox, but are hard to look after. The axolotl Evie found was hurt, with damage to its tail and stomach. Her family took it home, named it Dippy, and asked experts about how to care for it. They are now setting up a bigger tank for Dippy and plan to keep it healthy.

Experts think someone may have released Dippy into the wild because they could not care for it. It is illegal and dangerous to release non-native animals like axolotls into nature. Axolotls are endangered in the wild because their habitat in Mexico is shrinking. However, they are common in captivity and even shown on Mexican money.

The article says if you find an endangered animal, you should tell the right animal welfare group or the government. Evie’s quick action likely saved Dippy’s life because axolotls cannot survive long in cold UK rivers. Her story has made her popular at school, and her family has no regrets about helping Dippy.

In the Hacker News comments, many people were surprised and happy about Evie’s discovery. Some said it shows how curious kids can notice rare things adults might miss. Others pointed out how video games like Minecraft have made axolotls famous, leading to more people keeping them as pets. But several commenters warned that axolotls need special care and should not be bought just because they look cute or are popular online. A few people talked about the problem of releasing pets into the wild, explaining it can hurt local wildlife and the pet itself. Some wondered if Dippy could be used to teach others about endangered animals. Others discussed the sad fact that axolotls are almost extinct in their home country, Mexico, but common in tanks around the world. ## UK Biobank health data keeps ending up on GitHub

- 原文链接: [UK Biobank health data keeps ending up on GitHub](https://biobank.rocher.lc)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=47875843)

The article talks about how sensitive health data from the UK Biobank keeps ending up on public GitHub repositories, even though strict rules say this data must not be shared. The UK Biobank holds detailed health and genetic information from about half a million people and only gives access to approved researchers.

Many researchers have accidentally uploaded this private data to GitHub while sharing code or research, sometimes even when they were not allowed to have the data. When this happens, UK Biobank tries to remove the data by sending takedown notices to GitHub, using copyright laws. These notices often target specific files, like Jupyter or R notebooks, genetic data files, or spreadsheets that might have sensitive information. Sometimes, only a small amount of data is shared, but even with just a little bit, a person can be identified. For example, with only a birth date and a surgery date, a volunteer was matched to their health record by journalists.

The article says there have been at least 110 takedown requests covering 197 repositories by 170 developers in at least 14 countries, with most developers based in the US and China. The process for finding and removing the data relies on searching GitHub’s public DMCA notices and matching them with UK Biobank cases. The article also explains that the UK does not have a strong legal process for fast data privacy takedowns, so UK Biobank uses copyright laws, which are usually for things like pirated software, not privacy.

Some months, there were no takedown notices, but the article suggests this probably means problems were missed, not that the issue stopped. After a news story highlighted the problem in March 2026, takedown activity restarted.

In the comments, many people are worried about how easy it is to accidentally leak sensitive data. Some say researchers need better training about data privacy and sharing. Others think the rules from UK Biobank might be too strict or confusing, making mistakes more likely. A few point out that using copyright laws for privacy problems is not a good solution and shows that data protection laws are weak. Some users say sharing even a small amount of data can be dangerous, as people can be identified from little information. Others argue that open science and sharing code are important, but it needs to be done safely. There are also worries that this problem is not unique to the UK Biobank and could happen with other big data sets. Some commenters feel that UK Biobank should do more to help researchers avoid mistakes, maybe by giving better tools or guidance. Finally, a few think that the main problem is how hard it is to control data once it leaves the original owner, and that mistakes are almost impossible to stop completely.

## Astronomers find the edge of the Milky Way

- 原文链接: [Astronomers find the edge of the Milky Way](https://skyandtelescope.org/astronomy-news/astronomers-find-the-edge-of-the-milky-way/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=47879239)

Astronomers have found where the Milky Way’s star-forming disk ends, at about 40,000 light-years from the center. They did this by studying over 100,000 giant stars and using computer simulations and data from space missions like Gaia.

The team saw that, in the inner part of the galaxy, stars are young because new stars are forming. As you move farther out, stars get younger until a certain point. After about 35,000 to 40,000 light-years from the center, this trend stops and stars become older again. This creates a “U” shape when you look at the ages of stars across the disk.

This means stars outside this edge are not new. They did not come from outside galaxies but moved out from the center over time. These older stars were pushed by waves in the galaxy, a bit like surfers riding waves to the shore. The edge found marks where star formation drops sharply and shows how our galaxy grows and changes.

The study used data from LAMOST, APOGEE, and Gaia, focusing on red giant stars because their ages are easier to measure. This new finding matches what scientists see in other galaxies, showing the Milky Way is not special but follows a common pattern.

We still do not know why star formation stops at this edge. It could be because the galaxy’s shape bends near the edge or the center’s gravity moves gas away. New telescopes, like 4MOST and WEAVE, may help answer these questions.

On Hacker News, some readers were excited to see a clear answer to an old question: where does our galaxy end? Others pointed out that “edge” is tricky, since the galaxy is not a solid object and has stars and gas even farther out. Some shared that this discovery helps us understand how galaxies change over time and how stars move inside them. A few users wondered how this new “edge” fits with the known halo of old stars around the Milky Way. Many liked how new data and better models are giving us sharper pictures of our galaxy, while some reminded that these findings will keep changing as we learn more. One reader joked that the real “edge” is wherever our imagination stops, showing the wonder this kind of news creates.

## My phone replaced a brass plug

- 原文链接: [My phone replaced a brass plug](https://drobinin.com/posts/my-phone-replaced-a-brass-plug/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=47877715)

This story is about how someone used their phone and machine learning to replace a simple brass tool used for scoring target shooting cards. The author, an iOS engineer, got into shooting because of their love for cooking and wanting to hunt for meat, but found the manual scoring process slow and annoying.

At the shooting range, people use brass plugs to judge if a bullet hit within a scoring ring. The author wanted to automate this with their phone. First, they tried using Apple’s Vision framework to detect bullet holes, but it did not work well because bullet holes are “negative space” (missing paper), not objects. They also tried other image tricks like inverting colors and edge detection, but accuracy was low.

Looking for a better way, the author found a 2012 paper that used computer vision to score targets. This worked better but still had problems, because the UK’s NSRA targets are different from the targets in the paper, and bullets sometimes tear the paper in tricky ways. The author then tried adding machine learning. They trained a YOLOv8 model to detect bullet holes, using both machine learning and geometry (like the target’s rings) to get better results.

The final system uses OpenCV for geometry and a trained YOLOv8 model (converted to CoreML) for detecting holes. The author had to tune the system to make sure the scoring matches real life, as the torn hole is smaller than the bullet. They also created an app to do all this offline, so shooters can use their phones instead of brass plugs.

In the comments, many people liked the mix of old and new tech. Some thought it was cool to see machine learning used for such a classic, simple task. A few pointed out that traditional tools like the brass plug are still needed for fairness and settling disputes. Others questioned if the phone method could ever fully replace the trusted brass plug in competitions, since the human side and the rules are important. Some readers loved the “hacker” spirit of the story—solving a real problem for fun. A few shared stories of their own about using tech to replace simple tools, but some warned about over-complicating things that work perfectly well. There was also talk about how image recognition can be tricky with real-world objects and messy data, and some praised the author’s honest notes about tuning and edge cases. Finally, several people said they’d like to try the app, while others joked that sometimes a simple brass plug is just the best tool after all.
