## πFS

- 原文链接: [πFS](https://github.com/philipl/pifs)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48480978)

This project introduces πfs, a funny and clever idea for a filesystem that does not store your data on your hard drive. Instead, it claims to store your data inside the digits of pi (π). The idea is based on the mathematical theory that every possible sequence of numbers can be found somewhere in the infinite digits of pi—so, in theory, every file that ever existed is already present in pi, if you can find where.

The article explains how to install and use πfs. You need some common Linux tools (like autoconf and libfuse), then you build and mount the filesystem. When you save a file with πfs, it does not write the real data to disk. Instead, it searches for each byte of your file in the digits of pi and remembers where it found it. The “metadata” (information about where in pi your data is stored) is kept in a separate place. This metadata is very important, because you need it to get your files back.

Pi is a special number in mathematics, and some people think its digits are “normal.” This means every possible group of numbers might appear somewhere in its endless sequence. If this is true, then in theory, every possible file exists inside pi. The filesystem uses a formula called the Bailey–Borwein–Plouffe (BBP) formula to find digits in pi. But, searching for long sequences is very slow and not practical, so πfs only looks for one byte at a time.

The article jokes that you could even store the metadata (the index locations) in pi too. It also mentions some future ideas, like making the search faster, using cloud services, or running πfs on big data systems like Hadoop.

In the comments, many people see this as a fun parody, not a serious product. Some laugh about how pi could be used to “store” anything, even things you never created. Others point out that, in practice, it’s impossible to really find long files in pi, because pi’s digits are infinite and you would need huge computing power to search. Several users discuss that the “normality” of pi—the idea that every possible file is in there—has not been proven, so the whole thing is just a funny thought experiment.

A few people talk about copyright and security. They joke, if every file exists in pi, then so do all copyrighted works and secrets, but that doesn’t mean you can actually find or use them. Some say this is like a joke about 100% data compression, which is impossible in reality. Others bring up the limits of such ideas, like the huge size of the metadata compared to the “space saved.” Some also mention similar jokes or theoretical ideas from the past.

Overall, most comments enjoy the humor and creativity. People appreciate the way πfs makes you think about data, storage, and the weird side of math and computing. Some even say it’s a clever way to learn about filesystems and math, while others remind everyone not to take it too seriously.

## How JPL keeps the 13-year-old Curiosity rover doing science

- 原文链接: [How JPL keeps the 13-year-old Curiosity rover doing science](https://spectrum.ieee.org/curiosity-rover-jpl-mars-science)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48479705)

This article talks about how NASA’s Jet Propulsion Laboratory (JPL) keeps the Curiosity rover doing science on Mars after 13 years. Curiosity still works well, even though Mars is a very hard place for any robot, and JPL can only send software updates to fix problems.

Over the years, Curiosity has driven almost 37 kilometers, drilled 42 rocks, and taken over 763,000 photos. The rover is still busy doing science, which is impressive because it faces cold, dust, and rough ground. JPL engineers have had to be very creative. For example, Curiosity has two computers, called A and B. When one had memory problems, they switched to the other, copied data slowly between them, and even used tiny parts of memory meant for other tasks to keep the rover working. One big problem is the wheels, which got damaged by sharp rocks. To help, engineers started driving Curiosity backwards to protect the wheels. The rover’s power comes from a nuclear battery, which loses power as it gets older, so the team has made new ways for Curiosity to save energy, like turning off computers and heaters when not needed. They also make the rover do more tasks at the same time, so less energy is wasted.

Curiosity’s team watches every part that can wear out, including the robotic arm. Some tasks, like selfies, are done less often now to save the joints. The rover’s computer is old and uses more power than new ones, but it is still good enough for the mission because of these tricks. Lessons learned from Curiosity help improve newer rovers like Perseverance, which can drive by itself and uses some better software.

For the future, the biggest limit is power and maybe the budget. The team thinks Curiosity can keep working at least until 2035. Even if the arm breaks, the cameras and sensors can still do science.

In the comment section, some people are amazed at how long Curiosity has lasted and how clever the JPL team is. They compare the rover’s old computer to modern phones and are surprised by how much can be done with so little. Some commenters note that most consumer devices do not last nearly as long as Curiosity has, and wonder why our everyday technology is not built to last. Others praise the value of careful engineering and planning for problems, saying this is why space missions succeed. A few discuss the software tricks, thinking it is cool that memory meant for flight programs can become a file system in an emergency. Some people say this story is inspiring for all engineers, and that it shows how important it is to keep learning from every mission. Finally, a few worry about the costs and hope that future missions get enough funding to keep doing good science.

## What Is It Like to Be a Bat? [pdf] (1974)

- 原文链接: [What Is It Like to Be a Bat? [pdf] (1974)](https://www.sas.upenn.edu/~cavitch/pdf-library/Nagel_Bat.pdf)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48482293)

This article asks a simple question: what is it like to be a bat? The author, Thomas Nagel, uses this question to talk about the problem of consciousness and how hard it is to explain.

Nagel says that the mind-body problem becomes really hard when we try to explain consciousness. Most scientists try to explain the mind by comparing it to things we understand, like machines or chemical processes, but Nagel thinks this does not work. Consciousness is special because it has a “subjective character”—there is something it is like to be you, or to be a bat. This “what it is like” cannot be explained just by looking at brain cells or behavior.

He gives the example of bats because they use sonar to “see” the world, which is very different from how humans sense things. Nagel says we cannot really imagine what it is like to be a bat. We can guess some things (like bats fly or use sound), but we can never know how it feels to use sonar, because our minds are not built that way.

Nagel argues that this problem is not just about bats. Even between people, we cannot fully know what another person’s experience is like, especially if their senses are very different. He says there might be facts and experiences in the world that humans can never know because of our limits.

He also says that trying to explain consciousness only with physical science or objective facts misses the point. Consciousness is tied to a point of view. If you remove that point of view, you lose what makes experience special. He thinks maybe in the future we can find new ways to talk about subjective experience more objectively, but right now, science does not have the tools.

In the Hacker News comments, many people found the article deep and thought-provoking. Some agree with Nagel that science has trouble explaining consciousness. They say we might never really solve the mind-body problem or know what it’s like to be another creature. Others mention that Nagel’s arguments are important for debates about artificial intelligence and whether machines can be conscious.

A few commenters point out that humans have some abilities to imagine other experiences, like blind people using sound to “see” or people using virtual reality. They wonder if technology might help us understand other points of view better.

Some think Nagel is too negative. They believe that, with enough science and technology, we might be able to explain consciousness someday. Others say this article shows why philosophy is still important, because science alone cannot answer every question.

A few users share personal stories about trying to imagine being an animal, or about sensory changes in their own lives, and how hard it is to describe those feelings to others. Some mention related ideas, like “the hard problem of consciousness” or “qualia,” and suggest further reading.

Overall, people appreciate the article for making them think about what it means to be conscious and how little we really know about the minds of others—whether animals, people, or future machines.

## PgDog is funded and coming to a database near you

- 原文链接: [PgDog is funded and coming to a database near you](https://pgdog.dev/blog/our-funding-announcement)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48476466)

PgDog is a new open-source tool that helps Postgres databases scale much bigger and faster. The team just raised $5.5 million to keep building it and says it can handle over 2 million queries per second across many deployments.

The main idea is simple: Postgres is a great database, but it struggles when you need to deal with huge amounts of data or very high traffic. Other databases, like MongoDB or DynamoDB, exist mostly because Postgres did not scale easily before. PgDog tries to fix this by putting a smart proxy in front of Postgres. This proxy lets you "shard" your data, which means splitting it up across many servers. Now, you can use Postgres for much bigger workloads without switching to something else.

You can run PgDog anywhere—on your own servers or in any cloud. All you have to do is change your database connection and use their Docker image. PgDog is already being used in production, with over 1.4 million Docker pulls and weekly updates. The team has experience scaling Postgres at large companies and is active in their Discord, helping users with problems.

PgDog is run by a very small team, but they say their experience comes from running large Postgres systems at places like Instacart. They want people to trust them because they know how to make Postgres work at massive scale and are focused only on this problem. There’s also an Enterprise edition coming, made to work better in AWS and with extra support.

In the comments, many people are excited about PgDog. Some say it’s great to see open-source tools making Postgres scale so well, and they like the simple setup. Others are curious about how hard it is to migrate existing databases to PgDog, and some warn that sharding can make things more complex—especially with joins and transactions.

A few users share worries about trusting a small, new team with important data, but others point out that the open-source nature lets anyone check the code. Some people ask about benchmarks and want to see more real-world examples, while others suggest PgDog could help smaller companies avoid expensive managed databases. There are also technical questions about how PgDog handles failover, backups, and version upgrades. Overall, commenters think PgDog could make scaling Postgres much easier, but they want to see how it works in more situations and over time.

## L'Affaire Siloxane

- 原文链接: [L'Affaire Siloxane](https://mceglowski.substack.com/p/laffaire-siloxane)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48456808)

This article tells the story of how a chemical called siloxane started causing big problems in the International Space Station’s (ISS) water system. It explains how astronauts began recycling their water, but soon found mystery contamination that was hard to identify. 

The ISS needed to reuse water to save money and avoid sending heavy water shipments from Earth. In 2008, a new water recycling system was installed, which could turn astronaut urine back into drinking water. About a year later, sensors showed that the water had rising organic carbon levels—a sign of some unknown contaminant. With no way to test the water in space, samples had to be sent back to Earth for analysis. At first, even experts could not figure out what the contaminant was.

Eventually, Boeing engineers discovered the culprit: dimethylsilanediol (DMSD), part of the siloxane family. Siloxanes are chemicals found in products like deodorant, hair conditioner, and lotions; they make things feel smooth. These everyday items, brought by astronauts for personal use, slowly released siloxanes into the ISS air. Space radiation helped break down siloxanes into DMSD, which then got into the water system.

DMSD is hard to remove because it does not react easily with filters. It built up quietly in the water system’s filters, then suddenly “spiked” into the drinking water when the filters became saturated. This caused repeated water contamination problems. NASA tried to solve the problem by changing filters and even developed special air filters to catch siloxanes before they entered the water. However, these new filters led to other issues, like mold growth, so NASA had to keep tweaking the system.

The article uses this siloxane story to show how small, boring problems can turn into big headaches in space. It points out that space missions are full of “unknown unknowns”—risks that no one thinks about in advance. It’s very hard to keep every possible contaminant off a spacecraft because siloxanes are everywhere on Earth. Life support systems are complicated, and fixing one problem often causes another. Also, problems in space are hard to test for on Earth, because the space environment is unique.

In the comments, many readers found the siloxane problem fascinating because it’s so mundane but so hard to solve. Some were surprised by how common siloxanes are in everyday products and how difficult it is to keep them out of a closed environment like the ISS. A few pointed out that this is a classic example of how unexpected problems can pop up in engineering, especially in unique environments.

Others discussed the difficulty of doing chemical analysis in space. They noted how the lack of lab equipment on the ISS made it hard to identify what was causing the contamination. Some commenters suggested that future missions, especially to Mars, need better ways to test and fix problems on the spot, rather than waiting for help from Earth.

A few people joked about the “devious” nature of siloxanes, saying it’s funny how something as simple as deodorant can cause such major trouble for astronauts. Some worried that these kinds of small, “boring” issues could become life-threatening on longer missions far from Earth.

There were also comments about the cost and effort needed to keep the ISS safe. Some thought NASA’s approach of managing rather than solving the problem was sensible, while others hoped for new technology to finally beat siloxanes. Overall, most agreed that stories like this show how space engineering isn’t just about rockets and computers—it’s also about fixing very human, everyday problems in a place where mistakes can be very expensive.

## GeoLibre 1.0

- 原文链接: [GeoLibre 1.0](https://geolibre.app/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48479852)

GeoLibre 1.0 is a new, cloud-based tool for working with maps and geospatial data. It helps people view, explore, and analyze map data both on the web and desktop.

The app lets you move around a map, zoom in or out, and look at the map from different angles. You can use different tools like measure, bookmark, or minimap on the map. GeoLibre works with many kinds of map data, letting you load files from your computer or from the internet. You can see the data in tables, change how layers look, and arrange or refresh them as you want.

Plugins add more features, like street view, time sliders, LiDAR support, and you can install even more from a built-in marketplace. It supports many types of data services and file formats, like XYZ, WMS, WMTS, GeoParquet, COG, and 3D tiles, and you can connect to databases like DuckDB or PostgreSQL. You can also convert between formats and run batch processes using a tool called Whitebox or with Python helpers.

GeoLibre lets you write and run SQL queries on your data directly in the browser. After running a query, you can see the result on the map or export it. There are also built-in tools for common map tasks like drawing buffers, finding centroids, combining shapes, or clipping data. For working with images (raster data), you can do things like create hillshades, reproject, or make contours, using Python tools.

The app can be used inside Jupyter notebooks, so you can code and view maps together. There are guides and tutorials for new users, showing how to add data, use tools, or build your first map. GeoLibre runs fully in your browser for privacy, with no server involved unless you choose to share data. You can also embed it in other websites, and there are options to hide panels or toolbars if you want a simpler look.

The first release is called a stable prototype. It has all the main features: map workspace, project saving and sharing, plugin support, many data types, and tools for both vector and raster data. There are installers for different platforms and Docker support. The team shares their roadmap for future updates.

In the Hacker News comments, many people are impressed by the wide range of features for a 1.0 release. Some users like that it works both online and offline, and that it is privacy-friendly. Others are happy with the support for modern formats and cloud data. A few users compare GeoLibre to bigger GIS tools like QGIS, noting this one feels lighter and easier to use.

Some commenters ask about performance with large datasets and if the app can handle heavy processing. Others wonder how plugin support works and if the community can add more features. A few users mention they want to see more real-world examples or bigger tutorials to help new users. Some are excited about Jupyter integration, while others point out possible limits due to running in the browser. Overall, the community seems positive, but they hope the project keeps growing and stays open to user feedback.

HelixDB is a new database made for building AI tools, combining different types of databases—like graph, vector, document, and relational—into one system. The main idea is to make it easy for developers to manage and query all their data without needing many separate tools.

HelixDB uses a graph and vector data model, but it also supports key-value, document, and relational data. You can install it easily with a simple command. There is a tool called `helix chef` that helps you start a new project quickly, even letting you describe what you want in one sentence. For more control, you can set up a local instance by hand, run it in memory or on disk, and send queries through a built-in command.

For coding, you can write queries using Rust or TypeScript. Both languages use a special syntax to build queries, and you send these to HelixDB with a POST request. There are code examples for adding and getting a user, showing how simple it can be to work with the database in your code. You can run HelixDB locally, but there’s also a cloud version. The cloud version uses object storage, supports full-text and vector search, ACID transactions, and can scale with high availability. You can manage cloud clusters using the command-line tool, switching between projects and syncing your setup.

HelixDB offers commercial support, with a managed cloud service for teams that need it. Documentation, a querying guide, and community support are available through Discord and Twitter.

Mercedes-Benz has started making a new kind of electric motor, called an axial flux motor, at its Berlin factory. This motor will first appear in the high-performance Mercedes-AMG GT 4-door coupe.

The axial flux motor is different from normal electric motors because it is smaller, lighter, and can make more power for its size. It uses two flat rotors and a stator in the middle, which makes the motor very compact. Mercedes says this is good for making fast electric cars with lots of power but not much weight. To make the motor, Mercedes had to invent new ways to bend copper wires and weld parts using lasers. They even use AI (artificial intelligence) to check if everything is made correctly. The final assembly is tricky because powerful magnets pull with a force as strong as 900 kilograms, and parts must be placed very carefully.

The Berlin factory now uses advanced machines and smart controls to build many of these motors quickly. The motors are made on seven production lines in three big halls. This new technology came from a British company named YASA, which Mercedes bought in 2021. The motors can spin very fast and help the AMG GT coupe go from 0 to 100 km/h in just over 2 seconds. The car has three of these motors: one in the front and two in the back.

Mercedes thinks this new motor will help electric cars become more popular and better for people who want both speed and efficiency. The Berlin factory is also important for testing new digital tools for car production.
