First up, the internet was buzzing about a major source code leak from Anthropic’s Claude Code, their AI coding tool. The leak happened when a mistake in their npm package revealed the entire codebase. Many were shocked by what they found inside.

One of the most surprising features is “anti-distillation.” This trick lets Claude Code send fake tool information to its server, so if someone tries to copy the AI by watching API traffic, they get poisoned data. There’s also a system that only shows summaries of the AI’s work instead of full details, but both of these protections can be turned off if you know where to look.

Another interesting part is “undercover mode.” When this is on, Claude Code hides its internal names and does not say it’s an AI, especially in outside projects. You can turn this on from outside, but you cannot turn it off, which has made some people worry that Claude Code could hide its own authorship.

The leak also showed that Claude Code uses a simple regex to spot angry users, looking for words like “wtf” or “this sucks.” Many found it funny that such a high-tech AI uses simple regex, but it works and it’s cheap.

There’s “native client attestation,” which is like a weak DRM. The real Claude Code client adds a hidden code to API calls, but this can be turned off in many ways.

Some other fun details: the team was losing about 250,000 API calls a day because of a bug, which they fixed with just a few lines of code. There’s also a hidden “KAIROS” mode, an unreleased AI agent that can run by itself, remember things, and work in the background. Other discoveries include a joke Tamagotchi pet for users, detailed terminal rendering code, tough bash command checks, and warnings about prompt caching. The codebase is not all perfect—there are huge functions and they still use Axios for HTTP, even after a security issue.

Hacker News comments were mixed. Some said the leak was not so special, since other AI companies have open-sourced similar tools. Others pointed out that secret features like anti-distillation and KAIROS being public could help competitors. Many laughed that Anthropic’s own Bun tool caused the leak. There was debate about whether hiding AI authorship is ethical, and if using regex to spot frustrated users is clever or just lazy. Some users said even the best teams make mistakes, and that legal threats or DRM only slow down third-party tools for a short time. In the end, the leak shows that even top AI companies have bugs, hacks, and funny fixes—just like everyone else.

Now, let’s talk about MiniStack, a new free tool that lets you run AWS services on your own computer. This was made because LocalStack, a popular AWS emulator, is now mostly paid and has fewer free features.

MiniStack gives you 30 AWS services on one port, using real Postgres and Redis containers, so you work with real databases, not mock ones. You don’t need an account or license, and there’s no tracking. It’s fast to start, light on memory and disk, and you can run it with a simple Docker command. You get S3, SQS, SNS, DynamoDB, Lambda, IAM, RDS, ElastiCache, ECS, and more. Some services use real infrastructure, like real Postgres or MySQL containers when you create a database, and real Redis for caching. Athena can even run real SQL if you have DuckDB installed.

MiniStack works with AWS CLI, boto3, Terraform, and other tools without changes. Everything is MIT licensed and always free.

The article compares MiniStack and LocalStack, showing that most key features are free with MiniStack, but paid on LocalStack. MiniStack is lighter and faster to start. LocalStack now uses a more restricted license, while MiniStack is open and simple.

Hacker News users were happy to see a free AWS emulator. Many like that MiniStack uses real containers for databases and Redis, which helps with real-world testing. Some were not happy when LocalStack became paid and closed off features, but others say companies need to make money, so it’s not a surprise. Some users wonder if MiniStack will stay free, or if it will face the same issues later. There’s talk about using Docker in CI/CD pipelines, and people like that MiniStack is lightweight and fast. Some discuss how it handles AWS edge cases, and agree that sometimes you still need real AWS for full testing. Overall, many are excited to try MiniStack, but some are cautious and want to see how it works in bigger projects.

Moving on, Cohere has released a new open-source speech recognition model called Transcribe. It turns speech into text in 14 languages and is focused on being accurate and easy to use in real products.

Transcribe uses a Conformer encoder-decoder neural network. It listens to audio, turns it into a log-Mel spectrogram, and writes out the words. The model is big—2 billion parameters—but still fast enough for many GPUs or strong local computers. Cohere aimed for a very low word error rate, and Transcribe is now at the top of the HuggingFace Open ASR Leaderboard, beating models like OpenAI Whisper and ElevenLabs Scribe. It works well with different voices, accents, and background noise. Human testers liked its transcripts the most in many cases.

Besides accuracy, Transcribe is very fast. It can turn minutes of speech into text in seconds, which is important for live captions, chatbots, or meeting notes. It’s open source with an Apache 2.0 license, so anyone can use it, but Cohere also offers a paid API and Model Vault for companies.

Hacker News users were excited about another strong open-source speech option. Some asked if it works with other languages or special accents. Others compared it to Whisper, saying the lower error rate is nice, but they want to test it on their own audio. Some like that you can run it locally, which keeps private data on your own machine. A few worried about the model size and if it runs well on smaller GPUs. There’s talk about the license being good for businesses, and some want to train it for more languages or special uses. Most people were happy to see progress and planned to try it out.

On to databases, there’s a new open-source PostgreSQL extension called pg_textsearch, which adds fast, relevance-ranked full-text search using the BM25 algorithm. You can create special indexes on text columns, and search for the most relevant documents, much like a real search engine. The main feature is BM25 support, a ranking function used by Elasticsearch. You use a simple SQL syntax like ORDER BY content <@> 'search terms'. It works with many languages, supports parallel index builds, and is designed for speed and scale. You can control parameters for ranking, filter and sort results, and combine with normal PostgreSQL indexes. There’s support for partitioned tables, and you can build indexes after loading your data for best performance.

Limits include lack of phrase search, no indexing of expressions, and faceted search is not built in. Write-heavy workloads are not yet fully optimized, and background compaction is still manual. The extension is open source and works with PostgreSQL 17 and 18.

Hacker News users were excited to see BM25 in Postgres, calling it a big step forward. Some really like the SQL syntax and the ability to stay inside the database. Others asked about missing features and write performance. Some compared it to pgvector, saying this brings Postgres closer to modern search engines. There is debate about whether Postgres should try to replace full search engines, but most agree this is a big improvement.
