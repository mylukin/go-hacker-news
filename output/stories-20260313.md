## Can I run AI locally?

- 原文链接: [Can I run AI locally?](https://www.canirun.ai/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=47363754)

This article is about CanIRun.ai, a website that helps you check if your computer can run different AI models locally. It lists many open-source AI models and shows how well they will work on your machine.

The site gives each model a score from S (best) to F (worst), based on how much memory and power it needs. Small models like Qwen 3.5 0.8B, Llama 3.2 1B, Gemma 3 1B, and TinyLlama 1.1B get high scores and run well on most devices. They use less than 1 GB of memory and can answer questions or code fast, making them great for phones, laptops, and Raspberry Pi.

Medium models, like Mistral 7B and Qwen 2.5 7B, use about 3–4 GB of memory. These can still run on many laptops, but they are slower and may lag. Large models, like Llama 3.1 8B or Qwen 3.5 9B, need 4–5 GB and are marked as “barely runs.” Very big models—12B, 14B, 24B, and up to hundreds of billions of parameters—need 10, 20, or even 200+ GB of memory. These only run on high-end desktops or servers, and many are marked “too heavy” with a score of F.

The website uses public data from llama.cpp, Ollama, and LM Studio, and shows the memory use, speed, and best tasks for each model (like chat, code, or vision). It also explains which architecture (dense or MoE) each model uses and if it supports extra features like long context or multi-language.

In the top Hacker News comments, some people say this tool is very helpful for picking the right AI model for their computer. Others like that it makes trying AI at home much easier. Some users warn that the numbers are just estimates, and real performance might differ depending on your setup (CPU, GPU, RAM). A few people share tips to run large models with tricks like quantization or using swap memory, but they say it is still slow.

Some commenters ask for more details, like benchmarks for different GPUs, or help with installation steps. A few say the site is great for beginners, but advanced users might want more technical data. There are also comments about privacy—many are happy that running AI locally means less data is sent to the cloud.

Others mention that open models are getting better, so soon we may run even bigger models on normal hardware. Some are excited about using tiny models on phones or edge devices. In general, the community thinks CanIRun.ai is a useful project that helps bring AI to everyone, not just big companies.

## Mouser: An open source alternative to Logi-Plus mouse software

- 原文链接: [Mouser: An open source alternative to Logi-Plus mouse software](https://github.com/TomBadash/MouseControl)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=47368033)

Mouser is a free, open-source app that lets you change the button functions on the Logitech MX Master 3S mouse, just like Logitech’s Options+ software, but without accounts, cloud, or tracking. It works on Windows and macOS, is easy to use, and you don’t need to install anything—just download and run it.

The main features are: you can remap all six programmable buttons, set different profiles for different apps, and pick from 22 built-in actions (like browser back, copy, paste, or media controls). There’s also a slider to set the mouse speed (DPI), and you can flip the scroll direction if you want. It shows if your mouse is connected, runs quietly in the system tray, and saves your settings in a simple file on your computer. The app has a modern, simple interface with a picture of the mouse—just click the button you want to change. If your mouse disconnects or turns off, Mouser will automatically reconnect and keep working. For technical users, it’s written in Python and uses Qt for the interface. You can also run it from the source code, and there are guides for both Windows and Mac.

The project is only for the MX Master 3S right now, but it may support more Logitech mice later. It does not work on Linux yet. Also, you must close Logitech’s own software before using Mouser, or the mouse might not work right.

In the comments, many people are happy to see an open alternative to Logitech’s software, which is often disliked for being slow, using too much memory, and requiring an account. Some users say they need this for other Logitech mice, not just the 3S model. There are requests for Linux support, since many open-source fans use Linux. Others ask about supporting extra features, like mouse gestures or more key combinations. A few people are curious about how well it works compared to Logitech’s own app, and if it can handle things like sleep/wake, Bluetooth, and all the special buttons. Some commenters warn that Logitech may change their hardware or software to block third-party tools like this in the future. Others share tips for running Mouser alongside other software, or ask how safe and private it is. One user thanks the developer for not including telemetry and for keeping everything local. There’s also some technical talk about how the app hooks into the mouse signals, and some users offer help to add new features or Linux support. Overall, most people in the discussion are excited and hopeful this project will grow to support more devices and systems.

## Hammerspoon

- 原文链接: [Hammerspoon](https://github.com/Hammerspoon/hammerspoon)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=47367932)

Hammerspoon is a tool for automating tasks on macOS using the Lua programming language. It acts as a bridge between your computer’s system and Lua scripts, letting you control many parts of your Mac with code.

The project gives users many “extensions” that unlock system features, like moving windows, changing settings, or reacting to events. To use Hammerspoon, you first install it, then create a file called `init.lua` in your home folder. You put your Lua code there to make things happen. You can install Hammerspoon manually by downloading it, or with Homebrew using a simple command. Out of the box, Hammerspoon does nothing until you write some code for it. The project’s website and GitHub have guides, sample scripts, API docs, and a FAQ to help new users. Hammerspoon is open source, under the MIT license, and has active contributors. It started as a fork of another tool called Mjolnir, but aims to offer a more complete and easy-to-use experience. The project wants to cover more system features, make extensions work better together, and keep improving how easy it is to use.

In the comment section, many users say Hammerspoon is very powerful and flexible. Some share how they use it to move windows, control music, or automate daily tasks. People like that they can write their own code to do almost anything on macOS. A few users say the learning curve can be steep, especially if you don’t know Lua. Others mention the documentation is helpful, but it can be hard to find good examples for complex tasks. Some compare Hammerspoon with other automation tools like Keyboard Maestro or BetterTouchTool; they say Hammerspoon gives more freedom, but those other tools are easier for beginners. A few worry that strong automation tools might break when Apple updates macOS, but many say Hammerspoon is well maintained. Some people wish there were more pre-made scripts they could use. Others are surprised more developers don’t use Hammerspoon, given how much it can do. Overall, users agree it is a great tool if you like to tinker and want deep control over your Mac.

Qatar has shut down its helium production, and this is causing big problems for the global chip supply chain. The article explains that helium is very important for making computer chips, and most of the world’s helium comes from Qatar.

The shutdown happened because of political problems in the Middle East. Helium is used to keep chip-making machines clean and cool. Without helium, these machines cannot work well. The world only has a small amount of helium stored, so if Qatar is closed for more than two weeks, chip factories could stop working. This could cause delays for computers, phones, and cars. The article says that some companies have a little extra helium, but not enough for a long time. The last time something like this happened, chip prices went up, and some factories had to slow down. The article also says that making new helium plants takes years and costs a lot of money. Experts warn that this shutdown could hurt many businesses, not just chip makers, but also hospitals and research labs that need helium.

In the comments, some people are surprised that so much helium comes from just one place. Others ask why chip companies do not keep more helium in storage. A few say this shows how fragile the chip supply chain is. Some readers point out that helium is used in other important things, like MRI machines in hospitals. Others think companies should try to recycle helium or find new ways to make chips without it. A few commenters remember that there have been helium shortages before, and nothing much changed. Some people worry that the price of electronics will go up again. Others suggest that governments need to help make the supply chain stronger. Finally, a few readers say this is a reminder that small things can have big effects on technology.

## Stanford researchers report first recording of a blue whale's heart rate (2019)

- 原文链接: [Stanford researchers report first recording of a blue whale's heart rate (2019)](https://news.stanford.edu/stories/2019/11/first-ever-recording-blue-whales-heart-rate)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=47368430)

Stanford researchers recorded a blue whale’s heart rate in the wild for the first time. They used a special sensor tag with suction cups that stuck near the whale’s left flipper to collect the data.

The researchers were not sure the tag would work, but it did. The tag stayed on while the whale swam, and it picked up the heart’s signals. When the whale dove deep, its heart slowed down to as low as two beats per minute. During foraging dives, the heart rate went up a bit, but the highest rates—25 to 37 beats per minute—happened at the surface when the whale was breathing.

This finding was surprising because the heart rate at the surface was higher than expected, and the rate during dives was lower than scientists thought. The team thinks the whale’s heart has special features, like a stretchy aortic arch, to help with these extremes. The heart needs to work very hard to support the whale’s huge body, and this may be why blue whales can’t get any bigger.

The team wants to improve the sensor and test it on other big whales. They hope this work will help us learn more about animal biology and help protect endangered whales.

In the comments, people were amazed at how slow a blue whale’s heart beats. Some compared it to the heart rates of other animals, like hummingbirds, which beat very fast. Others wondered how the researchers got the tag on such a big, wild animal. A few commenters talked about the risk to the whale, but most felt the study was safe and important.

Several users discussed how the heart’s limits might set the maximum possible size for animals on Earth. Some thought this research could help us understand more about evolution and animal physiology. There were jokes about how big the whale’s heart must be, with people comparing it to car engines. Others praised the technology and teamwork needed to do this study.

A few commenters asked if these findings could help humans, or if whale hearts could teach us about medicine. Some users wanted to know if this technology could be used to track health in other large wild animals. Overall, the discussion was excited and curious about what we can learn from the largest animal in the world.

## New 'negative light' technology hides data transfers in plain sight

- 原文链接: [New 'negative light' technology hides data transfers in plain sight](https://www.unsw.edu.au/newsroom/news/2026/03/New-negative-light-technology-hides-data-transfers-in-plain-sight)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=47341432)

This article is about a new technology called 'negative light' or 'negative luminescence' that lets people send secret data by hiding it in heat radiation. The technology was made by researchers in Australia and uses special LEDs and diodes to blend data signals into normal heat, so only special receivers can see the data.

Normal data transfers can be seen, even if they are encrypted, but with this negative light method, the data looks invisible to outsiders. This makes it very hard for hackers or spies to know if any information is being sent at all. The system uses the mid-infrared part of the light spectrum, which is the same range as the heat we feel but cannot see. The special diodes, called thermoradiative diodes, can create a pattern that is either brighter or even "darker" than the usual background heat, making it possible to send a secret code.

In lab tests, the researchers managed to send about 100 kilobytes of data per second. They believe that, with better materials, the speed could go up to gigabytes per second. They are testing new materials, like graphene, which could make the technology much faster and safer to use. This new way of sending data could be useful for the military, banks, or anyone who needs to keep communications secret. Even if someone used a thermal camera, the data would not stand out from normal heat, so it would be almost impossible to detect.

Experts say that this kind of hidden communication could add another layer of security, especially when combined with regular encryption. The team hopes to make this technology available for real-world use soon and is working on making it even better.

In the comments, some people are excited about the idea, saying it could change how we think about secure communication. Others wonder if it will be practical outside the lab or if it might be easy to block or jam. A few users ask if this could be used for spying, and if there are ways to detect it with advanced tools. Some worry about the materials used, like mercury, which can be dangerous, but are glad to hear the team is looking for safer options. There are also jokes about how this is like having an "invisible flashlight" or "sending Morse code in the dark." Many agree it’s a clever idea, but some want to see more real-world tests to prove it really works as promised.

The tool supports over 20 TUI components, such as buttons, tables, lists, text inputs, and more. You can use different layout systems, like Flexbox and Grid, just like in web development. There are 8 color themes to choose from, such as Solarized and Monokai, and you can quickly switch between them to see how your UI looks. TUI Studio shows a live preview in ANSI so you see exactly what the UI will look like in a real terminal. You can zoom in and out on the canvas, save your designs as .tui JSON files, and open them later or share them with others.

One big feature is exporting your design to code for different frameworks, like Ink (TypeScript), BubbleTea (Go), Blessed (JavaScript), Textual (Python), OpenTUI (TypeScript), and Tview (Go). However, this export feature is not ready yet, as the app is still in alpha. The app works on Mac, Windows, and Linux, but you may get warnings because the app is not code-signed. The editor is free to use for now, but there may be a paid version with extra features later. You do not need an account or an internet connection to use it.

From the comments, many people are excited about the idea of a visual editor for TUIs. Some users say this will make it easier to create terminal apps without writing lots of code by hand. ## Show HN: Context Gateway – Compress agent context before it hits the LLM

- 原文链接: [Show HN: Context Gateway – Compress agent context before it hits the LLM](https://github.com/Compresr-ai/Context-Gateway)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=47367526)

Context Gateway is a tool that compresses the chat history of AI agents before sending it to a large language model (LLM), so the conversation does not slow down or hit size limits. It works in the background and helps AI tools like Claude Code and Cursor keep their context short and clear.

The main idea is that when you talk to an AI agent for a long time, the chat history gets too big for the LLM to handle. Normally, this means you have to wait for the agent to cut down or summarize the history, which can be slow. Context Gateway solves this by always summarizing the conversation in the background. When the history gets close to the LLM’s limit, the summary is ready, and the agent can keep working without delay. The tool works as a “proxy”—it sits between your agent and the LLM API, handling the compression for you. It supports several agents out of the box, or you can add your own. You install it with a simple script, and there’s a setup wizard to help you choose your agent and settings. Features include a dashboard to monitor savings, prompt history for debugging, and tools for enterprise use. It also supports Docker for easy deployment, and the code is open source under Apache 2.0.

In the comments, many people think this is a useful idea, especially for those running their own AI agents or who want to save on LLM costs. Some users ask about how well the summaries work, and if important information gets lost when compressing the context. Others say they have built similar tools at work and agree that automatic, background compaction is important for a smooth user experience. A few people wonder how this tool compares to just using bigger LLM context windows, and if it will still be needed when models can handle more tokens. There are questions about which summarizer model is best, and if you can plug in your own. Some commenters worry about privacy—does the gateway store chat logs, or leak data? Others like that it is open source, so they can check or improve the code themselves. A few see possible problems if the summary is wrong, since the agent might lose track of the conversation. Overall, most agree this tool helps make AI agents faster and more reliable, but there are tradeoffs between speed, accuracy, and how much context you keep.
