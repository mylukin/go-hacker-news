## FreeInk: Open ecosystem for e-readers

- 原文链接: [FreeInk: Open ecosystem for e-readers](https://freeink.org/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48996318)

FreeInk is a project making open-source software, firmware, and hardware for e-readers. Everything is public, so anyone can use, change, or build their own e-reader.

The main software is CrossPoint Reader. It can read EPUB 2 and 3 books, lets you change fonts and text settings, and works with many languages. You can upload books over WiFi, sync reading progress, and easily add bookmarks. There is a “focus reading” mode to help you read faster, and menus are translated into many languages. 

The FreeInk SDK is a toolkit for making firmware that works on many different e-reader devices. Instead of rewriting code for every device, developers add a simple profile and configuration. The API is clean and stable, so the same code works for many devices. Features like touch screens or frontlights are easy to switch on or off, making each device’s software small and efficient. It even includes secure networking for syncing across devices.

There are already several supported e-readers, like Xteink and M5Stack, using different screens and chips. New or related devices can use the same firmware without major changes. 

The hardware side is called de-link. It’s a small board you can build for about $60, using easy-to-find parts and open schematics. It has WiFi, Bluetooth, USB-C charging, microSD storage, e-paper display support, and extra pins for adding your own features. The battery is swappable, and you can repair it yourself.

The whole idea is to make e-readers that are open, repairable, and fully owned by the user. Books are kept in open formats on SD cards, and there are no subscriptions or locked-down systems. The community helps develop everything, so you can share and change both software and hardware.

In the comments, many people are excited to see an open-source e-reader project, since most e-readers are now closed and hard to change. Some worry about the cost and ask if it can compete with cheap, mass-produced devices. Others say the $60 price for hardware is good for a DIY project, especially since it is repairable and not locked down.

A few readers mention they would like more support for PDFs or comic book formats. Some developers like the idea of a hardware-independent SDK, saying it would help more devices stay updated over time. Others point out that the open approach means you can add new features or fix things yourself, which is rare for big-brand devices.

People also talk about privacy. With FreeInk, you don’t need an account, and your data stays on your device. Some say this is important, as big companies now collect too much personal data. A few commenters share concerns about battery life compared to famous brands, but others say this can be improved by the community.

Overall, the comments show strong interest in having an open, hackable, and user-owned e-reader, even if it is not as polished as the big brands yet. Many hope projects like this will encourage more companies to open up their devices in the future.

## A digestion of the Jacobian conjecture counterexample

- 原文链接: [A digestion of the Jacobian conjecture counterexample](https://terrytao.wordpress.com/2026/07/21/a-digestion-of-the-jacobian-conjecture-counterexample/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48998362)

This article explains a new counterexample to the Jacobian conjecture in three dimensions. The Jacobian conjecture is a big math problem that asked: if a polynomial map has a non-zero constant Jacobian, is it always invertible with a polynomial inverse? Now, using Fable AI, someone found a map in three dimensions that breaks this rule.

The article says the Jacobian being non-zero means the map is locally invertible, like you can turn it around nearby any point. The hard question was: does local invertibility mean you can always invert the whole map everywhere? For one or two dimensions, the answer is still open or known to be true, but in three and more dimensions, this new example shows it is false.

The article gives a specific polynomial map as a counterexample. It shows how the map works and why it is not invertible, even though its Jacobian is a non-zero constant. The construction is tricky. At first, it looks like a miracle that the polynomial works, because there are many variables and many ways things could go wrong.

To explain the miracle, the author uses geometric ideas. He rewrites the map using multiplication of polynomials and shows how symmetries help. There is a scaling symmetry and other group actions. After removing these symmetries, the map is still not globally one-to-one. For most points, three different inputs give the same output, so the map is three-to-one in general.

The author uses more geometry and algebra to break the problem down. He shows that, using certain slices or restrictions, the map becomes locally invertible but not globally invertible. With careful changes of variables, he proves the variety (the set of solutions) is actually the same as ordinary three-dimensional space, so the counterexample really works.

He finishes by computing everything to check that the Jacobian stays constant and the map is not invertible. He also says he used an AI chatbot to help check the math.

Commenters on Hacker News have many reactions. Some are amazed that AI helped solve such a deep math problem, and they discuss the possible future of AI in math research. Others ask for simpler explanations, since the article is still very technical. Some mathematicians in the comments try to break down the key steps for non-experts, and there are links to extra resources for people who want to learn more.

A few commenters debate if this really settles the question, since the conjecture is still open in two dimensions. Some are surprised that the counterexample is so explicit and not just an existence proof. Others are sad to see a long-standing open problem solved in a way that closes the door to some elegant theories.

There’s also discussion about the role of symmetries in math, and how modern techniques like birational geometry and group actions help find these sorts of results. Some people share stories of trying to tackle the Jacobian conjecture in the past. Finally, a few users make jokes about needing new conjectures for AI to solve next.

## Long presumed dead, a thriving coral reef is discovered in West Africa

- 原文链接: [Long presumed dead, a thriving coral reef is discovered in West Africa](https://e360.yale.edu/digest/benin-coral-reef)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48993816)

Scientists in Benin have found a large, healthy coral reef off their coast that many thought was lost or dead for over 60 years. This reef was first mentioned in a small part of a report from the 1960s, but after that, no one knew exactly where it was or if it still existed.

Back in the 1960s, researchers noticed corals in fishing nets but did not study them further. Over time, with many coral reefs dying worldwide due to bleaching and overfishing, people believed the Benin reef was gone too. Recently, local scientists, led by Gérard Zinzindohoué, decided to look for this reef again. After getting an old report and a grant, they bought sonar equipment, but had trouble with payment and getting the right boat, since Benin has no research ships. They borrowed a fishing boat and, after many failed attempts and technical problems, finally found the reef using sonar and underwater cameras.

The reef is about 24 miles long, deep underwater, and is called a mesophotic coral ecosystem (MCE). It has at least eight types of coral and eight fish species, including some rare fish. The discovery brings hope for the reef’s future and could help scientists learn about the past climate in the region. The team wants to protect the area and hopes it becomes a marine protected zone, especially because it may also be home to sharks and rays.

The scientists also hope their work will inspire more research in West Africa, a place that is not well-studied. They believe local people should lead the way in learning about and protecting their own sea.

In the comments, many people feel happy and surprised that the reef has survived after so many years. Some users say it’s rare to hear good news about coral reefs, and this gives them hope. Others discuss the hard work and clever problem-solving by the Benin team, who kept going despite money and equipment troubles. A few people talk about how important it is for local scientists to do this kind of work, instead of waiting for foreign teams. ## Jack Dorsey launches Buzz to combine team chat, AI agents and Git hosting

- 原文链接: [Jack Dorsey launches Buzz to combine team chat, AI agents and Git hosting](https://runtimewire.com/article/jack-dorsey-block-buzz-team-chat-ai-agents-git)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48995213)

Jack Dorsey, the cofounder of Block, has launched a new open-source tool called Buzz. Buzz is made to combine team chat, AI agents, and Git code hosting into one workspace for software teams.

Buzz works by using a system called Nostr relay. Every action—like messages, code updates, or approvals—is saved as a signed event. Both people and AI agents have their own identity, keys, and can join channels or leave audit trails. Agents can do more than normal bots; they can search old chats, review and write code, run tasks, and even make new channels. Buzz brings together chat, code, workflows, and search into one app. For Git, Buzz plans to let each feature branch become its own chat channel, where code changes, reviews, and comments are all saved together. This means you have one place to see code, discussions, and history. Buzz works on macOS, Windows, and Linux, and you can host it yourself.

Buzz is called decentralized because each company can run its own relay server and keep control of their data and who uses it. But all activity inside a workspace still goes through that one server, so it is not fully peer-to-peer. This makes it easy to own your data but means you must handle backups, security, and server problems yourself.

Buzz is still early. Some features, like mobile apps or push notifications, are not finished yet. The current version lets you use chat, threads, code, shared notes, search, and basic agent features. Block (Dorsey's company) is the first user, but there are no details about other users, pricing, or how many people are using it yet.

Many Hacker News users like the idea of one tool for chat, code, and workflows, since switching between apps is annoying. Some people are excited that Buzz is open-source and not tied to big companies like Microsoft (GitHub) or Salesforce (Slack). Others are worried about needing to run your own server, which can be hard for small teams. A few users say that Buzz is not truly decentralized if all data still goes through a single server. Some comment that mature tools like Slack and GitHub are hard to replace because they are stable and have many features. Others point out that if Buzz succeeds, it could help push the software world toward more open and flexible tools. Some users are also cautious, saying that Dorsey’s projects sometimes start big and then lose steam, so they want to wait and see. ## 'VPNs are lawful technical tools,' says EU Court in landmark copyright ruling

- 原文链接: ['VPNs are lawful technical tools,' says EU Court in landmark copyright ruling](https://www.techradar.com/vpn/vpn-privacy-security/vpns-are-lawful-technical-tools-says-eu-court-in-landmark-anne-frank-copyright-ruling)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48997221)

The article is about a big legal case in the European Union where the top court said VPNs are legal tools, not illegal, and that VPN companies or publishers cannot be blamed if people use VPNs to get around country blocks on websites. The case started because a website in Belgium shared Anne Frank’s diary online, blocking Dutch users since the diary is still under copyright in the Netherlands but not in Belgium.

The court looked at whether the website and VPN providers should be responsible if someone in the Netherlands used a VPN to pretend they were in Belgium and read the diary. The court said no. If a website uses up-to-date geo-blocking (blocking by country), the publisher is not breaking the law just because someone uses a VPN to get past it. This means publishers don’t need to make perfect, unbreakable blocks, just reasonable ones. 

The court also said VPNs are “lawful technical tools.” This is important because it protects VPN companies from being sued if users do something wrong. The ruling is good news for people who use VPNs for privacy, security, or to access information online. It also means copyright owners can’t say that the existence of VPNs makes all blocks useless. The legal decision helps show that the internet can have country-based rules but still let people use privacy tools.

In the Hacker News comments, some people are happy about the court’s decision, saying it is good for internet freedom and privacy. Others think it is common sense that VPNs are just tools, like a car or a phone, and not bad on their own. A few worry that this might make copyright holders ask for even stronger blocks or new laws to limit VPNs in the future. Some users point out that this ruling could protect other privacy tools too, not just VPNs. Others share stories about how geo-blocking can be annoying and doesn’t always work well. 

Some commenters think the court’s focus on “state-of-the-art” blocking is interesting, asking who decides what is good enough. A few people worry that companies might use this as an excuse for weak blocks. Others debate if VPNs will still get targeted by governments later, but most agree this is a win for privacy for now. Overall, the mood is positive, and many see this as a step forward for technology users.

## "Drawing" the Mona Lisa with GPT-5.6, Claude, Gemini, and Grok

- 原文链接: ["Drawing" the Mona Lisa with GPT-5.6, Claude, Gemini, and Grok](https://www.tryai.dev/blog/ai-drawing-arena-colored-pencils-claude-gpt-grok)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48998404)

The article is about testing four AI models—GPT-5.6 Sol, Claude Fable 5, Gemini 3.6 Flash, and Grok 4.5—to see how well they can “draw” complex images like the Mona Lisa using colored-pencil tools on a digital canvas. The team gave each model the same tools: pick a color, adjust brush size and pressure, draw or erase, smudge to blend, and review their work, trying to match a target image or draw from a text prompt.

The models had to copy two famous paintings (Mona Lisa and Starry Night) and also create five drawings from text prompts, such as “a cozy cabin interior” or “a red rose in a glass vase.” The process was open-ended; models could see the target as much as they wanted and decide what to fix. The team measured the results by comparing the AI drawings to the originals using SSIM, a score for image similarity, and also looked at cost, time, and how the AI used the drawing tools.

Every model used the same tools, but in different ways. GPT-5.6 Sol used mostly draw and smudge, setting color and brush inside each draw call. Grok 4.5 changed settings constantly and made the most tool calls, but its art was the worst. Claude Fable 5 reviewed its work the most and smudged a lot, but was slow and expensive. Gemini 3.6 Flash reviewed its work almost obsessively.

In the end, Gemini 3.6 Flash got the highest SSIM scores, meaning its drawings matched the originals most closely by the numbers, but the team thought GPT-5.6 Sol’s art actually looked better to human eyes. GPT-5.6 Sol was fast, detailed, and cheap. Claude Fable 5 was much slower and cost about twenty times more than the others, but its results were only second-best. Grok 4.5 was the cheapest, but its drawings were not very good and often didn’t make sense.

The team also noticed that when models kept reviewing and editing, their drawings often got worse after reaching a peak—more edits didn’t mean better art. They shared that these tests are not perfect benchmarks for AI skill, but are fun and show where each model stands. The code is open source, so anyone can try these tests themselves.

In the Hacker News comments, some people liked the open, playful nature of the experiment and said it’s a better way to see what AIs can really do, compared to just looking at test scores. Others pointed out that AI art is still far from human creativity, and the results looked more like a child’s drawing than real art. A few commenters worried about the high costs, especially for Claude Fable 5, and wondered if these models are practical for real creative work yet.

Some users discussed the importance of the toolset, saying that maybe a more advanced set of tools or a different interface could help the models make better art. Others debated if SSIM is the right way to judge art, since a drawing can be closer in score but still look worse to people. There were also comments about the open-source code, with several users excited to try their own experiments.

A few people felt that Grok’s poor results showed it isn’t ready for serious creative tasks, while others defended it, saying maybe it’s better at other things. ## Apple defeats liability for not scanning iCloud for CSAM

- 原文链接: [Apple defeats liability for not scanning iCloud for CSAM](https://blog.ericgoldman.org/archives/2026/07/apple-defeats-liability-for-not-scanning-icloud-for-csam-but-the-judge-was-not-pleased-amy-v-apple.htm)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48992870)

Apple was sued for not scanning iCloud for child sexual abuse material (CSAM), but the court said Apple is not legally responsible. Apple had tried its own tool for scanning CSAM called NeuralHash, but it stopped using it and instead made iCloud files end-to-end encrypted.

The lawsuit argued that Apple should use available tools, like PhotoDNA, to find and block CSAM. The court said Section 230 protects Apple because Apple acts as a publisher of content made by others. This means Apple is not responsible for what users upload, even if Apple could have used better tools to scan for harmful material.

The court also said that Apple is not legally required to report CSAM it does not detect, and that changing iCloud’s design to scan for CSAM would mean Apple acts as a content moderator, which Section 230 covers. Other arguments from the plaintiffs, trying to use exceptions to Section 230, did not work. The judge noted that, under current law, companies do not have to scan for CSAM unless lawmakers make a new rule.

The judge was not happy with this outcome. She said the law allows companies like Apple to avoid scanning for CSAM, but also that making scanning mandatory would reduce privacy for millions of people. The judge pointed out that end-to-end encryption is important for user safety and privacy, but this same privacy can also help criminals hide CSAM.

She said lawmakers, not courts, should decide if companies must scan for CSAM. The judge also said that children harmed by CSAM are not protected well by current laws and deserve better. She worried about the balance between privacy and safety, and how legal immunity for tech companies can have real costs for victims.

On Hacker News, some people agreed with the court, saying Section 230 is important for internet freedom and privacy. Others thought Apple could do more to stop CSAM, even if it is hard. A few people worried that forcing companies to scan files would break encryption and harm everyone’s privacy. Some pointed out that real criminals would just use other ways to hide files or use other services. Others suggested that the main job of stopping CSAM should fall to law enforcement, not tech companies.

There were comments about the dangers of weakening privacy for everyone because of a few bad actors. Some users remembered past data leaks and hacks and did not trust big companies or governments to keep private files safe. A few people said that even if Apple could do more, the law is clear, and any change should come from lawmakers. Some users had sympathy for the victims, but still felt breaking privacy protections would be worse in the long run. Overall, the comments showed a split between wanting to protect children and wanting to protect privacy.

## The Birth of Prolog (1996)

- 原文链接: [The Birth of Prolog (1996)](https://dl.acm.org/doi/10.1145/234286.1057820)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48953468)

This article is about the history of Prolog, a programming language made in the 1970s for logic and artificial intelligence. Prolog was created by Alain Colmerauer and his team in France to help computers understand logic and language.

The article explains that Prolog is different from other languages because it uses logic rules, not lines of code, to solve problems. With Prolog, you tell the computer facts and rules, and then ask it questions. For example, you can write facts like “Socrates is a man” and rules like “All men are mortal.” Then, you can ask, “Is Socrates mortal?” and Prolog will answer based on the rules.

Prolog helps with problems where you need to search for solutions, like making schedules or understanding human language. It was important for early artificial intelligence because it could handle these logic problems better than other languages at the time. The article also talks about how Prolog became popular in Europe for teaching and research, especially in AI.

One challenge with Prolog is that it can be slow for some tasks, and it is not as common today as languages like Python or Java. Still, it was a big step forward for logic programming and thinking about computers in new ways.

In the comments, many people say Prolog helped them learn about logic and problem-solving. Some remember using it in college to solve puzzles or build simple AI programs. Others think Prolog is hard to use for big projects, but great for learning and small, smart tasks.

A few commenters wish more people knew about Prolog today, because it teaches a different way to think about programming. Some say it is hard to find real jobs using Prolog, but it is still respected for its ideas. One person mentions that Prolog’s way of thinking shows up in newer tools and AI systems. Others note that even if Prolog is not popular now, its ideas still matter in tech.

Some users also discuss the differences between Prolog and other logic programming languages, with a few saying they prefer other tools for real-world problems. But many agree that learning Prolog changes how you see computers and programming.
