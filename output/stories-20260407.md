## S3 Files and the changing face of S3

- 原文链接: [S3 Files and the changing face of S3](https://www.allthingsdistributed.com/2026/04/s3-files-and-the-changing-face-of-s3.html)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=47680404)

This article talks about a new feature called S3 Files, which lets you use Amazon S3 storage like a normal file system. The writer explains why this is important, especially for people who work with lots of data, like scientists and engineers.

Before, moving data to and from S3 was hard because S3 is an object store, but most tools want a file system. People had to copy files back and forth, making extra work and sometimes causing mistakes. The article gives an example from genomics, where researchers wanted to use cloud storage but struggled because their tools expected local files, not S3 objects.

AWS tried to fix this by making S3 Tables and S3 Vectors, which help with special data types like tables for analytics and vectors for AI search. But many users still needed to work with normal files. So, the team started to build S3 Files, letting users mount an S3 bucket as a file system on EC2, containers, or Lambda. Now, you can open, change, or save files directly on S3, just like on your computer.

The team found that file systems and object storage are very different. Files can be changed in small ways by many users at once, but S3 objects are usually replaced all at once and are treated as unchangeable. The designers decided to keep these differences clear, instead of hiding them, by making a "stage and commit" system. Changes are made in the file system, then saved to S3 in batches, usually every 60 seconds.

There were also problems with things like permissions and file names. File systems and S3 handle these differently, so the team made rules to keep each system working as expected. If a file or name doesn't fit on both sides, it is left out and a message is sent to the user. If two people change the same file at once, the version on S3 wins, and the other copy is saved in a lost+found folder.

Performance was another challenge. S3 is good at handling lots of big files at once, while file systems are better for many small changes. To help, S3 Files only loads file data when you need it, so you can start working right away, even with millions of files. For reading large files, it uses a "read bypass" trick to get high speeds.

The team chose not to mix the two systems too much, to avoid breaking old apps. Most users use either the file view or the object view at one time, not both. The system syncs changes between the two, so each side sees the latest data.

S3 Files is not perfect yet. Some things, like renaming big folders, are slow, and some file names won’t work. But the team is open about these limits and wants to keep improving.

In the Hacker News comments, many people liked the idea of S3 Files. Some said this solves a big problem they face every day, copying data between systems. Others were worried about the costs and possible confusion from mixing file and object storage. A few warned about edge cases, like file name issues or slow renames, and hope AWS is clear about these.

Some users asked if S3 Files could replace other file systems like EFS or NFS, while others think it works best for special cases, not all workloads. Some were excited about using S3 Files for data science or machine learning, since now tools like pandas can read data straight from S3.

A few commenters shared technical concerns, like what happens if two apps write to the same file at once, or how fast S3 Files is for lots of small files. Some said they want manual control over when changes are saved to S3, not just every 60 seconds.

Overall, the community thinks S3 Files is a good step forward, but they want to see how it works in real life. Some hope it will make S3 even more useful, while others say to watch out for the differences between file and object storage. People agree the clear boundary between files and objects is smart, and they like that AWS is honest about what works and what doesn’t.

## Bitcoin and Quantum Computing

- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=47681274)

This article talks about how quantum computers could break Bitcoin’s security in the future. The writer explains that if a strong quantum computer appears, Bitcoin’s digital signatures will not be safe, and changes are needed to keep Bitcoin secure.

The main points are: First, it is not clear when quantum computers will be able to break Bitcoin, but there is a real chance it could happen in the next 5-10 years. Second, to protect Bitcoin, the code and millions of wallets would need to upgrade to new, stronger cryptographic methods called post-quantum (PQ) signatures. This upgrade is not easy. It needs agreement from many people and new software on wallets and exchanges. If Bitcoin does not upgrade in time, there is a risk Bitcoin could fail.

The writer gives an example: if there’s a 10% chance a quantum computer can break Bitcoin by 2030, and a 50% chance Bitcoin upgrades in time, then there’s a 5% risk Bitcoin will break. Investors should consider this risk when deciding how much Bitcoin is worth. Even if you use Bitcoin and do not invest, the problem is still important. Cryptographers are starting to avoid working on systems that are not quantum-safe.

The article also explains that while new cryptographic methods exist, they are not perfect. They make transactions bigger and slower, and it is unclear which method is best. It is also hard to get all users to upgrade in time. If Bitcoin changes too early, it could pick the wrong method; too late, and it might be too risky.

The writer answers some common questions. For example, some people call quantum threats “FUD” (fear, uncertainty, doubt), but the writer says it is a real problem, not just hype. Some say quantum computers may never exist, but experts think there is a real risk, and it is better to prepare. Others say banks and the internet also need upgrades, but the writer trusts those systems will upgrade faster than Bitcoin, which is slow to change.

In the Hacker News comments, many users agreed that quantum computers are a real threat, but they debated how soon this will happen. Some think the timeline is long, so there is no need to rush. Others worry Bitcoin moves too slowly to respond in time. Some users believe new cryptographic methods will save Bitcoin, but others worry about the extra costs and complexity.

A few comments said that other systems, like banks and websites, are also at risk, so Bitcoin is not special. Some users do not trust big companies, like Google, and think they spread fear to hurt Bitcoin. Some point out that even if a quantum computer is built, it is not clear that anyone will use it to attack Bitcoin right away.

Many people agreed that getting everyone to upgrade their wallets and systems will be very hard. Some said the Bitcoin community needs to discuss and test new ideas now, not wait. Others believe that, in the end, Bitcoin will adapt, just as it has before, but there is no clear plan yet. Some wish the writer would help more with solutions, not just warnings. Overall, the comments show there is both worry and hope, but no easy answers yet.

## How to get better at guitar

- 原文链接: [How to get better at guitar](https://www.jakeworth.com/posts/how-to-get-better-at-guitar/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=47650887)

This article talks about how to get better at playing guitar, using a method shared by the teacher Justin Sandercoe. The author explains that this idea changed his own playing and could help others too.

First, the author remembers learning guitar in the 1990s using magazines with song tabs. He says just reading tabs did not make him a good player. Now, he learns by listening to songs and writing down what he hears—this is called transcribing. He suggests starting with easy songs that use simple riffs and are not too fast. The steps are: play the song, pause after each note, find the note on your guitar, and write it down on paper. Repeat this for every note in the song. 

After finishing, you should check your work by comparing it with tabs you find online. Sometimes you will make mistakes and need to fix them. Watching videos of guitarists can also help you see how they play, as some techniques are hard to hear. By doing this, you will start to remember the song and play it better, even the first time you try. You also get better at matching what you hear to the guitar.

The author says it is important to play whole songs, not just riffs, because this is what skilled guitarists do. He makes a playlist of songs he has learned and plays along for fun. He also tries playing different parts of the song, like the second guitar or bass line, to keep things interesting. Some example songs in his playlist are “Someday” by The Strokes and “Killing in the Name” by Rage Against the Machine.

He sums up the method: pick an easy song, play and pause to find each note, write them down, check your work, and keep practicing. Over time, you learn new techniques and get better.

In the Hacker News comments, some people agree that listening and transcribing is the best way to learn guitar and music in general. They say it helps with ear training and understanding music deeply. Others remember starting with tabs and feeling stuck, just like the author. A few share that it feels slow and hard at first but gets easier over time. Some suggest using software or apps that slow down songs, making it easier to hear each note. One commenter warns not to worry about making mistakes, because learning by ear always includes guessing and fixing errors. Another person says playing with friends or joining a band is also a great way to improve, because it forces you to listen and adapt. Some people mention that learning whole songs is more fun and motivating than playing short riffs. Others still enjoy reading tabs, especially for hard solos, but agree that listening is important. A few note that this method works for other instruments too, not just guitar. Finally, many thank the author for sharing a simple, honest way to practice and get better.

Magawa was trained by a Belgian charity called Apopo. He started working in 2016 and found explosives using his strong sense of smell. Magawa could search a large area, like a tennis court, in just 20 minutes. In total, he helped make more than 141,000 square meters of land safe. Because he was small and light, he could walk over mines without setting them off. This made him safer than humans for this job. Magawa even got a special gold medal for his work, which is like a big award for animals. He retired because he got old and died in 2022. The statue was made from local stone and was shown to the public on the International Day for Mine Awareness. The group that trained Magawa continues to train other “HeroRATS” to find landmines and even diseases like tuberculosis. Another rat, Ronin, has already broken Magawa’s record for finding the most landmines. Cambodia hopes to be free of landmines by 2030.
