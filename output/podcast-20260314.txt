Next, we look at “Han,” a new compiled programming language that uses Korean (Hangul) keywords and is written in Rust. Han lets people write code in Korean, supporting functions, variables, loops, conditionals, and printing—all using Hangul. It has five basic types, arrays, structs, pattern matching, closures, error handling, file I/O, and more. Han is both a compiler and interpreter, with a REPL and LSP server for coding help. The documentation gives examples like counted words, calculators, to-do lists, and line counters. You install it from GitHub and use simple commands to run or build code. The creator wants to show programming can use any language, not just English, and that Hangul is logical and clear. This is also good for people learning Korean. Some features are still missing, like enums, tuples, async, and better garbage collection, but Han is useful for many tasks. On Hacker News, people liked the idea and the cultural message. Some worry about sharing code across languages, but others see it as good for teaching or for fun. There was also talk about technical choices, like generating LLVM IR as text. Some shared stories about other languages in native scripts, and many respected the effort, even if they would not use Han themselves.

In hardware news, Fedora 44 is now running on the new Raspberry Pi 5. The new images work on all Pi 5B models, with support for serial console, micro SD boot, HDMI graphics, network, USB, and both KDE and GNOME desktops. Some features do not work yet, like NVMe storage, audio, thermal controls, and support for Pi 500 and CM5. Users need to add a line to the kernel command for graphics to work well, and should turn off auto-suspend. There are three images to download: Minimal, KDE, and GNOME. On Hacker News, readers are excited and thank the author, but some wish for audio and NVMe support. There is talk about Fedora versus Raspberry Pi OS, and some share performance tests or ideas for using the Pi 5 as a server or desktop. Some warn about overheating because thermal controls are not ready. In general, people are happy with the progress.

Now, let’s talk about the Model Context Protocol, or MCP. A few months ago, MCP was very popular, but now many in tech say Command Line Interfaces (CLIs) are better. The article explains that CLIs can save tokens for AI, but only if you use known tools. For new or custom tools, you still need to explain them to the AI. MCP is more useful for big teams, especially over the internet, because it helps with sharing, security, tracking, and updates. This is less useful for small projects, where CLIs are simpler. The comments agree that hype in tech comes and goes. For small teams, MCP feels like too much, while big companies may need its features. Some worry about centralizing secrets and vendor lock-in, but others like always having up-to-date docs. The best choice depends on your team size, security, and workflow.

In U.S. legal news, Montana passed the Right to Compute Act in 2025. This law gives people more freedom to use, fix, and change their own software and hardware. People can bypass digital locks and repair devices they own, as long as it is for personal use and does not hurt others. This helps tinkerers, small businesses, and even farmers who fix their own machines. Many see this as a win for user freedom and hope other states follow. Some worry about piracy or copyright, and others wonder if big companies will fight back. Some say this is good for the environment, since more repairs mean less waste. Most support the law, but agree details of how it is enforced will matter.

In tech graphics, there is a new recursive algorithm for rendering 3D Signed Distance Fields, or SDFs. SDFs use math to describe shapes, not polygons, and are popular for creative graphics. The new method splits the screen into smaller parts and can skip empty space quickly, making rendering three times faster on a CPU. If a patch is flat, you can shade all pixels at once by interpolation, making it even faster. This allows simple SDF scenes to run at 50 to 100 frames per second on a single CPU core. People on Hacker News are excited, sharing similar ideas like cone marching, and wonder if this can be used for bigger scenes or on the web. Some compare it to GPU methods and discuss the limits, like with complex shapes or animation. Many see this as a good way to teach graphics programming, and hope for more open-source projects using these ideas.

Lastly, Baochip-1x is a new chip made for secure, high-assurance software, and it includes an MMU—a feature rare in small microcontrollers. Created by bunnie, this chip uses RISC-V, is partly open-source, and runs real operating systems like Linux or Rust-based Xous. Bunnie made the chip affordable by fitting it into unused space on another company’s project. This chip uses RRAM memory, which is fast and keeps data when off. Security certification is hard for open chips, but hundreds of Baochips are already in developer hands. On Hacker News, people are excited to see an MMU in a small chip and hope it breaks ARM’s market hold. Some want the analog parts to be open too, but agree that “partly open” is a big step. There is debate about certification and adoption speed, but most support open hardware and look forward to running Linux or Rust on these chips.
