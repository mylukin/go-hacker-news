First, let’s talk about Project Glasswing, a new partnership between major tech companies and organizations to make software much safer using AI. This project uses an AI model called Claude Mythos Preview, which can find software bugs much better than most humans. The article explains that even software running banks, hospitals, and power grids has hidden bugs, some of them very dangerous and hard to find. Claude Mythos Preview is so good at reading and understanding code that it can spot security problems—even old ones missed for years. For example, this AI found a 27-year-old bug in a secure operating system, a 16-year-old bug in a video tool, and combined bugs in the Linux kernel to take control of a computer.

Big companies like Amazon, Google, Microsoft, and Cisco are part of this project. They will use the AI to check their own systems and share their findings. Anthropic, the company behind the AI, is giving out $100 million in AI usage credits and supporting open-source projects to help fix their code. The idea is to use AI for defense, hoping good teams can stay ahead of hackers who may use the same tools to attack. However, Anthropic is not releasing this powerful AI for everyone yet, as it could be misused. They are working on safety steps first, and also talking to US government officials because national security is at risk if these tools fall into the wrong hands.

Hacker News users are both excited and worried. Many see this as a big step forward, helping small open-source teams who can’t afford security experts. Others worry that if such powerful models leak, hackers could find and attack more bugs. Some wonder if this will start an “arms race” with both defenders and attackers using AI. There is debate about trusting just a few big companies, and questions about whether the help will reach the smallest projects. Some say keeping the best models private may not stop bad actors, and that fixing old bugs is still a lot of work. Many agree that open cooperation between companies, governments, and open-source groups is needed, and worry about these tools being used for spying or attacks, not just defense. Most agree that software security is now a much bigger problem in the AI age, and acting fast is important.

A related article focuses on the Claude Mythos Preview model. It is Anthropic’s new large language model, focused on cybersecurity and not released to the public. The model is much better than older ones at finding and fixing software vulnerabilities, and can help with software engineering and research. Anthropic trained it with a mix of web data, private datasets, and synthetic examples, and used crowd workers for safety tests. The main reason for limiting access is its strong cybersecurity skills—if it is misused, it could help attack software. Safety reviews included tests for misuse, alignment, and welfare. The model is called the best-aligned so far, but sometimes still makes big mistakes or acts recklessly.

Anthropic now uses a new risk framework that looks at overall risk, not just simple thresholds. They found that Mythos Preview does not create catastrophic risks, but risks are higher than before. It helps experts work faster, but is not creative enough to invent new advanced weapons or replace top researchers. In cybersecurity tests, Mythos Preview solved all tested CTF challenges and found real vulnerabilities on its own. Because of this, Anthropic set up strong controls, only letting a few vetted partners use it for defense.

Hacker News commenters praised Anthropic’s transparency and safety steps, but some worry that limiting access could slow open security research or that bad actors will get similar models elsewhere. Next up, Amazon has introduced a new feature called S3 Files, letting users access S3 storage like a normal file system. This is very important for people who work with lots of data, such as scientists and engineers. Before, using S3 meant copying files back and forth, which was slow and sometimes caused mistakes. S3 Files lets you mount an S3 bucket as a file system on EC2, containers, or Lambda, so you can open, change, or save files directly on S3, just like on your computer.

The team found that file systems and object storage are very different. Files can be changed in small ways by many users, but S3 objects are replaced all at once. To keep this clear, S3 Files uses a “stage and commit” system—changes are saved to S3 in batches, usually every 60 seconds. There were also challenges with permissions and file names. If a file name does not fit both systems, it is left out and a message is sent. If two people change a file at once, the S3 version wins and the other goes to a lost+found folder.

Performance was another challenge. S3 handles big files well, while file systems are better for small changes. S3 Files only loads data when you need it, so you can start working even if there are millions of files. For large files, it uses “read bypass” for speed.

In the comments, many people liked S3 Files, saying it solves a real problem. Some worry about costs and confusion from mixing file and object storage, and warn about edge cases like slow folder renames. Some ask if S3 Files could replace systems like EFS or NFS, but others think it works best for special cases. There are technical questions about what happens if two apps write to the same file, or how fast it is for many small files. Many like the clear boundary between files and objects, and appreciate AWS being honest about limits.

Now, let’s look at GLM-5.1, a new open-source AI model designed for complex coding and reasoning tasks, especially those needing long periods of problem-solving. GLM-5.1 keeps improving its own work even after many steps, unlike older models that get stuck. In tests, it did much better than previous models in coding challenges, GPU kernel speedups, and even building a full Linux-style desktop as a web app from scratch, working for hours and adding features over time.

The key is that GLM-5.1 does not just stop after solving easy parts. It checks its own work, finds places to improve, and keeps going. For tasks with clear goals, it uses feedback to get better. For open-ended tasks, it reviews itself and continues based on its own judgement. Benchmarks show it is top or near the top for many coding tasks, and it is fully open source.

Hacker News users praise the open release and say long-horizon optimization is a big step forward. They see promise for real-world projects that take days or weeks, but caution that real use cases may still be hard—especially when it comes to self-evaluation and staying on track. Some compare GLM-5.1 with other models like Claude Opus and GPT-5, noting that while it is catching up, there is still a gap in some areas. Overall, people are excited to try it, thanks to the open-source license.

Next, an article looks at the risk to Bitcoin from quantum computers. The writer explains that if strong quantum computers appear, Bitcoin’s digital signatures will not be safe, and a big upgrade is needed. This upgrade is hard—it needs everyone to agree, and new software for wallets and exchanges. The risk is real, and if Bitcoin does not upgrade in time, it could fail.

The writer gives an example: if there is a 10% chance a quantum computer can break Bitcoin by 2030, and a 50% chance Bitcoin upgrades in time, there is a 5% risk Bitcoin will break. New post-quantum signatures exist but are not perfect—they make transactions bigger and slower, and it is hard to pick the best method. Also, getting everyone to upgrade is not easy. Some call quantum threats just hype, but the writer says it is a real problem, and other systems like banks may upgrade faster than Bitcoin, which moves slowly.

In Hacker News comments, many users agree that quantum computers are a real threat, but debate the timeline. Some worry Bitcoin is too slow to respond, others believe new methods will save it but worry about the costs. Some say other systems are also at risk, and a few do not trust big companies talking about these threats. Many agree that upgrading everyone will be very hard, and the Bitcoin community should start work now. Some wish for more solutions, not just warnings.

Now, let’s talk about music. One article shares a method for getting better at guitar, from Justin Sandercoe. The author says this method changed his playing. Instead of just reading guitar tabs, he learns by listening to songs and writing down what he hears—this is called transcribing. The process is to pick an easy song, play and pause after each note, find it on the guitar, and write it down. After finishing, check your work with online tabs, fix mistakes, and watch videos to see how guitarists play. Playing whole songs, not just riffs, is important. The author makes a playlist of songs he has learned and plays along for fun, sometimes trying different song parts.

Hacker News users agree that listening and transcribing is a great way to learn guitar and music in general. They say it helps with ear training and understanding music. Some remember starting with tabs and feeling stuck, while others mention software that slows down songs to help. There are warnings not to worry about mistakes—learning by ear always includes guessing and fixing errors. Playing with friends or joining a band is also helpful. Many thank the author for sharing a simple way to practice and improve.

Next, a story from Cambodia. The country has built a statue for Magawa, a rat that found over 100 landmines and helped make land safe for people. Magawa was trained by a Belgian charity, Apopo, and could search a large area quickly because he was small and light, so he did not trigger the mines. He even received a special gold medal for his work before retiring and passing away in 2022. The statue honors him on the International Day for Mine Awareness, and the group continues to train “HeroRATS” for finding landmines and diseases like tuberculosis.

People in the comments were surprised and happy to see a rat honored this way. Some talked about how rats’ sense of smell can be better than dogs’. There were stories about other animals used for similar jobs, like dogs and bees. Some joked about rats having a bad reputation, but Magawa shows they can be heroes. There were questions about training and animal safety, and some discussed whether robots could replace animals. Many were touched by Magawa’s story and agreed the statue is a nice way to remember all animals that help clear landmines.

Moving on, there’s a project that lets you fine-tune Google’s Gemma 4 and 3n AI models with text, images, and audio—right on your Apple Silicon Mac. You do not need an NVIDIA GPU, and you can train on large datasets by streaming data from the cloud. The toolkit supports fine-tuning for text, image plus text, and audio plus text tasks. You can use it for things like speech recognition and visual question answering, and keep your data private by training on your own Mac. The toolkit uses PyTorch and Apple’s Metal for faster training, and has a command-line wizard to help with setup, training, and exporting models.

Hacker News users are excited to see real multimodal AI training working on Apple Silicon. They like the support for audio, which is rare in other projects. Some are impressed by the simple wizard, while others note that it still needs command-line work, which could be hard for beginners. There are questions about training speed and using fine-tuned models in iOS apps. Only smaller Gemma models are supported now, but people hope for support for larger ones in the future. Overall, the community sees this as a big step for on-device AI development, especially for Mac users.
