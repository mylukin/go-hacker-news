Let’s start with the story from Europe. Members of the European Parliament, including Stelios Kouloglou, have been attacked with Pegasus spyware. Kouloglou was working with the PEGA Committee, a group looking into how governments use spyware in the EU. His phone was infected with Pegasus at two important times: October 2022 and March 2023, when the committee was busy writing reports and planning country visits. Pegasus, made by NSO Group, is very powerful—it can read your messages, listen to your calls, and steal private information. The Citizen Lab found the infections but did not say which country was responsible. They think the attacker may be allowed to spy in several European countries, and the same group may have targeted Russian and Belarusian journalists too.

Kouloglou, who is also a journalist, was in the hospital when his phone was hacked, and he met another journalist who was also a spyware victim. This means private medical and committee details could have been stolen. Apple sent warnings to Kouloglou, but he missed them. Other members of the Parliament have also been targeted, and there are worries that more people were hacked without knowing. Experts say this kind of spying is dangerous for democracy, as hackers could see secret talks and maybe influence the Parliament’s work. The article calls for better device checks, more support for members, and stronger investigations.

In the comments, readers are shocked that even top officials are not safe. Some say Apple’s warnings are not enough. Others think spyware like Pegasus is too strong and should be banned. Next, let’s talk about SearXNG, a free and open-source metasearch engine. SearXNG lets you search the internet without being tracked or profiled. It collects results from many search engines, like Google, Bing, and DuckDuckGo, and shows them in one place. The project is on GitHub, uses the AGPL license, and is mainly written in Python. You can run it on your own server or use public ones set up by volunteers. SearXNG does not store user data, and you can change its settings as you like. Many people use SearXNG for better privacy and to avoid big tech companies tracking their searches.

In the comments, users like that SearXNG protects their privacy. Some run their own servers for more control, while others use public instances but worry these can be blocked. Some say results are not always as good or fast as Google, but think privacy is worth it. There are tips about running SearXNG with Tor or a VPN, and some discuss problems with search engines changing their APIs, which can break SearXNG. Running a public server can be hard because of high traffic and risk of being blocked. Some suggest using SearXNG with other private tools, and remind that you still need to trust the server you use. Many wish more people knew about privacy tools like SearXNG.

Next, we have a story about moving a server from Ubuntu to FreeBSD, and confusion over how FreeBSD reports RAM usage. Modern operating systems use all available RAM for cache, so systems run faster. FreeBSD splits memory into different types: active, inactive, laundry, wired, and free. With ZFS file systems, it also uses ARC cache, which takes a lot of RAM for faster file access. Tools like top, htop, and btop report memory differently, sometimes showing strange numbers because of how they read system data. The author fixed some of these tools to better handle ARC and memory stats, and learned how tricky memory reporting can be.

In the comments, many users share the same confusion when switching to FreeBSD or Linux. People say “unused RAM is wasted RAM,” so low free memory is actually good. Others point out that different tools can be misleading, and it’s better to know how your OS really works. There are jokes about “bad RAM tools,” and reminders that swap use is normal, even with lots of RAM. Some thank the author for fixing tools and sharing their findings. Overall, the discussion shows that memory reporting is hard, and you should learn your system instead of trusting one tool.
