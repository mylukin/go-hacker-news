Next, let’s talk about a project that lets you run a huge AI language model, GLM-5.2 with 744 billion parameters, on a normal computer with only 25 GB of RAM. This is done using a tool called colibrì, written in pure C with no extra dependencies. The main idea is to keep only a small part of the model in memory; the rest is split into 21,000 "expert" parts and stored on disk. As you use the model, it loads only what is needed, using smart caching to keep things fast. The tool uses its own tokenizer, manages memory, and avoids using GPUs or Python at runtime.

When you start, there is a heavy load on your SSD, and the project warns this could wear out cheap drives. But as you use it more, the tool gets faster. If you have more RAM or a better SSD, the speed improves. The developer asks the community to help test accuracy with compressed model weights, as they cannot do it alone.

Comments are very positive. People are amazed that such a large model can run on a regular computer. Some worry about SSD wear and slow speed, but many like that this method makes big models open to everyone, not just those with expensive GPUs. There’s talk about using this for private AI assistants or research, and some share ideas for more optimizations. There are warnings that for real-time use, you still need fast hardware. Overall, people are excited, and many offer to help with testing and benchmarks.

Now, a change in privacy law: the EU has passed “Chat Control 1.0,” which allows tech companies to scan people’s private messages for child abuse, even without any reason to suspect them. The law passed even though most parliament members voted against it, so it will stay until 2028. There is a small exception for encrypted messages, but this does not change much. Some politicians wanted scanning only for people under investigation, but this did not pass.

Patrick Breyer, a civil rights activist, says this law is bad for democracy and does not really protect children. He says scanning everyone causes many false alarms, wastes police time, and rarely helps find new crimes. Most useful reports come from public posts, not private messages. The article says there is no proof that scanning everyone’s messages leads to more arrests or rescues.

In the comments, many people worry this law hurts privacy and could be abused. Some agree the law does not really help children, and just creates more work for police. Others point out that criminals using encrypted apps will not be caught. A few support the law, saying child safety is important, but most want a more careful approach. Some abuse survivors say privacy is important, and mass scanning could hurt victims. There are also worries about false alarms and innocent people being investigated. Most commenters do not think this law will solve the real problems.

Next, there’s a tip for parents: you can turn an iPhone into a safe, simple phone for kids using a hidden feature called Assistive Access, found in Accessibility settings. With this, parents can choose only a few apps for the child, like Calls, Messages, and Maps. The interface becomes simple, with big buttons. Web browsers can be blocked, and parents control who the child can contact. The phone cannot be turned off in this mode, and only the parent can reset it.

This means parents don’t need to buy a new “dumb phone.” An old iPhone can be made safe and simple, with tracking still possible. Downsides include slower speed, no voicemail, and some minor bugs.

Comments are mostly positive. Many are happy to find a free way to make a child’s phone safe. Some wish Apple would advertise this feature more. Others say the feature is useful for older relatives too. There are worries about losing an expensive phone but most think the benefits are worth it. Some debate if this is better than buying a basic phone, but most agree that tracking and control are big advantages.

Now, a story about system design. The article says many companies use big, complex distributed systems when a single laptop or server would be faster and much cheaper. In a famous study, a single laptop beat a large cluster in big data tasks. The author explains that most companies do not have enough data to need complex systems, but still use them out of habit, to build resumes, or because cloud vendors push them.

The advice is to always test if a single-machine setup can do the job before building a distributed system. Many readers agree and share stories of wasted money and time on clusters. Some say distributed systems are needed for high reliability or special cases, but most think simple systems are better for most teams. There are worries about “premature optimization,” but many agree measuring and testing is the smart way.

Some people are happy there is no leap second this time, making programming easier. Others say we should stop using leap seconds, or look for new ways to handle time. Some share stories about bugs caused by past leap seconds. Most agree that time in code is always tricky.

Finally, we have an article about Lisp, a unique programming language. Lisp looks strange at first, with many parentheses, but is very powerful. It lets you write code that changes the language itself, thanks to macros. Lisp treats code as data, which makes it easy to create new features and tools. It also supports live coding with its REPL, letting you update and test code on the fly.

Lisp is used in tools like Emacs and AutoCAD, letting users write plugins and automate tasks. The author says learning Lisp can change how you think about programming, even if you don’t use it daily.

In the comments, some people say they found Lisp hard at first, but later loved its power. Others warn that macros can be dangerous if used badly. Some talk about how REPL development makes fixing bugs fast, and share stories of automating tasks in Emacs. A few say Lisp’s design is elegant but hard to read for beginners. Many note that new languages have borrowed ideas from Lisp. Most agree that learning Lisp can open your mind to new ways of programming.
