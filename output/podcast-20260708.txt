First, let’s talk about how OpenAI checks if its AI models are good at coding. OpenAI has used special tests, called benchmarks, to see how well its AI understands and writes code. At first, they used SWE-bench Verified, but soon found it had big problems, like letting in outside information by mistake. They changed to a new test, SWE-Bench Pro, which is more realistic. This test asks the AI to change real code and make sure new features work, without breaking anything else. The best AI models got much better on this test in less than a year.

However, OpenAI found that about 30% of the tasks in SWE-Bench Pro were broken. Problems included tests that were too strict, missing details, or having confusing instructions. OpenAI used both computers and real software engineers to review the tasks. The computer program flagged about 27% as broken, but humans found even more—about 34%. Sometimes, the computer and humans disagreed, but often they found the same problems. Humans were especially good at seeing when tests didn’t check enough things.

OpenAI says these problems are hard to avoid because the tests come from real open-source projects, which are often messy and made for people, not AI. Tests can be too strict, expecting a certain answer and not allowing other good solutions. But as AI gets better, it can also help find these broken tests faster.

Now, OpenAI says coding tests should be made by expert software developers, just for testing AI. They no longer recommend using SWE-Bench Pro and want better, fair tests that are hard to cheat and really show if the AI is good at coding.

In the comments, many people agree that coding benchmarks are often flawed. Some say it’s very hard to make tests that are both fair and realistic, because real-world code is complicated. Some developers share stories about their own code passing all tests but still being rejected because it looked different from the example solution. Others point out that benchmarks made from open-source projects are messy, since real code involves a lot of hidden rules and communication. Some like the idea of using AI to check tests, but warn that AI can also miss things. Many agree with OpenAI: better benchmarks made by experts are needed.

Moving on, let’s look at Chatto, a new open-source group chat app that you can host on your own server for free. Chatto is fast, easy to use, and focuses on privacy, with full data encryption and no tracking or ads. You can install it quickly using Homebrew or download it for Linux, macOS, and Windows. The setup is simple—just run one executable, and both front and backend are ready. Each Chatto server hosts one community, with no data sharing between servers. If you want to join or host more communities, you just connect to or run more servers.

Chatto supports group voice and video calls, including screen sharing, all end-to-end encrypted. It uses very little system resources and is lightweight. The roadmap includes more features like moderation tools and better support for running many servers. The developer aims for version 1.0 in a year, but breaking changes may happen before then.

If you don’t want to self-host, a paid hosted version called Chatto Cloud is coming soon, focused only on hosting with no ads or extra features. It will use European servers, offer automatic backups, and let you move your data at any time.

On Hacker News, many are excited about a new open-source chat tool, especially one that is easy to set up and focused on privacy. Some users ask how Chatto compares to Matrix, Mattermost, or Discord, especially about federation and features. Not having federation means you can’t message users on other Chatto servers, which is both good for privacy and bad for connecting communities. Some worry about adoption, since many groups already use tools like Slack or Discord, but others like the simpler, no-tracking approach. There are questions about the strength of encryption and trusting a new tool. People praise the easy self-hosting and clean design, but some want more moderation tools and mobile apps. Overall, there is both excitement and caution, with many helpful suggestions for the creator.

Now, let’s enjoy a lighter story. Uniqlo has a new t-shirt with a special design: a chunk of obfuscated Bash script on the back, made by Akamai for their “Peace for All” campaign. The author describes how his wife showed him the shirt, which features a heart on the front and a block of code on the back. He noticed a shebang (`#!/bin/bash`) and saw that the real code was hidden with base64 encoding, a common trick for hiding what a script does. He used OCR tools to copy the code from the shirt, since base64 must be perfect. Some shirts had incomplete code that would not run.

After decoding, he found the script prints a looping, colorful “PEACE FOR ALL” message in a sine wave shape across the terminal—a fun Easter egg. The author also discussed the font, learning from Hacker News that it was Roboto Mono, not Consolas.

Akamai said the shirt is a tribute to the early days of the internet, with references to Linux and “beige box” computers. Many people online have noticed and decoded the shirt.

In the comments, people found the shirt and code idea clever and fun. Some called it a nerd badge of honor or a way to find like-minded people in public. Others talked about how base64 and `eval` are used in malware, and joked about the risks of running unknown scripts. Font choice was also discussed, with some correcting the article’s guess. There were comments about the effort needed to transcribe such code, and some shared links to other shirts with real code. Overall, it’s a playful mix of tech and fashion.

Next, Mistral AI announced Robostral Navigate, a new AI model that lets robots move with only one regular camera and plain-language instructions. The model works in offices, homes, and outdoors, and does better than robots that need extra sensors like LiDAR or many cameras.

Robostral Navigate is an 8-billion-parameter model trained only with simulated data. It understands images and language, so you can tell a robot to go somewhere, and it will figure out how. The robot looks through its camera and points to the next place it should go, or gives simple directions if the target isn’t in view.

Training was made faster with prefix-caching, letting the AI learn from whole episodes instead of one step at a time. After initial training, the model improved by learning from its own mistakes in simulated worlds.

Robostral Navigate has a 76.6% success rate in new places, better than other robots with more sensors. It works with many robot shapes and can handle unexpected obstacles and camera differences.

On Hacker News, many are impressed that Robostral Navigate works so well with just one camera. Some say this can make robots cheaper and easier to build. Others wonder how it will work in places with bad lighting or strange layouts, and if it can handle crowded spaces. Some like that it was trained only in simulation, which saves time and money, but are curious about real-world results. There are debates about the pointing navigation method and questions about safety, power needs, and how it compares to big company projects. Many are excited about future uses in delivery, cleaning, or warehouses, and hope for open-source releases. Overall, people see this as a big step for low-cost, useful robots.

Switching gears, Cloudflare has released a new tool called "Drop" that lets you upload a folder or zip file to instantly create a website. You don’t need to use the command line, install anything, or even sign up. Just drag and drop your files—HTML, CSS, JavaScript—and your site goes live with a unique URL. It’s free and fast, built on Cloudflare’s global network, but meant only for static content, not for sites with backends.

On Hacker News, people like how simple and fast Drop is, saying it’s great for sharing small projects or testing ideas. Some compare it to Netlify Drop or Vercel, but note that Cloudflare’s version doesn’t even ask for your email. There are some worries about security or someone sharing private data by mistake. Some wish for features like custom domains or password protection, and wonder if Drop will stay free. But most agree it’s a useful tool for beginners or quick projects.

Microsoft has just launched Flint, a new tool for making charts for AI agents and developers. Flint lets you write a simple chart description, and then it creates a nice chart for you, without needing to set all the low-level details. You just give your data, say what each column means, and pick a chart type. Flint’s “compiler” does the rest—choosing axes, colors, and layout.

Flint supports 46 chart types and works with popular libraries like Vega-Lite, ECharts, and Chart.js. If you want to change chart types, you just edit your description. Flint uses “semantic types,” so it knows if your data is a date, category, or number, and picks the best display. It also automatically arranges charts as you add more data.

Flint is open source and can be installed with npm. There’s also a gallery with many examples.

On Hacker News, users say Flint could save a lot of time, especially for dashboards and quick visualizations. People like how Flint hides the messy details and lets you focus on what you want to show. Some ask if Flint is really new or just another layer on top of tools like Vega-Lite. Others worry that hiding details may make it hard to fix problems, or that Flint might not support every chart type. Some are happy Flint is open source and want to try it in their projects. Overall, most think Flint is a helpful step for making data visualization easier, but want to see how it works in real projects.

Let’s talk about DocuBrowse, a tool that lets you search and manage big piles of documents on your computer using AI. DocuBrowse can handle many file types—PDFs, Word docs, eBooks, and more. It builds a special index of your files, so you can search by meaning, not just keywords. For example, you can search for “that contract about the lease renewal,” and it will find the right document even if those exact words are not in the file. Each result shows an AI summary, so you can see what’s inside before opening it. Everything happens on your own computer with local AI models, so your data never leaves your machine.

DocuBrowse mixes keyword and semantic search for better results, using local AI models through Ollama. It can read many formats and scans for private info, removing files that have sensitive data. The interface is simple and fast, with dark and light themes, paged results, and easy ways to open, delete, or get summaries. DocuBrowse is easy to install on Linux, Windows, and Mac, but some setups may be tricky due to dependencies. Right now, it only works in English, and there’s no authentication—it’s designed for single-user, local use.

In the comments, people like that DocuBrowse runs fully offline and keeps data private. Some praise the support for many file types and fast, AI-powered search. There are some worries about setup, memory use, and missing features like OCR for scanned files. Some developers want to add more features or deploy it with Docker. Users compare it to other tools, but many like its strong privacy focus and flexibility.

Finally, OpenAI has launched GPT-Live, a new AI voice model for ChatGPT. GPT-Live makes talking with ChatGPT feel much more like talking to a real person. It uses a full-duplex system, so it can listen and talk at the same time. It uses small phrases to show it’s listening, lets you interrupt, and keeps the conversation smooth. For harder questions, it quietly sends them to a stronger AI and keeps chatting while it finds the answer. There are two versions: one for paid users and a mini for free users. You can now see visuals for things like weather or stocks while you talk, and choose between quicker or deeper answers. The voices are more natural, and GPT-Live can better ignore background noise. Safety is a big focus, with new rules to avoid unsafe content and special protections for teens. It won’t copy real voices and can warn or stop risky conversations. GPT-Live works on iOS, Android, and chatgpt.com, with more features like video coming soon.

On Hacker News, people are excited about how natural the conversations feel. Some worry about privacy and what OpenAI does with voice data. Others worry about users becoming too attached to AI, especially teens. Some mention that safety controls may block useful conversations by mistake. There are also concerns about the AI misunderstanding accents or not working well in all languages. Developers want details on pricing and limits for the new API. Some point out the voice is still a bit robotic at times. Others think GPT-Live could change how we use phones or help people who need hands-free help. Overall, it’s seen as a big step for AI voice, but there are still problems to solve.
