## A Better R Programming Experience Thanks to Tree-sitter

- 原文链接: [A Better R Programming Experience Thanks to Tree-sitter](https://ropensci.org/blog/2026/04/02/tree-sitter-overview/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=47799573)

This article is about how Tree-sitter, a code parsing tool, has improved the R programming experience, thanks to a new grammar file made for R. Many new tools and features for R developers are now possible because of this work.

Tree-sitter is a fast code parser written in C. It helps computers understand the structure of code by turning it into a tree. This is useful for things like code formatting, finding functions, and giving help in editors. Before, R had its own ways to parse code, but Tree-sitter can parse many languages and is faster, especially when editing code live.

To work with a new language, Tree-sitter needs a “grammar” file that explains the language rules. Davis Vaughan and others made this file for R, and now tools can use Tree-sitter to work with R code. Some of these tools include Air (for reformatting code), Jarl (for linting and fixing code), and the Positron IDE (for autocomplete and help). GitHub now uses Tree-sitter to give better search and navigation for R code.

The article explains that Tree-sitter-based tools can also help with searching code, finding dependencies, refactoring, and even running mutation tests. Tools like ast-grep help search code smarter than regular expressions. There are also fast command-line tools for diffing code (like difftastic) and parsing files like JSON and TOML, while keeping comments.

The article says that many of these tools are built using Rust, which makes them faster and easier to use outside of R itself. The R ecosystem is getting richer, and the author invites readers to join in building or improving these tools.

In the Hacker News comments, many people say Tree-sitter is a great tool and are happy to see it used for R. Some users explain how Tree-sitter has also improved editors like VS Code and Neovim for other languages. Others like that Tree-sitter is fast and can be used in many places, not just R. A few users talk about how hard it can be to write a grammar file, and they thank the people who did this work for R.

There are also comments discussing how tools like Air and Jarl could help teams write better, cleaner R code. Some people wish other languages would get similar grammar files to help their communities. A few users worry about how to keep these grammar files updated as languages change, but others point out that Tree-sitter has an active group of maintainers.

Some users note that using Rust for these tools is smart because it makes them run quickly and helps with big codebases. There are also questions about how well Tree-sitter handles tricky R code, and some answers say it is already very good, but could get even better.

Overall, the comments show excitement about Tree-sitter’s future in the R community and are glad to see open tools making R coding easier and more fun.

## The Beginning of Scarcity in AI

- 原文链接: [The Beginning of Scarcity in AI](https://tomtunguz.com/ai-compute-crisis-2026/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=47799322)

This article talks about a new problem in AI: there are not enough powerful computer chips for everyone who wants to use them. The price to rent Nvidia’s newest AI chips has gone up almost 50% in just two months, and companies like CoreWeave are making customers sign much longer contracts.

Big companies like OpenAI say they cannot do all the projects they want because they do not have enough computer power. Some AI companies are only giving access to their best AI models to about forty other companies, so not everyone can use the newest AI tools. This is a big change from before, when new AI models were easier to get.

The article lists five signs of this new “scarcity era.” First, companies will sell their best AI models only to their best or biggest customers. Second, using the best AI will become very expensive, so only companies with lots of money will get it. Third, even if you can pay, you might have to wait because the computers are busy. Fourth, prices will keep going up as more people want these chips, but there are not enough to go around. Fifth, developers will need to try other things, like using smaller AI models or building their own systems, until there are more chips and data centers, which may take years.

The writer says the time when AI was cheap and easy to get is now over, and it could stay this way for a long time.

In the comments, many people agree that chip shortages are making it harder to do AI work, especially for small companies and startups. Some say this could slow down AI progress and make it less fair, because only big companies can afford the best tools. Others point out that shortages might push people to build better, cheaper, or more efficient AI models that do not need so much computer power.

A few commenters worry that this scarcity will stop open-source AI projects, because they cannot pay high prices for chips. Some people say that renting chips is now like fighting for cloud servers was years ago, but even worse. Others think that this problem will not last forever, since companies will build more data centers and make new kinds of chips.

Some suggest this is a good chance for new businesses to make money by offering cheaper or smarter chip solutions. A few users also warn that if AI gets too expensive, it could slow down progress in other parts of tech, not just AI. Overall, the comments show people are worried about fairness, speed of progress, and new business ideas in this new age of AI scarcity.

The article explains how Android CLI, new Android skills, and the Android Knowledge Base make app development easier and more modern, both with and without Android Studio.

The Android CLI is a command-line tool that lets you set up environments, create projects, and manage devices quickly. For example, you can use commands like `android sdk install` to get only the parts you need, and `android create` to start a new project from an official template. This saves time and makes sure your app uses the latest best practices. Google says their tests showed agents using Android CLI finished tasks three times faster and used much less LLM processing compared to older methods.

Another key feature is "Android skills," which are small instruction sets for common tasks like navigation setup or moving from XML to Compose. These skills help AI agents follow Google’s latest advice, so they don’t use old or bad patterns. Developers can add, use, or share these skills easily, and they work together with the CLI. The "Android Knowledge Base" is a new resource that lets agents get the newest official documentation and tips, even if their training data is out of date. This helps agents give correct answers for new tools and patterns.

The article also says these updates make it easy to start a project with the CLI and later move it into Android Studio. Android Studio still offers the best advanced tools, like visual editing and deep debugging, and now has built-in AI agents to help with every step of app building. There is a new AI-powered project flow to help prototype ideas quickly, and these tools support all types of Android devices.

Others are more skeptical. A few worry that relying on AI agents and skills may hide what is really happening under the hood, making it harder for beginners to learn Android well. ## Guy builds AI driven hardware hacker arm from duct tape, old cam and CNC machine

- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=47800033)

A hacker made a DIY robot arm that uses AI to help with hardware hacking. He built it from cheap parts like duct tape, an old camera, and a CNC machine, then shared all the details on GitHub.

The project is called AutoProber. It scans electronic boards, finds interesting parts like pins and chips, and then probes them safely. The robot uses a microscope and a custom probe to look at the board and take pictures. It moves over the board, takes many small photos, and puts them together to make a big map. It marks pins and parts it finds, and shows them on a web dashboard so you can pick which ones to test. If you approve, the robot arm will carefully touch those pins and report what it finds.

AutoProber has strong safety steps. It always checks a special signal (on Channel 4 of an oscilloscope) to make sure it doesn’t move if something seems wrong. If there’s a problem, it stops right away and waits for the user. The system won’t continue until the problem is fixed. All control can be done from a web dashboard, Python scripts, or by the AI agent itself.

The project includes Python code, files to 3D-print custom parts, and clear setup instructions. It uses a cheap CNC machine, a USB microscope, an oscilloscope, and some other simple parts. The dashboard lets you control everything, see what the robot sees, and manage what it tests. The software is open source for noncommercial use, but you need to contact the author for business use.

AutoProber leaves out some things, like example data and private lab info, for safety and privacy. The creator warns the tool must only be used in labs where you have permission, and not on devices you do not own.

People on Hacker News were impressed by the project’s creativity. They liked that the builder used cheap and simple parts to make something useful. Some users were happy that the full code and setup are open and easy to follow. Others said that automatic probing is tricky and can damage things, so the strong safety features are important. A few people talked about their own experiences with CNC machines and how hard it is to get them working safely. Some wondered if AI could really help much or if most of the work is still manual. ## Qwen3.6-35B-A3B: Agentic coding power, now open to all

Qwen3.6-35B-A3B is a new open-source AI model for coding and reasoning, released by the Qwen team. The model is special because it uses a “mixture-of-experts” method: it has 35 billion total parameters, but only 3 billion are active at any time.

The article says this design lets the model work much faster and use less memory than older, bigger models, but still do very well on coding and reasoning tasks. Qwen3.6-35B-A3B beats earlier Qwen versions and even matches or passes other popular models like Gemma4-31B, especially in coding and multimodal (text and images together) tasks. The model is available for free to download or use through APIs and can be used with tools like Qwen Code, OpenClaw, and Claude Code. It has strong results in many tests, such as SWE-bench and Terminal-Bench for coding and MMLU and Mathvista for knowledge and reasoning. The model is also good at understanding images and videos, with high scores in vision-language tasks like RealWorldQA and RefCOCO. The article shares example code for using the API, explains how to set up the model with coding tools, and says the model can handle large context windows and preserve “thinking” steps for better agent work.

Qwen3.6-35B-A3B’s open weights mean anyone can self-host it or build new tools. The Qwen team plans to release more models in the future and wants feedback from the community.

In the Hacker News comments, many users are excited about the strong performance with so few active parameters. Some people say the “mixture-of-experts” idea is clever because it’s efficient—only a part of the model works at once, so it saves resources. Others ask if this model can really replace bigger commercial models for daily coding or agent tasks. A few users share their test results: some see very good code completions and logic, but others note the model is still slower than smaller models on weak hardware. There are questions about the “open” license—some want to know if it can be used in commercial products without limits. A few developers are happy that it works with popular open-source coding agents like OpenClaw and Qwen Code, saying this helps the open-source AI ecosystem. One person wonders how well the vision abilities work in real projects, not just benchmarks. Some are concerned about the high context window—does it really work smoothly with long inputs? Others say the documentation and setup are easier than with other models, and praise the team for sharing code examples. ## Cloudflare's AI Platform: an inference layer designed for agents

- 原文链接: [Cloudflare's AI Platform: an inference layer designed for agents](https://blog.cloudflare.com/ai-platform/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=47792538)

Cloudflare announced a new AI platform that lets developers use many different AI models from one place. The main idea is to make it easy to build smart agents that can switch between models and providers without extra work.

The article explains that AI moves fast, and the best model today may not be the best tomorrow. Real-world AI agents often need to use more than one model to do their job well. For example, a chatbot might use a fast model to sort messages, a bigger model to plan answers, and another model to do small tasks. If one provider is slow or goes offline, it can slow down or break the whole agent. That’s why developers need a way to use any model from any provider, check costs in one place, and keep everything fast and reliable.

Cloudflare’s new platform gives developers a single API to call models from over 12 providers, like OpenAI, Google, and Alibaba Cloud. Developers can switch models by changing just one line of code. They can also see all their AI spending in one dashboard, even if they use models from many companies. The API lets developers add custom tags to track costs for different users or workflows.

The platform will soon let users bring their own AI models by packaging them with a tool called Cog. This makes it simple to run custom models on Cloudflare’s network without worrying about complex setup. The system is designed to give fast responses by starting to send data as soon as possible, not waiting for the whole answer. Cloudflare has many data centers around the world, so responses can be very quick no matter where users are.

If a model provider goes down, Cloudflare’s system can automatically switch to another provider, so the agent keeps working. Streaming responses are saved, so if a connection drops, the agent can continue without starting again or paying extra. The company also shared that Replicate, a popular AI hosting service, has joined Cloudflare, and their models will be available on this new platform.

In the top Hacker News comments, some users liked the idea of having one API for many models. They said it could save time and make it easier to try new models as they come out. Some were happy that developers can track costs from different providers in one place, which is often hard today. Others liked the idea of bringing their own models, saying it will help with privacy and custom needs.

A few users asked about how Cloudflare handles model updates or sudden changes in cost from providers. Some worried that using one API for everything could cause lock-in, making it hard to move away from Cloudflare later. Others wondered about speed and if Cloudflare’s extra layer would slow things down, but some pointed out that having data centers everywhere could actually make things faster.

One commenter questioned how Cloudflare deals with provider outages and if failover is as seamless as promised. There were also concerns about pricing and what happens if a provider raises costs suddenly. Some developers were excited to try the new features, while others said they would wait to see real-world performance. Overall, the comments showed both excitement and careful questions about how the platform works and what it means for building smart, reliable AI agents.
