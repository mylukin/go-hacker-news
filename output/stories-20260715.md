## Inkling: Our Open-Weights Model

- 原文链接: [Inkling: Our Open-Weights Model](https://thinkingmachines.ai/news/introducing-inkling/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48924912)

Inkling is a new open-weights AI model released by Thinking Machines, designed so that anyone can use, study, and fine-tune it for their own needs. The model is large and can work with text, images, and audio, making it flexible for many tasks.

Inkling has 975 billion total parameters (41 billion active at a time) and can process very long inputs, up to 1 million tokens. It was trained on a huge amount of data, including text, pictures, sound, and video—45 trillion tokens in total. There is also a smaller version, Inkling-Small, with 12 billion active parameters, which runs faster and costs less.

The model is built to be a generalist: it can reason, code, follow instructions, and deal with many types of information. For coding, it can build web apps and even refine projects over many steps, like making a multiplayer snake game after lots of feedback. Inkling is not always the top performer on every test, but it balances power with efficiency and is easy to customize.

A key feature is "controllable thinking effort," meaning you can choose how much time and resources the model spends to answer a question—useful if you want faster or cheaper results. Inkling does well on both audio and visual tasks, like understanding speech and analyzing images or charts. It can say when it is unsure about an answer, which helps with trust and safety.

Safety is a focus: the model is trained to refuse dangerous requests better than other open models, but not block harmless ones. Benchmarks show it is strong in safety, reasoning, and coding, though some closed (not open) models still score higher on certain tests.

Inkling was trained with modern techniques, using a Mixture-of-Experts Transformer and lots of reinforcement learning to improve its answers over time. They noticed that, as the model learned, its explanations became more compact and direct.

Inkling-Small, the lighter version, performs close to the big model in many tasks, making it a good choice for cheaper or faster jobs. Both models are available to try and fine-tune on the Tinker platform, and can be used through APIs and popular tools like Hugging Face.

Now, turning to the top Hacker News comments: Some users are excited to see a truly open, large, and multimodal model, saying this helps independent developers and researchers. People appreciate the detailed benchmarks and openness about strengths and weaknesses. Several point out that Inkling’s focus on customization and fine-tuning is useful for real-world tasks, not just for academic benchmarks.

Others are cautious about the practical requirements—running such a big model needs a lot of hardware, which may be out of reach for smaller teams. Some ask how it compares to other open models, like Llama or Mistral, and wish for more open evaluation tools. There is interest in the model’s safety measures, but a few worry that updates or fine-tuning could reduce safety if not managed well.

A few discuss the licensing and true openness, asking if there are any hidden limits. Others wonder if the model can be really "generalist" or if it will still need expert tuning for specific jobs. Finally, some are hopeful that this release will push other groups to be more open and transparent with their own models.

## Running Gemma 4 26B at 5 tokens/sec on a 13-year-old Xeon with no GPU

- 原文链接: [Running Gemma 4 26B at 5 tokens/sec on a 13-year-old Xeon with no GPU](https://www.neomindlabs.com/2026/06/08/running-gemma-4-26b-at-5-tokens-sec-on-a-13-year-old-xeon-with-no-gpu/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48922434)

This article shows how someone ran Google’s big Gemma 4 language model on a very old 2013 Xeon server with no GPU, getting about five tokens per second. The server was never meant for AI, but with some clever work, it could run a modern 26-billion-parameter model.

The author used a dual Xeon E5-2690 v2 box with only DDR3 RAM and no GPU, costing less than $300. This old chip only supports AVX1, not newer instructions like AVX2 or FMA3, which many AI tools need. Inspired by a recent post about running Gemma 4 on a newer Xeon, the author wanted to try it on even older hardware. At first, the code failed to run because it expected AVX2 instructions. The author asked Claude AI for help, which found missing code paths and suggested fixes. The main issue was that some code parts only worked with AVX2, so fallback code had to be written for AVX1. Claude helped rewrite these “hot paths” so the old chip could run them, even if a bit slower.

Once fixed, the model could generate text at about five tokens per second, which is about as fast as reading. The author posted their patch and suggested others could use it to get old servers working with big models, especially when cloud APIs are too expensive or offline. The technical fix involved editing C++ code so the software would not call missing AVX2 code, and instead use simpler, slower code that still works with old CPUs. There was also a bug where the model produced nonsense text, which turned out to be because some memory was never filled in unless the right code path was present.

In the end, the author encourages people to “open the hood” and make things work, rather than just paying for cloud AI services. They share the steps and patches for others with old hardware.

In the Hacker News comments, many readers were impressed by the technical achievement and the value of reusing old hardware. Some said this is a good reminder that not all AI work needs the newest GPUs or cloud services. Others noted that running big models on old CPUs is not very practical for most users, as it is still slow and uses a lot of power. Some thought the project was only possible because the author had strong debugging and problem-solving skills. There were questions about how much electricity the old server uses compared to newer gear, and whether it is really cheaper in the long run. A few people talked about the joy of hacking on old machines, and how this keeps tech fun. Others asked if it makes sense to maintain code for pre-AVX2 CPUs, or if it is better to move on. One reader pointed out that open-source tools and AI models make these hacks possible, while closed systems would not allow it. Some shared their own stories of running AI models on old laptops or desktops just to see what is possible. Finally, some warned that this is a cool experiment, but for real work, renting a modern GPU or using cloud APIs is still much faster and easier.

## Speculative Growth and the AI "Bubble" [pdf]

- 原文链接: [Speculative Growth and the AI "Bubble" [pdf]](https://economics.mit.edu/sites/default/files/2026-07/speculative_growth_AI_public.pdf)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48927409)

This article looks at why AI companies are valued so highly and if this means there is an AI “bubble.” The author says there is a third option: even if prices later fall, the investments made during the boom can still leave a lasting, real impact on the economy.

The paper explains that when investors believe AI will bring big returns, they put lots of money into AI. This raises company values and leads to more investment in things like data centers and computers. As more AI capital is built, it increases what companies can produce and moves more income toward people who own the capital (not workers). These capital owners save more, which then lowers the long-term interest rate in the economy.

There can be two types of stable economies: one with low capital and little AI, and one with high capital and lots of AI. Normally, the economy stays in the low state, because investors are careful and don’t want to risk too much. To move to the high-AI state, there needs to be a period where people are extra optimistic—maybe even too optimistic. This “bubble” phase pushes investment higher, and if enough AI capital is built before people realize their optimism was too much, the economy can stay at the higher level even after the bubble bursts.

If the correction (when people become less optimistic) comes too soon, the economy falls back to the low state. But if enough capital is built, the gains remain: workers get higher wages, even if their share of the total pie is smaller, and capital owners end up with more wealth. The paper uses math models to show how this works, and it discusses that these ideas match what is happening now with huge investments in AI.

In the Hacker News comments, some people agree with this view. They say history shows that “bubbles” like the dot-com boom left real things behind, such as internet infrastructure, even if many companies failed. Others warn that not every bubble leaves something useful; sometimes money is wasted on things that do not help in the long run.

A few users point out that AI might be different from past tech booms because AI tools can replace jobs. They worry about the impact on workers and if higher wages will really happen. Some think the paper is too optimistic about the benefits for workers and too focused on capital owners.

There are debates about whether low interest rates are a good sign or a warning. Some say it means the economy is healthy and has lots of savings; others say it can signal trouble if returns on investment are too low.

Many agree that the paper gives a helpful way to think about AI investment, not just as a bubble or not, but as something that could shape the future even if prices fall. But some feel the models are too simple and real life is messier. Overall, readers like the idea that even failed hype can build useful things, but they remind everyone to be careful about assuming all bubbles are good.

## We don't use AI in any of our design or production processes

- 原文链接: [We don't use AI in any of our design or production processes](https://mass-driver.com/article/from-human-hands)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48927373)

This article is about a type design company that chooses not to use AI for any of their design or production work. The writer explains why keeping the human touch in their craft is important, especially in making typefaces.

They start by talking about the history of writing, showing how every letter we use has changed over thousands of years, shaped by many people. For example, the letter "A" comes from an old drawing of an ox’s head, slowly changing shape as it passed through different cultures. Even small details in modern letters, like the thick and thin parts or the serifs (the tiny feet on letters), were shaped by how people held old brushes and chisels. These choices were made because of human hands, habits, and even discomfort.

The writer says that when we use AI for design, we remove this human story and effort. They point out that AI, like Midjourney or ChatGPT, cannot feel things or invent new ways to write, because it has no body or feelings. AI can only mix old ideas from its training data. The writer worries that if we let AI take over, design will stop changing and will only repeat what already exists. They also say that AI’s training data misses many small languages and cultures, so their styles may be lost or ignored.

The author believes that the world needs new typefaces, just like it always needs new music and books. Some languages have very few typefaces, and AI cannot help them if there is not enough data. If AI becomes too popular in design, fewer people will become type designers, and we may lose new ideas forever. The writer works in type design not for money or fame, but to help keep this human tradition alive.

Many comments on Hacker News agreed with the author, saying that human creativity is special and cannot be copied by machines. Some said they also prefer handmade work, even if it takes longer or costs more. Others pointed out that AI is just a tool, and if used carefully, it can help speed up boring parts of the job, letting people focus on creative ideas. A few people worried that companies that refuse AI might not survive if others use AI to make things faster and cheaper. Some commenters asked if it’s possible to mix both AI and human skills, so we get the best of both worlds.

There was debate about whether AI will really stop new ideas, or if it can actually help people invent new things by giving them more options. Some people felt sad about losing old traditions, while others were excited about what AI might bring. A few designers shared their own stories about how tough it is to make a living doing creative work by hand, but said it is worth it. Some said that if everyone uses AI, special handmade fonts will become even more valuable. Others worried that AI could make everything look the same, and we would lose the unique touches made by real people. Overall, the comments showed strong feelings on both sides, with many people wanting to protect human creativity, but also curious about how AI might change design in the future.

## Who Is America's Homer?

- 原文链接: [Who Is America's Homer?](https://www.plough.com/articles/who-is-americas-homer)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48913656)

This article asks if America has its own “Homer”—a poet or writer whose work speaks for the whole nation, like Homer did for ancient Greece. The piece gathers opinions from many writers and critics, each offering a different answer and explaining why.

Some say Walt Whitman is America’s Homer. Whitman wrote about the American spirit, people, and freedom. He believed everyone is as important as gods, and his poems celebrate the individual and the country’s energy. Others choose Robert Frost, who wrote poems that are simple and deep, and touch on all parts of American life—comic, tragic, romantic, and even satirical. Frost’s poems are loved by both everyday people and experts. Herman Melville, the author of “Moby-Dick,” is picked for telling stories of adventure and struggle in America, especially through the eyes of sailors and workers.

Emily Wilson suggests the real American Homer might be a songwriter, not a poet or novelist. She says Tracy Chapman’s song “Fast Car” is like a modern epic, telling a big, emotional story in a short time, and touching on hope, struggle, and longing to belong. Another view is that Laura Ingalls Wilder’s “Little House” books are America’s epic, with their stories of journey, survival, and family on the frontier, using simple, repeated language like old oral stories.

Some pick William Carlos Williams, whose poems capture real American life, from baseball games to city living, using clear and direct words. Hart Crane is also suggested, especially for his long poem “The Bridge,” which tries to unite America’s past and present with bold, beautiful language.

A few writers argue that America does not have a true Homer yet. They say the country is too young and too mixed, with many cultures and ideas, to have one epic or one poet that speaks for everyone. National epics, they say, usually come after a country has gone through big changes or hard times, and maybe America’s will be written in the future. Others note that American poets often focus on themselves and their own lives, not on the whole nation.

In the comment section, many people agree that Whitman and Frost are good choices because their works are well-known and feel “American.” Some think Melville’s “Moby-Dick” is too hard for most readers, but others say its themes fit the country’s spirit. Several commenters like the idea of music being epic—Chapman’s “Fast Car” and even songs by Bob Dylan or Woody Guthrie are mentioned because they capture life’s struggles and dreams.

A few readers argue that America’s diversity makes it impossible to have just one “Homer.” Some say the country’s real “epic” is still being written, and maybe one day a new voice will unite everyone. Others remember poems and songs from their childhood, showing how personal and local American stories can feel. A handful think the search for a national epic is not so important, since America’s strength is its many voices and stories. Some worry that modern poetry is too academic, so it can’t reach everyone like Homer did in Greece.

Overall, the comments show that people have many favorite writers and songs that feel epic to them, but there is no clear answer to the question. For now, America’s “Homer” may be many people, or no one at all.
