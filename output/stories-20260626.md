## A C++ implementation of a fast hash map and hash set using hopscotch hashing

- 原文链接: [A C++ implementation of a fast hash map and hash set using hopscotch hashing](https://github.com/Tessil/hopscotch-map)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48692090)

This article is about a C++ library called hopscotch-map, which gives you a fast hash map and hash set using a method called hopscotch hashing. The main idea is that hopscotch-map is faster and more memory-efficient than the standard C++ unordered_map, and it also gives you extra features.

Hopscotch hashing is a way to handle hash collisions, making the structure very cache-friendly and quick for most uses. The library has different versions: the regular hopscotch_map/set for speed, and a “pg” (prime growth) version that helps if your hash function is not so good. There are also “bhopscotch” versions that are more resistant against attacks that try to slow down your hash table by creating many collisions.

Some key features include: it is header-only (so easy to add to your project), supports move-only types, allows you to look up items using different but similar key types, and can store hash values for faster searches. The API is close to std::unordered_map, but there are some differences, such as how iterators and references become invalid if you insert or change the table. You also change values using it.value() instead of the normal way.

For growth, it offers several policies: power-of-two (fast, but can cause more collisions if your hash is not good), prime (a bit slower but safer against bad hashes), and a customizable one. If you get many collisions, you should consider changing the hash function or policy.

To install, you only need to add the include folder to your project. If you use CMake, you can link to the target easily. There are code examples showing how to use the map and set, including advanced features like heterogeneous lookups, where you can find items using a different type than the key. There are also secure versions to defend against Denial of Service (DoS) attacks on the hash table, which use a binary search tree to keep lookups fast even in the worst case.

The library works with any C++17 compiler and is open-source under the MIT license.

In the Hacker News comments, many users liked the library for its speed and clear documentation. Some people talked about their own experiences using hopscotch_map, saying it gave them big performance boosts compared to std::unordered_map, especially when storing lots of items. Others pointed out that the API differences (like how you have to use it.value() instead of ->second) can be surprising at first but are manageable.

Some users worried about the complexity of modern hash tables, saying that debugging can get harder with more advanced data structures. Others discussed hash security, noting that the secure versions are helpful if you might face DoS attacks, but not everyone needs them. A few commenters debated if the library should be used as a default, or if std::unordered_map is still fine for simple use cases.

There were also questions about memory usage, with some users happy about the low memory footprint, while others wanted more benchmarks and real-world comparisons. People liked that the library is header-only and easy to add to projects. A few users brought up problems with older compilers, but most found it worked well with modern C++ standards.

Overall, the comments showed strong interest in better hash maps for C++, with many developers glad to have more choices and happy about the open-source nature of the project.

## MicroVMs: Run isolated sandboxes with full lifecycle control

- 原文链接: [MicroVMs: Run isolated sandboxes with full lifecycle control](https://aws.amazon.com/blogs/aws/run-isolated-sandboxes-with-full-lifecycle-control-aws-lambda-introduces-microvms/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48642510)

AWS just launched Lambda MicroVMs, a new way to run code in small, secure virtual machines (VMs) with easy control over their start, stop, and pause. This is for developers who need to run code from users or AI in safe, separate environments.

The main problem before was that VMs were very secure but slow to start, and containers were fast but not as isolated. Lambda functions are fast for small jobs, but not good for long or interactive work that needs to keep its state. Lambda MicroVMs solve this by using Firecracker, a lightweight VM tech, to give each user or session their own fast, secure, and stateful environment. You don’t need to manage servers or have deep knowledge of virtualization.

To use Lambda MicroVMs, you package your app (for example, a Flask app), its Dockerfile, and upload to Amazon S3. You then create a MicroVM image using AWS tools or the console. When you launch a MicroVM, it starts from a saved snapshot, so it’s ready almost instantly. There’s no need to set up networking; AWS gives your app a unique endpoint right away. When the MicroVM is idle, it can suspend, saving memory and disk state. When you send a new request, it resumes where it left off, fast enough that users don’t notice the pause. Each MicroVM is completely isolated, so untrusted code can’t affect others. You can use up to 16 CPUs, 32 GB memory, and 32 GB disk per MicroVM, and you pay less when VMs are idle. Lambda MicroVMs are available now in several AWS regions.

In the comments, some people are excited because this makes it much easier to run untrusted code safely, like in coding apps, AI tools, or games. They like that you don’t need to be an expert in VMs or containers to use it. Others see this as a big step for serverless computing, making it useful for more types of apps.

However, some are worried about cost and vendor lock-in, since it’s tightly linked to AWS services. A few ask how well this will work for very large, heavy apps, and if the performance is as good as promised. Some users wonder about debugging and monitoring—will it be easy to see what’s happening inside each MicroVM? Others compare it to Google Cloud’s similar offerings and discuss which is better. There’s also talk about how this could be used for testing, security research, or running code in schools. A few commenters mention privacy and security, asking if AWS could access data inside the MicroVMs. Overall, the community finds it a promising tool but wants to see real-world results and pricing details before using it for everything.

## Ultrasound imaging of the brain

- 原文链接: [Ultrasound imaging of the brain](https://alephneuro.com/blog/ultrasound-brain)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48685558)

This article talks about a new way to take pictures of the human brain using ultrasound, without needing surgery or big machines like MRI. The team has made the first detailed 3D images of the blood vessels inside a living human brain, all through the skull.

Usually, getting good brain images is hard. You either need to drill into the skull and put in electrodes, or use tools like EEG that only show blurry pictures. MRI scans give clear pictures, but the machines are big and not wearable. The new method uses ultrasound and small bubbles, called microbubbles, injected into the blood. These bubbles help the ultrasound "see" tiny details, much smaller than usual. When neurons in the brain are active, blood rushes to them, and the ultrasound can map this by tracking the moving bubbles.

With this technique, they can see big and small blood vessels, and get images that are much clearer than CT scans. The team has shared their data and the software they use as open source, so others can try it too. This could help doctors find signs of problems like strokes or Alzheimer’s that current scans might miss.

The process works because the microbubbles reflect sound much better than blood cells. By tracking each bubble very carefully, the system builds a very sharp image. The bubbles are safe and already approved by the FDA. The team wants to go further and do this without using bubbles, just by reading signals from regular blood cells. They believe better hardware and new machine learning methods will help.

Now, let’s look at what people on Hacker News said about this. Many are excited and think this is a big step for brain research and health. Some say it could help in hospitals, and maybe even let us "read" thoughts one day. Others are careful, saying that even with better images, understanding brain activity is still very hard. A few point out that MRI and CT have their own uses, and ultrasound may not replace them for everything.

Some users talked about safety, asking if using ultrasound on the brain for a long time is safe. There are also questions about how well this works on people with thick or damaged skulls. Others are happy the data and code are open, saying this helps science move faster. There are also comments about possible future uses, like studying sleep, memory, or even building brain-computer interfaces.

In short, the community thinks this new method is promising. People are hopeful but also realistic about how much work is left. Most agree it’s a great step forward, and they’re curious to see what happens next.

## What Is a Nomogram and Why Would It Interest Me?

- 原文链接: [What Is a Nomogram and Why Would It Interest Me?](https://lefakkomies.github.io/pynomo-doc/introduction/introduction.html#what-is-a-nomogram-and-why-would-it-interest-me)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48689277)

A nomogram is a special kind of chart that helps you solve math problems quickly without a calculator. You line up a ruler or straightedge across the chart, and it shows you the answer for a formula with several variables.

The article explains that nomograms were invented in 1880 and were important before electronic calculators. Engineers and scientists used them for fast calculations in fields like railway building, water flow, medicine, ballistics, machine work, statistics, chemistry, aeronautics, and even military work. The nice thing about nomograms is that they can solve very complex equations just by drawing lines, even if you can’t rearrange the formula. For example, you can use one to solve the math for how parts move in a machine, just by matching the numbers on the scales. They are also cheap—just a piece of paper can do the job.

Nomograms are made of scales—lines with numbers for each variable—and sometimes include sample lines to show how to use them. Good nomograms should be clear so anyone can use them, and should show what formula they solve. The article introduces a tool called PyNomo, which lets you design your own nomograms on your computer, even for complicated equations. All you need is a basic understanding of algebra. You choose the type of formula you have, copy a sample script, change it for your formula, and PyNomo makes a PDF you can print. The tool also lets you change how the chart looks, like colors or labels, but those details are optional.

In the comments, some people say nomograms are a forgotten but clever tool, and they share stories about using them in engineering or medicine. Others are surprised they aren’t taught more often, since they give insight into how formulas work. A few remember seeing nomograms in old textbooks or on aircraft, and think they are beautiful and useful. Some users point out that in places without computers, nomograms still help people do important calculations. Others mention that learning to make nomograms helps you understand math better. There are also technical discussions about how to design nomograms, and some suggest PyNomo makes this much easier. One person notes that modern apps or spreadsheets do similar things, but nomograms are faster for certain tasks. Finally, some say nomograms should make a comeback because they are fun, visual, and practical for teaching or field work.
