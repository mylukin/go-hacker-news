Our first story is about a small plane crash in Gloucestershire, caused by a failed 3D-printed plastic part. The accident report says the part was an air induction elbow, made of plastic, and it collapsed after it got too hot from the engine. This part was bought at an air show in North America and installed in a Cozy Mk IV light aircraft. On March 18, during landing, the pilot tried to add power, but the engine lost power because the 3D-printed part had softened and collapsed. The plane hit a landing aid system, but luckily, the only person on board had only minor injuries.

The Air Accidents Investigation Branch found that the part was made from the wrong kind of material. After the crash, the Light Aircraft Association sent out a safety alert about using 3D-printed parts in planes. The report explained that the part was installed as a modification to the fuel system, but was not able to handle the engine’s heat.

The main point is that 3D-printed parts can melt or fail in high temperatures if the wrong plastic is used, which is dangerous in planes. Safety groups plan to warn other pilots and inspectors about these risks.

In the comments, many users say this crash shows why choosing the right material is critical for aircraft parts. Some point out that 3D printing is not always bad, but you have to use special heat-resistant plastics or metals. Others believe 3D-printed parts should never be used in important systems on planes unless tested very well. Some users say this kind of risk is more common in homemade or experimental aircraft, where rules can be less strict. A few people defend 3D printing, saying it is safe if used correctly and with proper checks. Others share stories about similar problems in cars and machines, showing this is not just an aviation issue. Many commenters hope this accident will lead to better safety rules and more education for pilots who want to use 3D-printed parts.

The next story is about CUDA-L2, a new system that uses AI and reinforcement learning to make GPU matrix multiplication faster than NVIDIA’s own cuBLAS library. CUDA-L2 focuses on half-precision matrix multiplication, or HGEMM, for the A100 GPU, and claims to beat not only torch.matmul but also NVIDIA’s best closed-source libraries.

The main idea is to let large language models and reinforcement learning try many ways to write CUDA kernels for matrix multiply, then pick the fastest ones. CUDA-L2 tested its code on 1,000 different matrix sizes and showed clear speedup over cuBLAS in benchmarks. The code is open source and includes tools to test and compare these new kernels. The project uses NVIDIA’s CUTLASS library, and you need to set some environment variables to get it running. You can run tests offline or in server mode, and you can pick problem sizes or GPU IDs. If your matrix size is not in the list, you can pad your matrix or ask the authors to add support.

Currently, CUDA-L2 only guarantees speedup on the A100 GPU, but the team plans to support more GPUs soon. The setup needs Python and PyTorch 2.6.0 or newer. For questions or requests, you can open a GitHub issue or email the authors.

In the comments, some users are excited about beating cuBLAS, which is known to be very fast and hard to improve on. They wonder if the speedup is real for all matrix sizes or just some special cases. Others ask if this idea can be used for other operations besides matrix multiply, or if it will work well on other GPUs. Some people say cuBLAS is so well-tuned that any speedup should be checked for correctness, not just speed. A few mention that reinforcement learning for code optimization is a hot topic now, and this project is a good example. Some appreciate the clear setup instructions, while others hope for more open benchmarks and tests. Overall, people like the progress but want to see more data before trusting it fully.

Our third story is about Multivox, a project for making your own volumetric display using spinning LED panels and a Raspberry Pi. The code and hardware plans are open source. You can build two devices: Rotovox, which has better vertical resolution, and Vortex, which is brighter and refreshes faster.

The display works by spinning two LED panels around a vertical axis, with a photodiode helping the software know the position as it spins. A Raspberry Pi 4 runs the system, and you use a Bluetooth gamepad to control demos and games. Audio can go over Bluetooth, but only at basic quality.

The software has two main parts: a driver that keeps track of the 3D pixels and updates the panels as they spin, and client code that creates the 3D content. You can run everything on the Pi inside the device, or send 3D data from a PC over WiFi. There are demo games and visualizations, like a tesseract, fireworks, and a multiplayer light cycles game.

If you don’t have the hardware, there is a simulator that shows the 3D effects on your computer using OpenGL. The project uses CMake to build, and you can set it all up to start automatically when you turn on the Pi.

In the comments, many readers were impressed by the engineering and creativity. Some loved the open source nature and the fact that all the plans and code are public. Others pointed out how hard it is to keep everything synced as the panels spin so fast. A few wondered about the safety of spinning parts at high speeds. Some liked using a Raspberry Pi for real-time tasks, but others worried about its limits. There was discussion about how Bluetooth audio quality is low and suggestions to use a wired connection. Many were excited about the simulator, since most people can’t build the hardware but still want to try the 3D effects. Some suggested improvements for the software or more controller support. Overall, people were impressed and happy to see such a detailed open source project.

Next, we have an article comparing two styles of leadership: servant leadership and what the author calls “transparent leadership.” The author managed a team and tried servant leadership, but found problems with it. Servant leadership is like “curling parenting,” where the leader tries to clear away all problems for the team. At first, this feels nice, but it can make the leader tired and overworked. If the leader leaves, the team may not know how to solve problems on their own. Sometimes, the team becomes disconnected from the rest of the company and does not understand their purpose.

The author suggests transparent leadership as a better way. A transparent leader coaches team members, teaches problem-solving, explains the company’s values, and helps people make their own decisions. Transparent leaders make sure everyone knows what is happening and why, and let team members grow by giving them more responsibility. They also train others to take their place, making themselves less important over time. If a leader becomes “useless” because the team works well without them, that is not a bad thing. The mistake is when leaders create extra work or rules just to stay important. Instead, a good leader should return to hands-on work, like coding or technical tasks, which keeps their skills sharp and earns respect.

In the comments, many people agreed that servant leadership can make teams too dependent on the leader and fall apart when the leader leaves. Others liked the idea of transparent leadership and said it helps teams learn and grow stronger. Some argued that servant leadership can still work if the leader also teaches and does not just solve problems for the team. There were stories about good managers who made themselves “redundant” and moved on to new teams or projects. Overall, most commenters liked the idea of transparency, teaching, and letting teams grow.

Our next story is about Stanford University, where 38 percent of students are now officially disabled—a much higher number than most people expect for such a top school. Most of these students are claiming mental health or learning disabilities, like anxiety, depression, or ADHD, and many are getting special help for schoolwork, such as more time on tests.

The article argues that this rise in reported disabilities is strange. Elite schools like Stanford only admit students who do very well in school and on tests, so it seems unlikely that so many have serious learning problems. At community colleges, only three or four percent get accommodations. One reason for the jump may be that it’s now easy to get extra help—students just need a doctor’s note. The culture around mental health and learning disabilities has also changed, and social media spreads broad ideas about what counts as a disability.

Because of this, many students may think normal struggles are signs of a disorder. The definition for things like ADHD has become much wider, making it easier to get a diagnosis. The article suggests that being afraid of failure pushes students to look for reasons why they are struggling, instead of seeing stress as a normal part of college. The writer warns that getting extra help when you don’t truly need it might hurt students in the long run, because they miss chances to learn to handle tough situations.

In the comments, some readers agree that the number is too high and that the system is being abused. Others point out that mental health problems are rising in general, so maybe the high number is not so strange. Some think the focus on labels and accommodations can make students too dependent on extra help. Others argue that society is just better now at recognizing and supporting people with invisible disabilities. A few say that elite schools are very stressful, which could cause more students to struggle. There are also calls for better mental health support, and some readers share personal stories about struggling with school, showing this is a complex and personal issue.

Our next story is about the “JavaScript” trademark, which is still owned by Oracle, even though almost everyone uses the name freely. The author argues that the trademark is abandoned and should be released to the public.

JavaScript got its name from a deal between Netscape and Sun Microsystems in the 1990s. When Oracle bought Sun in 2009, they got the JavaScript trademark. Oracle keeps renewing the trademark, but the author says they do not use it for any real product, and most people at Oracle may not even know they own it.

The law says a trademark can be lost if it is not used in business or if it becomes a generic term. The article claims Oracle has no main product called JavaScript, and the name is now used by millions as a generic term. This causes trouble for the developer community. The standards group that defines JavaScript uses the name ECMAScript instead because of the trademark problem, and conferences use names like “JSConf” to avoid legal risks.

In the comments, many people agree the trademark is outdated and that Oracle is not using it in a meaningful way. Some say “JavaScript” is now like “email” or “internet”—no one company should own it. Others point out that the legal process to remove a trademark can be slow and expensive, but they support the idea. Some recall the history of the name and wish it had been ECMAScript from the start. Some worry about what would happen if Oracle tried to enforce the trademark, while others say Oracle probably does not care. A few are not sure if the legal case is strong, but agree the name should belong to everyone. Overall, most people support freeing the name “JavaScript” and want it to become truly open.

Our next story is about Hammersmith Bridge in London, which has been closed to cars for six years after engineers found it was unsafe. The story compares how fast big repairs happened in the past to how slowly things move now.

The bridge once carried 25,000 vehicles each day. When it closed, people thought other roads would get too busy and local shops would lose money. But after six years, traffic did not get worse on nearby bridges; it actually got lighter. Many people switched to walking, cycling, or public transport. Local spending increased in some areas, and air pollution dropped. Some shops that needed car customers lost business, but small local shops did well.

Fixing the bridge to its old state is now estimated to cost 250 million pounds and could take 10 years. Much of the cost comes from safety rules, heritage laws, and the need for many approvals. There have been ideas for a new bridge, a toll road, or even letting a Chinese company rebuild it quickly, but nothing moved forward. One new idea was to use lightweight electric pods to move people across the bridge for a much lower cost, but local leaders refused to try this.

In the comments, some people agree this is a good example of how cities can adapt when cars are removed. Others share that similar things happened in their towns. Some point out that the drop in traffic is also due to COVID and changes in work habits. A few worry about people who lost out, like older or disabled residents, and ask for solutions that help everyone. There is debate about the high costs of UK infrastructure, with some blaming red tape and others saying safety is worth the price. Many like the pod idea, while others are skeptical. Most agree the bigger problem is government gridlock, not engineering or money.

Now, let’s talk about Django 6.0, the latest release of the popular Python web framework. This version only works with Python 3.12 and newer and drops support for some older versions and MariaDB 10.5.

New features include built-in Content Security Policy support to help stop cross-site scripting attacks, and template partials for reusing pieces of templates more easily. Django now has a simple background tasks system using a new @task decorator, though you still need an external worker to process jobs. Django also moved to Python’s modern email API, making email handling easier.

Other improvements include new admin icons, stronger password hashing, and new tools for working with GIS, Postgres, and static files. Async paginator classes have been added, and there are more ways to customize migrations, models, and templates. The default auto-increment field for models is now BigAutoField. Some details have changed, like how querystrings are built in templates and how test runners work.

Breaking changes mean you need to update any third-party database backends or custom email code, since some APIs and method names changed. Django dropped support for some old features and settings, so you may need to update your code.

In the comments, many people like the new CSP support and are happy it’s now standard. Others are excited about template partials, saying it will help make templates easier to manage. Some wish the background tasks system was more like Celery, but others are glad for a lightweight option. There’s some frustration about dropped support for older Pythons and MariaDB, but most users understand it helps Django move forward. Some complain about deprecating positional mail arguments, while others say it’s better for clarity and safety. A few worry about third-party library compatibility but appreciate the clear upgrade guide. Overall, the community feels Django 6.0 is a solid upgrade with helpful new features.

Our next story is about PyTogether, a web app where people can write Python code together in real time, a bit like Google Docs but for coding. PyTogether is made for teachers, students, and anyone who wants to learn or teach Python easily. You do not need to install anything—just open your browser, make a group, and start coding with others. It has features like real-time editing, live drawing, chat, and even voice calls. The app saves your code every minute and when you leave, so you do not lose your work. The code is checked for errors as you type, making it easier to learn. Teachers can draw on the screen to explain ideas, which helps in online classes.

On the technical side, the backend uses Django and PostgreSQL, and the frontend uses React and CodeMirror. Pyodide is used to run Python code in your browser. It is open source and easy to run locally with Docker and Node.

In the comments, some people like how simple PyTogether is, saying it is great for beginners and teachers. They think the live drawing and voice chat make online lessons better. Some compare it to tools like Replit or Google Colab, but say PyTogether is easier for new users. A few wish it had more features for advanced programming, but agree that the focus on learning is smart. Some ask about privacy and how safe student data is. Others want to know how well it works with many users at once or if it could be used for coding interviews. Some mention small problems when running it locally, but like that setup is quick. Overall, the response is positive, with most people happy to see another tool that helps people learn Python together.

Our last story today is about how elites can use AI to change what the public thinks, because AI makes it cheaper and easier to persuade large groups. The author builds a model to show how these elites might act when they want to shape public opinion for their own goals.

In the past, elites could only use things like schools or newspapers to influence people, which was slow and expensive. Now, AI can make messages that are very targeted and convincing at a much lower cost. This means elites can shape what people believe much more quickly and directly. The model in the article shows that if only one group is in power, they have a strong reason to push society to be more divided, because it helps them keep control. Better AI tools make this polarization happen even faster.

If there are two groups that take turns in power, things get more complex. Sometimes both groups try to make society stay in the middle, because it is harder for the other side to undo their changes. The main point is that AI-driven persuasion changes polarization from being just a side effect of politics to something that can be used as a tool by those in charge. This could have big effects on how stable democracy is as AI gets better.

In the comments, some are worried that AI could make it too easy for people in power to control everyone’s beliefs. Others point out that people have always tried to persuade and control, and AI is just a new tool. A few say that people are not so easy to fool, and that strong opinions often come from real problems, not just clever messaging. Some think we need new laws to stop abuse of AI in politics, while others believe that open AI tools could help the public fight back. There is also talk about how social media already made persuasion easier, but AI could take it to a new level. Many agree this is an important problem, but they do not all agree on how dangerous it will be or what to do about it.
