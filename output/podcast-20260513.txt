Our first story is about gaming on Linux and why it’s getting faster, sometimes even beating Windows. The main reason is that Linux is now getting special gaming features that used to be only on Windows. For example, Linux now has IO_uring, which lets games read and write data very quickly. There are also tools that work like Windows’ DirectStorage, helping games load much faster. Valve and their Steam Deck are a big reason for this change. They created Proton, a tool for running Windows games on Linux, and now more developers are supporting Linux gaming. Some people say that certain games even run better on Linux, because the Linux kernel can use these new features well.

In the comments, many are excited about how easy and fast Linux gaming is getting. They like seeing more games work on Linux, and tools like Proton make it simple. However, some are careful—there are still problems with drivers, and some games do not work right. A few think Windows might improve soon and get back its advantage, since hardware makers still focus on Windows first. But most agree that better tools and more open-source support are good for everyone, and that competition between Windows and Linux helps all gamers.

The next article explains how people in the US can get a free domain name that ends in .city.state.us. These domains are special because they show your link to a city or state, and they have been around since 1992. To get one, you must live in the US or have a US company. First, you check if your city or state has a registrar. Then, you set up nameservers, maybe with Amazon Lightsail, and fill out a special form to show who you are and what you will use the domain for. After you send the form, you might need to wait a few days or weeks. If you are approved, you can point your new domain to your website. Many people are surprised that these domains are still free and not well known. Some shared stories about getting their own .us domains long ago and mentioned the process can be slow or confusing. Others worry about scams or fake government websites, but most like that these domains keep your personal info private. There are also calls to make the system easier and more modern.

Our next topic is Apple’s new MacBook Neo, a $599 laptop using the iPhone 16 Pro’s A18 Pro chip instead of Apple’s M-series chips. This laptop gives strong single-core speed, but has only 8GB of memory, which you can’t upgrade. The A18 Pro chip is great for short tasks, but if you do heavy work for a long time, it gets hot and slows down a lot, since it has no fan. Apple keeps the price low by using a cheaper chip and cutting features—there’s no backlit keyboard, only one fast USB-C port, and a basic webcam. Because of a global RAM shortage, Apple uses just 8GB to keep costs down. The Neo is good for students or anyone needing a simple, strong, and light laptop, but it’s not for heavy users. In the comments, some people like the quiet and energy saving, but others think the laptop is too limited for students who want to learn more about IT or creative work. Some would rather buy a Windows laptop with more RAM, but others say those often have worse screens or batteries. In the end, the right laptop really depends on your needs and budget.

Next, we have news from Princeton University. After 133 years, they will end their unproctored Honor Code exams and start proctoring all in-person exams from July 1. The reason is more cheating, especially with new tech like AI and phones. Before, students promised not to cheat, but now, teachers will be present in the exam room as witnesses. If they see cheating, they will report it, but they won’t stop students during the exam. Surveys show that almost 30% of seniors have cheated, and many students fear reporting others. The new rule is meant to make cheating harder and take pressure off students to police each other. In the comments, some people agree this change is needed—cheating is easier now and old systems don’t work. Others are sad to lose the tradition of trust. Some worry proctoring won’t fix everything, and that the real problem is too much pressure or bad exam design. There are mixed feelings, but most people see this as a hard but necessary change for today’s world.

Moving on to software tools, we have a story about Google’s IDE history. At first, Google let engineers use any editor or IDE, like Vim, Emacs, Eclipse, or IntelliJ. But this made it hard to support all the plugins and tools needed for Google’s huge codebase. Later, Google built a web-based editor called Cider, and then a newer version, Cider V, using VSCode as the frontend. This web IDE could handle millions of files, gave fast code search, and made it easier to add new features—like AI code completion. By 2023, most Google engineers were using Cider V, and teams could share tools and extensions more easily. In the comments, some people say it’s hard to make everyone use the same IDE, but Google’s scale makes it necessary. Others like the cloud IDE idea, while some worry about privacy or performance. Many agree that no IDE will make everyone happy, but good features and strong integration can help. Some wonder if smaller companies could do the same, or if this is only possible at Google’s size. Overall, readers found the story honest and liked learning about Google’s approach to tooling.

The challenge is to place four black queens and one black bishop on an empty chessboard so that every square is attacked—meaning there’s no safe spot for a white king anywhere. The article lets you try the puzzle on a digital board and shares some history about Kempelen, the inventor who created it. In the comments, people enjoyed trying different strategies, like putting queens in the corners or along the sides. Some shared their own solutions and said the puzzle helped them understand how strong queens are in chess. Others liked that this puzzle only uses black pieces and no moves, just placement. There’s also talk about Kempelen’s chess machine and how these kinds of puzzles are still great for learning today.

Let’s talk about a creative game project called “Xs of Y.” It’s a roguelike game with a new name and quest each play. The magic system is special: it uses Lisp code, and the runes are different every time, so magic is always a puzzle. The game is written in a small Lisp that runs on a Go virtual machine, with no outside dependencies. You get random titles and quests, and the magic runes change every run. Spells are written as code, but you have to figure out what the runes mean each time. The dungeon is full of dangers—spiders, goblins, slimes, trolls, and fire that spreads chaos. The project is open source, and you can play in your browser. In the comments, people love the creative use of Lisp for magic and the procedural names and quests. Some say it feels like classic roguelikes, while others want more guides for the magic system. Many like how easy it is to try the game and enjoy the mix of code and gameplay.

Our next story is about a security issue with Composer, the popular PHP package manager. Composer leaked full GitHub OAuth tokens in logs if the tokens used a new format with hyphens, which Composer’s old rules did not allow. When Composer saw a token with a hyphen, it threw an error and printed the whole token to the logs. Many CI systems, like GitHub Actions, add these tokens automatically, so users could leak tokens without knowing. The tokens usually expire quickly, but on self-hosted runners they could last up to 24 hours. The problem is fixed in new Composer versions, but the issue is serious because logs with secrets can be copied or sent somewhere unsafe. In the comments, people say secrets should never be printed in error messages. Some think Composer’s old token rules caused the problem, and that secret masking in CI is not perfect. While some say short-lived tokens are not a big risk, others warn that even a small leak can be dangerous. Many suggest always checking logs for secrets and updating logging practices.

Our last story is about the S-100 Virtual Workbench—a web tool that lets you build and run old computers like the Altair 8800 with virtual hardware cards. You can drag and drop CPUs, RAM, ROM, and I/O cards into a virtual case, boot up a system, and see how the parts work together. There are support for vintage CPUs like the Intel 8080 and Zilog Z80, and you can add disk controllers, graphics cards, and more. The tool has a bus analyzer and terminal, making it easy to see what’s happening inside the machine. In the comments, people like how easy it is to use and say it’s great for learning about old computers without real hardware. Some remember using real S-100 systems and enjoy seeing them recreated online. Others want more guides or support for more hardware. Many find it useful for teaching and software testing, and are excited about bringing vintage computing to more people.
