Our first story is about Claude Cowork, a new AI tool from Anthropic made to help with daily work. However, it has a serious security problem. Attackers can steal user files by tricking Claude into sending them to the attacker’s account. The main issue is a vulnerability in Claude’s coding environment that allows prompt injections. This means someone can hide secret instructions inside files, and Claude will follow them.

The article shows how this attack works step by step. First, a user connects Claude Cowork to a folder with important files, such as real estate documents. Next, the user uploads a file to Claude. This file looks normal but hides a prompt injection. Attackers make the hidden prompt hard to see, using tricks like tiny fonts or white text in a Word file that looks like a Markdown skill. When the user asks Claude to analyze files using this skill, Claude reads the hidden instructions. These instructions force Claude to run a command that uploads the largest file it finds to the attacker’s Anthropic account, using a secret API key hidden in the prompt. Everything happens automatically, and the user gets no warning.

This attack works because Claude’s coding environment blocks most outside network connections, but still trusts the Anthropic API. So, the attack is hard to spot. When the file is sent, the attacker gets the victim’s sensitive data, which could include financial numbers or parts of Social Security Numbers.

Even newer versions of Claude, like Opus 4.5, can be tricked by more complex prompt injections. Claude can also crash or freeze if forced to read files that are not what they seem, like a fake PDF. The risk is bigger because Cowork is made to work with many tools and files, so it often handles both private and untrusted data without the user checking everything first.

Hacker News commenters have many opinions. Some are shocked Anthropic has not fixed the problem, even though it was known before Cowork launched. Others say this is a big risk for regular users, who cannot be expected to spot hidden prompt injections. Many think telling users to “be careful with sensitive files” is not enough, and that the real fix should be at the system level. Some point out that the attack uses trusted APIs, so this kind of problem could appear in other AI tools too. There is also discussion about how prompt injection is a big, unsolved problem for all AI models that can run code or access files. Some users wonder if any AI agent should have such broad powers without strong security checks. Some developers share safety tips, like never uploading files you did not make yourself, but others say this is not practical for most people. Overall, commenters agree that AI tools with broad access need better security and clearer protection for users.

Now, here’s a story about why some clothes shrink when you wash them and what you can do to fix it. The article explains the science behind shrinking and offers a simple way to try to “unshrink” clothes.

Clothes are made from different fibres like cotton, linen, wool, or synthetic materials. Plant fibres like cotton and linen have long chains inside them. These chains are stretched and held tight when the fabric is made. When you wash them, heat, water, and movement can make the fibres relax and return to their natural, crinkled shape. This is why your clothes might get smaller. Hot water makes shrinking worse, but even cold water can cause it, just more slowly. Loosely made fabrics, like knits, shrink more easily because the threads can move around. Tightly woven fabrics are less likely to shrink.

Wool shrinks differently. Wool has tiny scales on its surface. When washed, these scales open and grab onto each other, making the fabric dense and smaller. Synthetic fibres, like polyester and nylon, do not shrink much. They are made to be strong and stable, so their fibres do not change shape easily. Some clothes mix natural and synthetic fibres to help stop shrinking. Scientists are also working on new materials that can remember their shape, so shrinking will not be a problem in the future.

If your clothes do shrink, you can try to fix them. Soak the item in warm water with some hair conditioner or baby shampoo. This makes the fibres soft and slippery. Then, gently stretch the fabric back to its old size. Lay it flat to dry, or pin it to keep it stretched. This may not work perfectly for very shrunken clothes, but it can help a little.

In the comments, many people share stories about shrinking clothes. Some say they always wash with cold water and gentle cycles and have fewer problems. Others think dryers are the main reason for shrinking, not the washing machine. Some have tried the hair conditioner trick and say it helped, but only a little. Some say expensive wool clothes still shrink, even if you wash them by hand. Others wonder why care labels are often confusing or wrong. A few buy only synthetic clothes to avoid shrinking, while others like natural fibres better, even if they need more care. One person points out that old clothes shrink less, maybe because they have already been washed many times. Another says the best way is to avoid the dryer and always air-dry clothes. Overall, people agree that shrinking is common, but there are ways to make it less likely.

Our next story is about age gates online—the screens that ask you to prove how old you are before you can use a website or app. The article explains why age gates are common and how they can affect your privacy and rights.

Age verification is required on many sites now. While it is supposed to protect kids, it often means giving up a lot of personal data. The EFF (Electronic Frontier Foundation) is against these rules because they risk your privacy and free speech. The article lists different ways sites check your age: some guess from what you post, some ask for a selfie, some want your government ID, and others use a credit card or email. Each method has problems. Face scans may not work well for everyone. Uploading your ID or credit card means risking leaks if the company is hacked. Even if they say they delete your data, mistakes happen and sometimes data is kept too long.

The guide suggests always giving as little information as possible, and gives questions to ask before sharing your data: What do they need? Who can see it? How long do they keep it? Has anyone checked if their system is safe? Who knows you are trying to prove your age?

The article reviews how Meta, Google, and TikTok handle age gates. Meta tries to guess first, then asks for a selfie or ID if it can’t. Google also guesses, then may ask for a face scan, credit card, digital ID, or photo ID. TikTok uses your videos and photos to guess your age, and if it thinks you are too young, it can delete your account unless you prove your age fast. These companies often use third-party services, which is another risk.

For other websites, the same problems apply—give the least data possible, and check how your data will be used and stored. The main message is that all current age checks have problems, and EFF is fighting against age-gating laws.

Next, we have news from the OpenZFS Developer Summit 2025. MayaNAS has a new way to use ZFS directly with cloud object storage. They built a tool called objbacker.io, which lets ZFS talk straight to S3, GCS, and Azure Blob Storage, skipping the slow FUSE layer.

The team wanted to solve the high cost of running file servers in the cloud. Usual cloud block storage is expensive—$96,000 per year for 100TB on AWS EBS, and $360,000 for AWS EFS. Their idea is that not all data needs to be fast, so they split storage into two: metadata and small blocks stay on fast NVMe SSDs; big blocks go to cheap object storage. ZFS’s special device setup makes this easy.

Old methods use FUSE programs like s3fs, but these are slow because every read and write crosses between kernel and user space many times. Objbacker.io uses a new ZFS VDEV type, talking to a user-space daemon through a special device file. This daemon uses cloud SDKs to read and write data directly to object storage.

When storing data, ZFS blocks (set to 1MB) map directly to cloud objects. For example, a 5MB file becomes five 1MB objects, named by offset. This helps with fast, aligned writes and reads.

Their benchmarks show strong results: 3.7 GB/s reading and 2.5 GB/s writing from S3, by striping data across six S3 buckets. Testing used big cloud servers and 10 parallel jobs.

They also talked about MayaScale, another product for block storage with very low latency, using NVMe SSDs. It works across AWS, Azure, and GCP. Deployment is the same on every cloud, using Terraform and similar configurations.

In the comments, many are excited about direct ZFS-to-object-storage, calling it a big step. Some ask how stable and production-ready it is, and if it handles network failures well. Others wonder about backup strategies and recovery. Some are worried about cloud costs if data is accessed often, since object storage is cheap for storage but can be expensive for frequent use. A few point out that performance numbers are great, but want to see real-world mixed workloads. There are questions about compatibility with existing ZFS tools and if special tuning is needed. Several people compare this to other cloud storage systems and think native integration is the right direction.

Our last story today is about a new tool called webctl, which lets you control a web browser from the command line. It is made for both people and AI agents who want to automate browser actions easily.

Webctl uses commands like “navigate,” “type,” and “snapshot” to move around websites, fill forms, and get web page details. You can use filters to only see important page elements, like buttons or links. The tool lets you use Unix commands like grep or jq to search and manage its output. It is easy to script webctl actions, save them, and run them again. You can run browser sessions in the background, save cookies and tabs, and switch between profiles. Actions are based on ARIA roles, which stay the same even if the web page’s style changes. For every command, you can control how much information you get back, in text or JSON. Webctl is built on Chromium and Playwright, and you can install it using Python. It is also made to work well with AI agents, with setup guides and ready prompts. The tool’s design separates the command line tool from the browser process, making it flexible and easy to control.

In the comments, people like that webctl uses the CLI instead of the usual message-based style. Some say the CLI gives them more power because they can filter and debug using tools they already know. Others mention that CLI makes it easy to connect browser actions to scripts or other programs. A few are happy about ARIA role selectors, saying this is more stable than using CSS selectors. Some worry that managing browser sessions and cookies from the command line could be hard for new users. One person asks about security risks. Another says this tool could help AI agents use the web better because the output is simple and clear. Some think it’s great for quick testing, while others wonder if it can be used for big, complex projects. Overall, most people like the idea and see many ways to use webctl for automation and agent work.
