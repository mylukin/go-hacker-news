## TanStack NPM Packages Compromised

- 原文链接: [TanStack NPM Packages Compromised](https://github.com/TanStack/router/issues/7383)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48100706)

Some new versions of the TanStack npm packages were found to be infected with malware. This attack may affect many users who use these packages in their projects. The problem was first seen in the @tanstack/history package, but other related packages were also quickly checked. The infected packages have a hidden dependency in their package.json file, which causes npm to fetch a special commit from GitHub. This commit runs a script during install, using the “prepare” step. The script then runs a large, hidden JavaScript file called router_init.js.

This hidden code tries to steal credentials from the computer, looking for things like AWS, GCP, Kubernetes, npm, GitHub, and SSH keys. It sends the stolen data over an encrypted network, so it’s hard to block or trace. The code also tries to find out what other npm packages the victim maintains, so it can spread itself further by infecting those packages too.

The attacker used GitHub Actions OIDC to publish the bad code, which means the problem is not just a leaked token, but maybe the whole publishing workflow is compromised. Simply rotating tokens will not fix it; the workflow must be disabled. If you want to check your packages, you can unpack and inspect them for the special optionalDependencies and the router_init.js file.

One commenter warned that the malware even installs a “dead man’s switch”: if you try to block the stolen token, the script will delete your files. It sets up a background service on Linux or macOS to watch if the stolen GitHub token still works. If it is revoked, it erases the user’s home directory, making recovery very hard.

People in the comments had different reactions. Some said this shows how dangerous the npm supply chain is and that we need better security. Others pointed out that using install scripts is always risky and should be avoided when possible. Some users wanted npm to block install scripts by default, or at least show a warning. A few were shocked at how sneaky the attack was, hiding in a commit and using a self-spreading method. Others asked for clear steps on how to clean infected systems and prevent future attacks. There were also calls for more transparency from maintainers and package registries when something like this happens. Some worried about how easy it is for one compromised workflow to cause so much damage. Finally, people reminded others to always check what dependencies they use, keep backups, and use strict security settings whenever they can.

## Library for fast mapping of Java records to native memory

- 原文链接: [Library for fast mapping of Java records to native memory](https://github.com/mamba-studio/TypedMemory)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48099616)

The article is about TypedMemory, a new Java library that lets developers map Java records onto off-heap memory using the FFM (Foreign Function & Memory) API in Java 25 or newer. This helps people work with memory outside the Java heap more easily, while keeping things strongly typed and safe.

TypedMemory makes it possible to use Java’s record types as schemas for structured data in native memory. Instead of writing lots of low-level code to manage memory layouts and offsets, you can use TypedMemory to read and write records with simple get and set methods. You can allocate memory, fill and copy regions, wrap existing memory from native libraries, and inspect the layout to make sure it matches what you need for native interop. The API works closely with Java’s FFM model, so you still have direct control over memory, but it removes a lot of the repetitive work. You can use this for things like graphics, simulations, high-performance data, and working with native code. Examples show how to use the library to store colors, points, and more complex structures off-heap. It also supports nested data, fixed-size arrays, and lets you check how memory is laid out. TypedMemory is still experimental and needs Java 25 or above, and it’s available through Maven.

In the comments, many developers are excited about TypedMemory and see its value for high-performance and native interop tasks. Some users say this can help Java feel closer to languages like C when handling memory. Others appreciate that it makes the FFM API easier to use and less error-prone. A few people point out the limitation that you need Java 25, which is still very new and not widely used in production. Some are unsure about how well arrays as fields in records will perform, since Java arrays are usually on the heap. There are also questions about how to handle union types and more complex data layouts, and some suggest looking at how carrier classes in Java might help in the future. A few developers mention they want to see benchmarks to know how fast this really is. There is interest from people working in graphics, games, and systems programming, while others are just happy to see Java getting more low-level features. Some warn that changes in the FFM API or Java records might break things as the library evolves. Overall, there’s excitement, but also a sense that this is early-stage and needs more testing and real-world use.

## I let AI build a tool to help me figure out what was waking me up at night

- 原文链接: [I let AI build a tool to help me figure out what was waking me up at night](https://martin.sh/i-let-ai-build-a-tool-to-help-me-figure-out-what-was-waking-me-up-at-night/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48100662)

The article is about someone who used AI tools to build a system that helps figure out what noises are waking them up at night in a noisy city. The writer already had a smart home setup and used cheap microphones, a Raspberry Pi, and data from a sleep watch to collect and connect noise events with sleep interruptions.

The main idea is that it’s hard to know what wakes you up at night because by the time you notice, the sound is gone. Guessing the cause is expensive and often wrong. So, the writer added microphones—one inside, one outside—to record short audio clips only when loud noises happen while they are in bed. The Raspberry Pi listens for noise, but only records when the system knows the writer is home and asleep. All the data—sleep stages, heart rate, noise, and other sensor events—are shown in a web app. This helps the writer see exactly when they woke up and listen to what happened at that moment.

AI helped build the tool fast, but did not listen to the sounds; the human still tells if a sound was a door or a dish. The writer let coding AI write most of the code and even gave it access to test things on the Raspberry Pi. The tool works well, showing when and why sleep was disturbed. The writer found common causes: doors slamming, dishes, and street noise. With this info, they used sound panels, better insulation, and talked with flatmates to reduce noise.

The technical part includes recording only when needed, filtering out steady background noise, saving clips when something loud happens, syncing everything on a timeline, and making a private web app. The writer also suggests future features, like smarter notifications and automatic sound grouping, but is happy with the current version. The key lesson is that AI makes it much easier to build personal tools for small problems, which used to seem too hard or not worth the effort.

In the comment section, many readers liked the DIY approach and how quickly the project was built with AI help. Some thought it was a clever use of smart home tech and wished consumer products were this flexible. Others worried about privacy, but were glad the data stayed on the home network. A few people asked if the writer would open-source the code, but the writer explained it’s too rough for public use. Some readers shared their own experiences with sleep and noise, saying even simple audio logs helped them find surprising causes. Others pointed out that sleep trackers can be inaccurate, but agreed that wake times are useful clues.

There were also debates about whether letting AI write and run code is safe, especially with device access, though some said it’s fine for personal use in a controlled setting. A few readers suggested using machine learning to classify sounds automatically, while others warned that could be much harder than it seems. Some liked the idea of measuring before fixing, instead of guessing and spending money on the wrong things. Lastly, several people said this project shows how AI tools are making small, useful projects possible for individuals—and that’s exciting.

## UCLA discovers first stroke rehabilitation drug to repair brain damage (2025)

- 原文链接: [UCLA discovers first stroke rehabilitation drug to repair brain damage (2025)](https://stemcell.ucla.edu/news/ucla-discovers-first-stroke-rehabilitation-drug-repair-brain-damage)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48098261)

UCLA scientists have found a new drug that helps mice recover from stroke, working like physical rehab but as a pill. Usually, people who have a stroke lose some movement and rehab only helps a little, with no drugs available to fix brain damage.

The team wanted to know how rehab helps the brain after a stroke. They studied both mice and people who had strokes. They saw that a stroke breaks connections in the brain, even far from where the stroke happened. This makes it hard for brain cells to “talk” to each other, so movements like walking become difficult.

They found that one special type of brain cell, called a parvalbumin neuron, loses its connections after a stroke. These neurons help keep the brain’s rhythm, called gamma oscillations, going. When the rhythm is lost, movement suffers. Rehab can bring this rhythm back, fixing some lost connections.

The UCLA team tested two new drugs to see if they could make these neurons work again. One drug, called DDL-920, helped mice recover movement after stroke almost as well as real physical rehab. This means the drug might help people, too, by fixing the same brain circuits that rehab does.

This discovery is important because it’s the first time a drug has copied the effects of rehab in the brain. But the drug is only tested in mice so far. More studies are needed to check if it is safe for people.

In the Hacker News comments, some people were excited and hopeful, saying this could help many stroke patients, especially those who can’t do hard rehab. Others were careful, pointing out that results in mice don’t always work in humans and drug trials can take many years. Some worried about side effects and wanted to know how the drug works in older animals or humans.

A few users talked about how hard it is for stroke patients to get enough rehab and hoped this drug could make recovery easier. Others asked about the cost and if insurance would pay for it. Some shared stories about friends or family who had strokes and said new treatments are needed.

There was also technical discussion about gamma oscillations and why reconnecting brain rhythms matters. A few users were skeptical, asking for more proof and saying we should wait for human trials before getting too excited. But overall, most people agreed this is a big step for stroke recovery research.

This article is about Nullsoft, the company behind Winamp, and what happened between 1997 and 2004. It tells the story of how a small team made a popular music player and then got bought by a big company.

The article starts by explaining how Winamp was created by Justin Frankel and a small group. They made it simple and easy to use, which made it very popular for playing MP3 music. The team worked fast and often released new versions. Winamp became a big success on the internet, even before online music stores and streaming. The company, Nullsoft, was small and relaxed, with people working from home and making music software for fun.

AOL bought Nullsoft in 1999. At first, things seemed good because AOL gave the team more money and resources. But over time, working with a big company slowed things down. The AOL bosses did not always understand what made Winamp special. They wanted to add ads and change the software in ways the team did not like. The article says the team tried to keep their style, but it became harder. Some team members left. In the end, Winamp lost its place as the most popular music player, and Nullsoft’s unique culture faded away.

In the comments, many people remember using Winamp and loving its easy design. Some say Winamp was so fast and simple compared to music players today. Others talk about the cool skins and plugins you could use to change how it looked and worked. A few people think AOL ruined Winamp by trying to make it like other big-company products. Some users say this is a common story: small, fun tech teams get bought by big companies and lose their spark. Others point out that it was hard for any MP3 player to stay popular when music streaming became big. There are even some funny stories about using Winamp to play music at parties or in college dorms. Some wish Winamp could come back, while others are happy with today’s music apps.

## Can Someone Please Explain Whether Cloudflare Blackmailed Canonical?

- 原文链接: [Can Someone Please Explain Whether Cloudflare Blackmailed Canonical?](https://www.flyingpenguin.com/can-someone-please-explain-whether-cloudflare-blackmailed-canonical/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48098537)

This article talks about a big attack on Canonical, the company behind Ubuntu, in April 2026. Canonical’s main websites and services went down for almost a whole day because of a strong denial-of-service (DDoS) attack. The attackers used a paid service called Beamed, which advertises ways to get past Cloudflare’s protection and is itself hosted by Cloudflare.

The article explains that Canonical’s most important servers—the ones that give updates to all Ubuntu systems—were attacked last. After trying to stop the attack on their own, Canonical put these two servers (archive.ubuntu.com and security.ubuntu.com) behind Cloudflare to keep them online. Other Canonical sites stayed down longer but were not moved to Cloudflare.

The article also looks into the companies and networks behind the attack tools. It shows that the people running Beamed and its domain registrar have links to old privacy and activism groups, even involving founders of The Pirate Bay. The network used by Beamed changed hands and countries, but stayed connected to the same types of services.

At the same time as these changes, Canonical updated their server certificates—needed for moving to a service like Cloudflare. This timing seems important, but no one has explained it yet.

The article says no ransom was paid, and there is no public sign of a payment or demand. But it points out that Cloudflare is both protecting the victim (Canonical) and hosting the attacker (Beamed). The writer compares this to old rackets, where one company controlled both the threat and the solution for money.

Now, looking at the Hacker News comments, people have many views on what happened. Some think Cloudflare is not doing enough to stop bad actors from using its service, so it looks like they profit from both attackers and victims. Others say Cloudflare can’t easily police all its users without hurting privacy or making mistakes.

A few users see this as a bigger internet problem: the same companies that protect against attacks often also help attackers hide. Some argue Canonical had no choice but to pay for Cloudflare’s help, because the attack was too strong for them to handle alone.

Several commenters compare the situation to “protection rackets” from history, where you must pay someone to protect you from harm they could stop. Others feel Cloudflare is caught in the middle, acting as a neutral tool, and the real problem is with the people running the attack services.

Some people ask if Canonical should have been better prepared, suggesting they should not rely on one provider. Others point out that many companies—big or small—have little choice but to use services like Cloudflare after such attacks, because the internet is too unsafe otherwise.

Lastly, a few commenters worry that if Cloudflare or similar companies started banning users more, it could lead to too much control or censorship online. But most agree the whole situation is troubling, and the line between attacker and protector is unclear.

## Gmail registration now requires scanning a QR code and sending a text message

This new step is meant to make account creation more secure and stop fake accounts, but it also makes it harder for people who want to stay anonymous or use temporary phone numbers. The article points out that while this may help stop spam and phishing, it also blocks privacy-focused users from making anonymous accounts.
