With five network cards, each supporting up to 400 gigabits per second, this setup can move data very quickly. Dell says this is good for AI, big data, and backup tasks, since it is both large and power-efficient. Kioxia points out that customers can now handle massive data streams and build big data lakes with fewer servers, which should lower costs. Other companies, like Micron, Sandisk, SK Hynix, and Samsung, are also working on very large SSDs, with some even planning for 1 petabyte drives soon. This shows that flash storage is quickly replacing hard disks for high-speed, large storage needs.

Next, let’s talk about WSL9x, a project that lets you run a modern Linux kernel inside Windows 9x without rebooting. This mixes Windows and Linux so you can use programs from both at the same time, using features like memory protection and multitasking. WSL9x has three main parts: a special Linux kernel, a driver for Windows 9x, and a small DOS program called wsl.com. The driver loads the Linux kernel into memory, sets up threads and stacks, and lets Linux and Windows talk to each other. When you run a Linux command, the driver checks if Windows can do it, and if not, sends it to Linux. The Linux kernel has been changed to use Windows system calls. The wsl.com program lets you use the normal DOS window as a Linux terminal.

To use WSL9x, you need special tools to build Linux and Windows code, a Windows 9x disk image, and then you can start using Linux commands right inside Windows. For color, you can add a DOS driver for ANSI colors.

Hacker News users were surprised and impressed that modern Linux can run on such an old system. Some ask why, since Windows 9x is old and not very safe, but many see it as a fun and clever challenge. There is talk about the hard parts, like sharing memory and interrupts. Some wonder if it’s a joke, but others see real technical skill, especially in rewriting Linux to use Windows APIs. There are also stories about old computers and nostalgia for Windows 95 and 98. Most agree this is mainly for fun and learning, not for serious work, but the documentation is clear and interesting for anyone who likes old and new tech together.

Now to AI video. NVIDIA has released SANA-WM, a new open-source AI model that can turn just one image and a camera path into a full minute of 720p video, all on a single GPU. The model has 2.6 billion parameters and is designed to make long, detailed videos that follow exact camera moves. It uses a hybrid linear attention method to make longer videos without using too much memory, and it controls the camera’s position in 3D space very carefully. It works in two steps: first making the basic video, then refining it for more detail. The team trained it on about 213,000 public video clips, finishing in just 15 days with 64 GPUs. Once trained, the model can make a minute-long video in about half a minute on just one good GPU.

SANA-WM can create many different scenes, from forests and caves to spaceships and alien worlds, starting from only a single image. The videos are consistent and natural, with moving details like snow or fireflies. The model can follow complex camera paths, letting users control how the scene changes. Its quality is close to much larger, closed models but it uses less computing power and is fully open-source.

Commenters are impressed by the speed and quality, saying this is a big step for open video AI. Some discuss how SANA-WM could help animation, film, or game design. Others ask about training data and how well it generalizes, and compare it to Google and OpenAI’s bigger models. There are technical questions about hybrid attention and the two-stage process. Some worry about the need for expensive GPUs for training, but most like that running the model is much easier than before. People are excited to try the code and hope to see the pre-trained model weights soon.

Now, a story from chemistry. Scientists have made a new molecule with a “half-Möbius” shape, which is very rare. It’s a ring of 13 carbon atoms and 2 chlorine atoms, with a twist of only 90 degrees, not the full 180 degrees of a real Möbius strip. This means special features only repeat after going around the ring four times, not two.

The team made this molecule by removing chlorine atoms one by one from a bigger molecule, using a sharp tip on a cold surface. They used special microscopes to look at the atoms and the way electrons move around the ring, finding spiral-shaped patterns. They could also switch the molecule between different states: two mirror-image forms with opposite twists, and a flat, normal state. This switching was done by touching the molecule with the microscope tip. Computer calculations, some on quantum computers, helped show how the electrons and atoms change.

This is important because molecules with strange shapes can have new kinds of stability and ways for electrons to move, which could help with molecular electronics or quantum computing. Usual chemistry rules do not apply here, so this is new ground.

In the comments, people are excited about the idea of a half-Möbius molecule and praise the clever experiments. Some want to know if these molecules can be made in larger amounts, or only one at a time. Others wonder about real-life uses, or if these shapes are just scientific curiosities. There are questions about the need for quantum computers for the calculations. Most agree this is a creative and important step in molecular science.

Let’s move to large language models. A new idea called δ-mem helps these models remember and use information from past conversations. Instead of just making the memory bigger, δ-mem adds a small, smart memory system—an 8x8 matrix that learns using the “delta rule.” This means it updates a little at a time as new information comes in. The model uses this memory to make small changes to how it pays attention to past words, improving tasks where memory is important. It does not need retraining or a bigger context window. Tests show it can improve scores by about 1.1 to 1.3 times over older methods, without hurting other skills.

Hacker News users like this practical approach, pointing out it can save money and time. Some remember the delta rule from earlier AI and are happy to see it return. There are questions about how much memory is enough, and if this will work for very complex tasks. Some users want open-source code or a demo. There are also concerns about security and leaking private data. Still, most see δ-mem as a smart and simple way to make language models remember better.

Now for a bit of computer history. The term “Halt and Catch Fire” or HCF started as a joke, but has real history. HCF is used for a computer instruction that makes the CPU stop working, sometimes only fixable by turning off the power. The “catch fire” part is a joke, but with old IBM machines, bad instructions really did make parts overheat. The story focuses on the Motorola 6800 chip, which had some undocumented instructions. If you ran the wrong code, the chip could freeze and start reading every part of its memory over and over, and nothing could stop this except a reset.

A 1977 BYTE magazine article described two dangerous codes for the 6800, called HCF. Sometimes engineers used them on purpose, as the signals made it easy to check things with special tools. Later, Motorola engineers explained this behavior was even kept on purpose for testing. Other processors, like the 6502 and the Pentium F00F bug, had similar problems. Today, people use random data to test for bugs, an idea called fuzzing.

In the comments, many readers share stories about old computers breaking or acting strangely with weird instructions. Some remember the TV show “Halt and Catch Fire.” Others say real hardware bugs can be serious and hard to fix, especially when only power-cycling helps. Some joke about making their own HCF commands, while others think the phrase is overused. Most agree that “Halt and Catch Fire” is a perfect example of old-school computer humor.

Now, a story from literature. There is a new translation of Albert Camus’s complete notebooks, which helps correct misunderstandings about his ideas and reputation. For a long time, many thought of Camus as an existentialist or a philosopher of the absurd, but this translation shows he saw himself more as an artist and a writer. The translation collects all his notebooks from 1933 to 1959, including the Oran Notebook, which shows his thoughts while writing “The Stranger” and “The Myth of Sisyphus.”

Camus did not want his books to be thesis-novels that teach a philosophy. He wanted to show human experience, not just ideas. He believed mixing philosophy and fiction too much makes stories artificial. The notebooks show he wanted to focus on the body, experience, and the present, not just abstract thinking. He argued against violence and turning people into ideas or enemies. Camus struggled with depression and fame, and did not want to be labeled as an existentialist.

In the comments, some readers agree that Camus has been misunderstood and praise the new translation for showing his real thoughts. Others say labels like “existentialist” are often used by marketing and media, not to explain ideas. Some discuss how hard it is to change public opinion once a wrong idea is popular. Many share how reading Camus’s notebooks changed their understanding of his work. Some say academic or media simplifications can hurt the real meaning of an author’s work, and new translations are important for seeing classic writers in a new way.
