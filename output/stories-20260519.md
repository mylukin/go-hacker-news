## I’ve built a virtual museum with nearly every operating system you can think of

- 原文链接: [I’ve built a virtual museum with nearly every operating system you can think of](https://virtualosmuseum.org/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48195009)

This article talks about a project called the Virtual OS Museum, which is a big collection of old and new operating systems that you can try out on your computer. The museum is made as a Linux virtual machine and works with QEMU, VirtualBox, or UTM, so it can run on Windows, macOS, and Linux. It comes with its own launcher, so you do not need to set up emulators or install anything extra. The launcher has a snapshot feature, which lets you undo changes if you break something while trying an operating system. The museum includes more than 250 platforms and over 570 different operating systems, from the first computers in 1948 to systems we use today. You can see rare systems like CTSS, early Unix, Xerox Star, and many more that are hard to find or run elsewhere. There are two versions: full (everything is downloaded and runs offline) and lite (downloads what you need as you use it). Both versions can be updated easily, so you always have the latest additions. The creator spent 20 years collecting and fixing these systems, sometimes needing to patch emulators to make them work. Each system is set up with extra software, so you see them as people used them at the time. The project aims to make computer history easy to access—just click and start exploring.

People on Hacker News had a lot to say about this project. Many users said it is an amazing resource for learning about computer history and for nostalgia. Some people thanked the creator for the huge effort and time spent on this, calling it a “labor of love.” Others were surprised by how many rare and forgotten systems are included, and shared their own memories using some of these old computers. A few users mentioned technical challenges, like the size of the full download and possible licensing issues for some software. Some asked about adding even more obscure systems, while others suggested ways to make the launcher or website even better. There were questions about how the museum handles copyright and legality, and some users worried about old commercial systems being shared. A few people talked about how useful this could be for teaching or research. Some users also gave tips for running the museum on slower computers. Many agreed this project is important for digital preservation and makes it much easier to study and enjoy the history of computing. Overall, the community was very positive, with lots of interest in trying the museum and helping it grow.

## OpenAI Adopts Google's SynthID Watermark for AI Images with Verification Tool

- 原文链接: [OpenAI Adopts Google's SynthID Watermark for AI Images with Verification Tool](https://openai.com/index/advancing-content-provenance/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48198291)

OpenAI now uses Google’s SynthID watermark to mark AI-generated images and has a new tool for people to check if an image came from OpenAI. This is to help people know if images are made by AI and trust what they see online.

The article explains that OpenAI wants people to understand where digital images and audio come from. They call this “content provenance.” OpenAI has joined a group called C2PA, which sets standards so that information about an image’s origin can travel with the file. This information includes who made the image and if it was changed. But sometimes, this information (metadata) can be lost if someone changes the file or takes a screenshot.

To fix this, OpenAI now adds a hidden watermark to images using Google’s SynthID. This watermark is invisible but can be found later, even if the image is changed a little. They will use both C2PA metadata and SynthID watermarks. If one is lost, the other may still be there. This way, it is harder to remove all proof that an image is from OpenAI.

OpenAI also released a public tool. Anyone can upload an image to check if it has OpenAI’s watermark or metadata. The tool cannot always say for sure where an image came from, especially if all signals are missing, but it will help people check more often. The tool only works for OpenAI images now, but they hope it will work for other platforms later.

In the comments, many people say this is a good step for transparency and trust. Some like the idea of having a public tool to check images. Others worry that people can still remove or break watermarks and metadata. A few say bad actors will always find ways to hide AI-generated images. Some developers ask if the watermarking will be open source or if others can use it. Others wonder if this will slow down creative uses of AI or make things complicated for people who just want to make art.

Some users ask about privacy and if checking images in the tool is safe. A few are concerned that watermarking could be used for tracking or control. There are also comments about how this might help fight fake news or deepfakes, but some say nothing is perfect and people should not trust any signal alone. Many agree that using several ways to mark and check images is better than just one, but they also think this is only the beginning and more work is needed.

## Show HN: Forge – Guardrails take an 8B model from 53% to 99% on agentic tasks

- 原文链接: [Show HN: Forge – Guardrails take an 8B model from 53% to 99% on agentic tasks](https://github.com/antoinezambelli/forge)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48192383)

Forge is a Python framework that helps run local language models (LLMs) for multi-step tasks with more reliability. The tool uses “guardrails” and special context management to make 8B models work much better on complicated, agent-like workflows—going from 53% accuracy to as high as 99% in some tests.

The main idea is that small local models often make mistakes when calling tools or following steps. Forge adds layers that check, fix, or retry answers, making sure the model does what you want. It offers three main ways to use it: (1) WorkflowRunner, which handles full workflows with tools and step control, (2) Guardrails middleware for adding reliability checks to your own loops, and (3) an OpenAI-style proxy server that adds guardrails between any client and your model server. Forge supports several backends, like llama-server, Ollama, Llamafile, and Anthropic, so you can pick the one that fits your hardware or needs.

To get started, you need Python 3.12+ and a running model backend. You can install Forge easily with pip. The docs give clear examples, like a weather-checking workflow, showing how to define tools, set up prompts, and run tasks. For more complex or long-running tasks, Forge’s context manager can keep track of history and memory use, which is important for small models with limited VRAM.

The proxy server mode lets you point any OpenAI-compatible client to Forge, and it will handle the tricky parts—like making sure the model uses tools correctly. This is done by injecting a “respond” tool, keeping the model in a tool-calling mode so it doesn’t get confused between regular text and tool outputs. There’s also a test suite, eval harness, and guides for picking models, setting up backends, and understanding how Forge works inside.

Forge is open source under the MIT license and has a published paper explaining its design and results.

Hacker News comments show strong interest in running agentic workflows with local models. Some users are impressed by the high jump in reliability—going from about 50% to almost 100% on complex tasks. People like that Forge makes small, cheap models much more useful, since you don’t need expensive cloud APIs. Some worry about the extra hardware or Python version needed, but others point out that local LLMs are now much easier to set up than before.

A few comments ask about performance on even smaller GPUs or older hardware. Others discuss the pros and cons of each backend: llama-server gives better results, but Ollama is easier for beginners. Some developers are excited about using Forge for chatbots, voice assistants, or long-running services, since it helps avoid the usual problems of models forgetting context or making bad tool calls.

There are questions about how well these guardrails work with new models or unusual tools, and whether they add much delay. Some users share tips on integrating Forge with their own projects or combining it with other orchestration frameworks. A few want better Windows support or more simple examples. Overall, the community sees Forge as a promising step for reliable, self-hosted AI agents.

## Dumb Ways for an Open Source Project to Die

- 原文链接: [Dumb Ways for an Open Source Project to Die](https://nesbitt.io/2026/05/19/dumb-ways-for-an-open-source-project-to-die.html)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48198127)

This article lists many reasons why open source projects stop working, even when people still use them. It talks about how projects often “die” in ways that are hard to notice at first.

The writer explains that sometimes the main person who looks after a project leaves. This can happen because they lose interest, change jobs, or the company who made the project stops caring. Sometimes a student makes the project, but after they finish school, nobody keeps it going. Projects can also stop when money runs out, or when the owner gets a new job and does not have time anymore. Sometimes, people want to help, but they cannot get access to the project’s code or publishing rights, so nobody can make updates.

Other times, a project can look alive, but really nothing important is happening. Maybe only small fixes are made, or bots are doing all the work. People might try to help, but the maintainer is rude, so new helpers leave. Sometimes, the person who knows how everything works leaves, and nobody else feels safe making changes. Sometimes, two maintainers fight, and the project stops moving.

The article also talks about bad things that can happen, like someone tricking their way into being a maintainer and then adding harmful code. Sometimes, a maintainer gets angry and breaks their own project on purpose. Projects can also have problems if the tool to publish new versions is broken, or if the latest code is too different from the last release, so nobody wants to publish it.

There are times when the world changes, and projects cannot keep up. Maybe the platform they depend on is gone, or a service they use is shut down. Sometimes, a project is not needed anymore because something better comes out, or the same thing is now built into the language.

Projects can also split into two or more versions if people disagree, and then users do not know which one to choose. Sometimes, the project changes its license, and people fork it, but most users do not switch to the new one.

In the Hacker News comments, many people agree with the list and share their own stories of projects dying. Some say this is normal because software changes fast. Others feel sad that important projects can stop because of small problems, like lost passwords or bad planning. A few think maintainers should do more to find new people to help before leaving. Some developers say it is hard to keep working on free projects when they need to earn money. Others talk about problems with companies owning open source code but not caring for it. Many agree that users rarely check if a project is alive as long as it works in their code. Some point out that even “dead” projects may keep being used for years because nobody wants to change old systems. A few people wonder if open source needs better ways to show if a project is still healthy. Others remind everyone that open source depends on volunteers, so we should not be surprised when projects stop. Some suggest making it easier to hand over ownership, while others think the current systems are too slow. Finally, several say it is important to have backups or forks in case the original project disappears.

## Apple unveils new accessibility features

- 原文链接: [Apple unveils new accessibility features](https://www.apple.com/newsroom/2026/05/apple-unveils-new-accessibility-features-and-updates-with-apple-intelligence/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48192224)

Apple just shared new accessibility features for their devices, many powered by Apple Intelligence, to help people with disabilities use technology more easily. These updates will roll out later this year across iPhone, iPad, Mac, Apple TV, and Apple Vision Pro.

VoiceOver and Magnifier now use Apple Intelligence to give better, more detailed descriptions of images and the world around users, helping people who are blind or have low vision. Users can ask follow-up questions with their voice to learn more about what’s on the screen or in the camera’s view. Magnifier gets a high-contrast mode and voice commands like “zoom in” or “turn on flashlight,” making it easier to use for people with low vision.

Voice Control is now smarter and lets people use natural language to control their device, like saying “tap the purple folder” instead of using numbers or memorizing button names. This is useful for people with physical disabilities and also helps when apps are not properly labeled for accessibility. Accessibility Reader can now handle more complex documents, including scientific articles with tables and images, and can give quick summaries or translate text while keeping your color and layout preferences.

A big new feature is automatic subtitles for videos that don’t already have captions. This means personal videos, family clips, and even some online videos can now show transcribed text, using on-device processing for privacy. Apple Vision Pro will also let some users control power wheelchairs using just their eyes, which could be life-changing for people who cannot use a joystick.

Apple is also launching the Hikawa Grip & Stand, a MagSafe accessory for iPhone, in new colors. It helps people with grip and mobility challenges hold their phones more easily. Other updates include larger text for Apple TV, better support for hearing aids, ways to reduce motion sickness in Vision Pro, and the ability to use Sony’s accessible game controller across Apple devices.

In the Hacker News comments, many people praise Apple for continually pushing accessibility forward and for bringing new AI-powered features to help users with disabilities. Some users share stories about how Apple’s accessibility tools have helped family members or friends in real life. A few developers note that Apple’s work often sets a standard that other tech companies follow.

Some commenters, however, worry about the limits of these features—like language support, regional availability, and how well AI can really describe complex images. A few point out that while Apple does well, third-party apps sometimes ignore accessibility, and hope Apple will enforce better standards in the App Store. There’s also discussion about potential privacy issues, but many are glad that subtitles and image processing happen on the device.

One person suggests these improvements still need real-world testing with disabled users to make sure they work as promised. Another commenter is excited about the wheelchair eye tracking, calling it a “huge step” that could help many people live more independently. Some wish these tools were available on cheaper devices, not just Apple’s newest hardware. Overall, the community feels these changes will make daily life better for many, but stay cautious about real-world impact and the need for ongoing support.

## Minnesota becomes first state to ban prediction markets

- 原文链接: [Minnesota becomes first state to ban prediction markets](https://www.npr.org/2026/05/19/nx-s1-5821265/minnesota-ban-prediction-markets)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48197980)

Minnesota just passed a new law that bans prediction market websites like Kalshi and Polymarket from working in the state. These websites let people bet money on things that might happen in the future, like sports games, elections, or even what words someone will say in a speech.

The new law makes it illegal to run or advertise these websites in Minnesota. It also bans tools like VPNs that people might use to hide their location and still use these sites. If companies break the law, they can be charged with a felony. The law will start in August.

There are some exceptions. For example, people can still use weather contracts, which farmers use to protect their crops from bad weather. The law also does not ban trading stocks or buying insurance.

This move started a fight between Minnesota and the federal government. The federal group in charge, called the Commodity Futures Trading Commission (CFTC), says only they should make rules for prediction markets. They are taking Minnesota to court to try to stop the law. The CFTC says the law hurts farmers and regular people who use these markets. Some people say these websites are more like stock markets than gambling, and banning them will just push users to websites in other countries.

Many people, including law professors, think prediction markets are now too big and popular to stop. Other states are also thinking about similar bans, but prediction markets keep growing fast.

The top comments on Hacker News show mixed opinions. Some people think banning prediction markets is not smart. They say it will just move the activity to other places, maybe overseas, where it is less safe. Others say prediction markets can be dangerous. They worry about people betting on real events and trying to cheat or change the outcome.

One commenter notes that prediction markets are useful for farmers and businesses, so a total ban might hurt the economy. Another person points out that these sites often mix gambling with more serious contracts, which makes regulation hard.

Some users think the federal government should be in charge, not states, to keep rules simple across the country. A few people say that banning these markets protects kids and stops gambling problems. But others reply that people will always find a way to bet, so bans do not really help.

There is also debate on whether prediction markets are really different from gambling, or just the same thing with a new name. A few are worried about insider trading and people using secret information to win bets. In the end, many agree the law will not stop prediction markets, but will make the rules less clear and maybe less fair.

## Tesla's lithium refinery discharges 231,000 gallons of polluted wastewater a day

- 原文链接: [Tesla's lithium refinery discharges 231,000 gallons of polluted wastewater a day](https://www.autonocion.com/us/tesla-lithium-refinery-texas/)
- HN链接: [Hacker News讨论](https://news.ycombinator.com/item?id=48198551)

Tesla has a new lithium refinery in Texas, and workers found it was releasing black wastewater into a local ditch. The plant was promoted as “clean” and “acid-free,” but it discharges up to 231,000 gallons of treated water daily, and the local drainage district was not told about this in advance.

The Texas environmental agency (TCEQ) gave Tesla a permit to release this water, but did not require checking for heavy metals or even lithium in the wastewater. When the drainage workers noticed the strange pipe and dark water, they ran their own tests. Their chosen lab found small amounts of dangerous chemicals like hexavalent chromium (a known cancer risk), arsenic, and high levels of lithium and other metals. Some of these chemicals can hurt people, animals, and plants. For example, the salty water is killing grass along the ditch, which could make flooding worse if the soil washes away. 

None of these chemicals were checked by the state investigation, because the permit didn't require it. Tesla says it follows the rules and the water is within legal limits. They also argue that the lab tested water in the ditch, not directly from their pipe, so the pollution might come from somewhere else. However, the drainage district says the mix of chemicals matches what you’d expect from a battery plant like Tesla’s.

The issue is not that Tesla broke the law, but that the rules may not be strict enough or may not cover all the right substances. This is especially worrying because the nearby city, Corpus Christi, is running low on drinking water due to drought.

In the comments, some people are shocked that a “clean” lithium plant is releasing toxic chemicals, even if they are in small amounts. Others say this kind of pollution is common for industrial factories, and the bigger issue is the government not making companies check for more chemicals. A few defend Tesla, saying they are just following the permit and rules, and the state is responsible for setting better guidelines. Some commenters argue that making batteries in the US is important, but it’s not truly “green” if it still causes pollution. There are also calls for more regular, independent testing and for permits to require checks for things like lithium and heavy metals. Finally, some people say this is a good reminder that “clean” technology often has hidden costs, and people should be careful about believing marketing claims.
