Hello everyone, this is the 2026-05-21 episode of Hacker News Daily Podcast. Today, we have a rich set of stories for software developers and tech fans. Let’s get started.

First, let’s talk about a special star map made for Andy Weir’s science fiction book “Project Hail Mary.” This online tool helps readers see which stars the main character visits in the story, using real data from the Gaia space telescope. The 3D map lets you move and spin to view stars from different angles, and shows star names and distances just like in the book. The spaceship’s path is marked, so fans can understand how far the hero travels. The project uses JavaScript and WebGL, and the author shared that matching the book’s story with real star data was not easy. Some star positions are not perfect, but the map tries to stay close to reality. The map works in most browsers, and the creator hopes it adds more fun to reading the book.

In the comments, many people said the map is fun and impressive. Some thanked the creator for making the story come alive. Others talked about how hard it is to match fiction with real data, and shared tips about other astronomy tools. Some users joked they wanted this map when they first read the book, while others pointed out small bugs or new features they’d like to see. A few said they learned new things about space thanks to this project, and most agreed it’s a great mix of science and fiction.

Moving to hardware, Flipper Devices is building a new open device called Flipper One. This is not just an upgrade to Flipper Zero, but a fully open Linux computer for hardware and network hacking. The team wants Flipper One to be the most open ARM device, with all hardware and software fully documented and supported by the mainline Linux kernel. There is one closed part left for RAM initialization, but they are working on it. The device has a strong CPU and a small microcontroller, so some features work even when Linux is off. It offers many ports, including two Gigabit Ethernet, Wi-Fi 6E, M.2 slots for modems or SSDs, and GPIO pins for custom add-ons. The hardware design is open, and you can download the 3D models to make your own modules.

Flipper OS, running on Flipper One, supports “profiles” to quickly switch between settings. There’s a menu-based UI for the small screen, so you don’t need a full desktop. Flipper One can be used as a desktop, media box, or network tool, and even run AI models locally. The development is fully open, and the team invites everyone to help.

On Hacker News, many users are excited about open hardware and hope it pushes the industry to avoid closed drivers. Some worry about the last binary blob, and others like the co-processor design. There are questions about security, and some people compare Flipper One to Raspberry Pi, wishing Pi had been this open. The team’s honesty about risks was praised, and others talked about the long time it takes to support new hardware in Linux. People are curious about the device’s price and hope to help with code or testing. The community is hopeful but realistic about the challenges.

Next, let’s look at a story about moving a personal blog from an old Ubuntu server to a new FreeBSD server. The blog ran for ten years on an old Digital Ocean server, but the system was outdated and not safe. The author picked a Hetzner VPS in Germany, running FreeBSD, which is known for stability and security. Instead of Docker, the author used FreeBSD Jails, which keep each website separate. A tool called Bastille made Jails easy to manage. Each site got its own Jail, and there was another for the main web server, which used Caddy instead of nginx. This setup makes it easy to fix problems without touching other sites.

The author benchmarked the new server and found it much faster, though some of the speed came from better hardware and network. They also fixed some tuning problems to make it run even better. 

In the comments, many praised the clear explanation of FreeBSD Jails and Bastille. Some were inspired to try FreeBSD, while others noted that it’s less common than Linux but good for learning and security. Some worried about keeping up with updates, and others argued that using Jails is “old school” but effective. There was debate about Caddy vs. nginx, with some liking Caddy’s easy SSL and others preferring nginx. A few readers wondered if the speed gains were mostly from hardware, but most agreed it was a fun and useful learning project.

Now, let’s talk about Freenet, a new peer-to-peer platform. Freenet lets people build and use apps without big tech or central servers. It connects computers in a ring, sharing data directly between people. Apps run in your browser and look like normal websites, but no one can easily take them down or collect your data. Developers can use Rust and TypeScript to build apps, and don’t need to pay for servers. The project is run by a small team and funded by donations.

In the comments, some people are excited about Freenet’s ideas, especially for privacy. A few remember older versions and are glad to see updates. Some worry about problems, like stopping illegal content or spam, since no one controls the system. There are questions about how easy Freenet is to use, and if it will last. Most people find it interesting, but want to see how it works in practice.

Next, we have a review about using Kagi Search, a paid search engine, by someone with low vision. The writer says normal search engines made them tired due to ads and clutter. Kagi is different—it has no ads, doesn’t track users, and is easy to customize. You can change the theme, font size, and even use your own CSS. There are special tools called “lenses” to filter results, and shortcuts to search other sites. The writer explains how to set Kagi as the default search engine in different browsers and shares accessibility tips.

Commenters praise Kagi’s clean design and privacy focus. Some say the paid model is worth it, while others wish for a free option. Users with low vision appreciate the customizations, and developers like the advanced search tools. Some think paying for search is still hard to accept, and worry about missing results. Others wish big search engines would offer similar features. Overall, people like Kagi’s focus on user needs.

Moving on, Google just announced new AI-powered ads for Search, using its Gemini model. The new ads aim to be more helpful and personal. There are two main formats: Conversational Discovery ads that answer your questions and Highlighted Answers that show products in lists, both with AI explanations and clear labels for sponsored content. Google also adds AI Shopping ads and a Business Agent for Leads, so you can chat with brands directly inside ads. There are new offers and faster checkout options too.

Hacker News users had mixed feelings. Some worry about privacy and data collection, and think AI makes ads more pushy. Others like the idea of more helpful ads that are less annoying. Developers want more control and worry about transparency and fairness. There’s concern that small companies might not keep up. Some are excited about chat-based ads, while others just want a way to turn off all ads.

Next, someone shared their story of quitting a big tech job and spending $48,000 to build a six-GPU server at home. The main question: is buying better than renting GPUs in the cloud? The server, named “grumbl,” uses RTX 6000 Ada GPUs for machine learning. Building the server at home was hard due to power limits, so they later moved it to their parents’ house. After tracking usage and costs, they found that if the server is used over 85% of the time, owning saves money compared to cloud rental. They saved about $17,000 so far, but maintenance and downtime were stressful.

The writer warns that building a custom GPU server is risky, takes a lot of time, and is not for everyone. Most people should rent or use smaller setups. Many in the comments agreed: owning makes sense for heavy users, but hidden costs are high. Some like the control and speed, but others point out that hardware can break, power and cooling are expensive, and insurance is a problem. Some shared their own stories, and most agreed that having your own server is fun, but full of surprises.

Now, let’s look at Seattle Shield, a private intelligence-sharing network run by Seattle police. Big companies like Amazon and Facebook are members, and the network shares information about local events, threats, and protests, not just terrorism. Sometimes, reports include photos of people or cars, and are sent to many agencies and private groups. There is little public oversight, and civil rights groups did not know about the program. Some members, like the Church of Scientology and the Navy, have left. The article raises concerns that legal protest activity could be seen as a terrorism risk, and private companies are being asked to report on people.

In the comments, many say the network is dangerous and could be used to track innocent people, especially protesters. People worry about privacy, and some ask if their own companies are involved. Some defend the idea of sharing information to stop threats, but most want more oversight and better rules to protect rights.

Next, Spotify wants to help music fans get better concert tickets by holding some seats just for them. If you listen to an artist a lot, Spotify might invite you to buy “Fan First” tickets before the public sale. The goal is to stop scalpers and bots from buying all the good seats, and to help smaller artists connect with their true fans. Spotify works with Ticketmaster and AXS, and does not charge extra fees for these invites.

Hacker News users have mixed views. Some are happy real fans have a better chance at tickets. Others worry about Spotify collecting more data and controlling which artists get promoted. Some want to know how Spotify picks “true fans,” and others are concerned about privacy. There is hope that this helps fans, but also questions about fairness.

Next, someone shared how they used a five-year-old MacBook to index a year of video footage using a local AI model called Gemma 4 31B. The writer works in Africa and has many unlabeled video clips. They first tried online AI tools, but found them costly and not right for their needs. So, they built a local system that creates a “sidecar” text file for each video, storing details like time, faces, location, and keywords using tools like ffprobe, exiftool, WhisperX, and insightface. The AI writes all this into the sidecar, making videos searchable in plain English. Most of the code was written by Claude AI, and the MacBook handled the heavy load with lots of swap memory.

The main lessons were to use strict data schemas, run big jobs local, and focus on indexing before editing videos. In the comments, people praised the creative solution and were amazed the MacBook could run such a large model. Some worried about SSD health, but others said short bursts are fine. Many liked the sidecar file idea, and some asked for the code. There was discussion about AI hype, and some wondered if hiring a human editor is still best for creative work.
