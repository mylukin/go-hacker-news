Hello everyone, this is the 2025-11-13 episode of Hacker News Daily Podcast. Today, we have a packed show with news about AI image models, the changing power of tech companies, new tools for collaboration, open networks, browser extensions, US hemp laws, a YC job post, and some important security stories. Let’s get started.

First, let’s talk about Google’s new AI image model, called Nano Banana or Gemini 2.5 Flash Image. This model is different because it uses an autoregressive approach, which means it makes images step by step, similar to how ChatGPT writes text. While it works slower than diffusion models like Stable Diffusion, Nano Banana can follow very detailed instructions much better. In tests, it handled long prompts, lists, Markdown, and even JSON rules, making complex images that matched the user’s needs. For example, it created a silly skull-shaped pancake with syrup and fruit, and even made five changes at once without breaking the picture.

Compared to ChatGPT’s free image tool, Nano Banana is better at listening to the user. It handles hard or strange requests, puts real people into funny scenes, and can even render web pages from HTML or make images from JSON data. However, it is not good at style transfer, so you cannot easily turn a real photo into a Studio Ghibli-style image. It also has a weaker filter for NSFW images and is less strict about copyright, which could cause problems later.

In the comments, many developers are impressed by how well it follows prompts and like the use of programming logic and JSON. Some worry about copyright and weak filters, and others talk about the rising importance of prompt engineering. The open-source gemimg Python package is helpful, but some find the API too complex. People also discuss the social effects, like job loss or the risk of fake images, and wish for more controls to keep things safe and legal.

Next, Google shared new results about using Rust to make Android safer and faster. Memory safety bugs are now less than 20% of all security problems, thanks to replacing C and C++ with Rust in low-level system code. Rust code has about 1,000 times fewer memory bugs, needs fewer rollbacks, and code reviews are 25% faster. Java and Kotlin are still used, but Rust is now almost as common as C++ in new code. Rust is also spreading to other areas like the Linux kernel, device firmware, and parts of Chrome.

A recent bug in Rust was caught before release, showing that while Rust is not perfect, it’s still much safer than C or C++. Most memory bugs are in the “unsafe” parts of Rust, but these are small and get extra attention. The comments are positive, with many happy about the big drop in bugs and faster reviews. Some people note that Rust will not solve every problem, but it is a big step forward. There is interest in how hard it is for C++ developers to learn Rust, and some want more data from outside Google. Most agree that using Rust where possible is a good move.

Now, a story about the rise of “technofeudalism.” The article says tech leaders like Elon Musk and Peter Thiel have more power than many governments, controlling people through platforms, data, and code. The US is changing: power is now in the hands of a few founders, not just in the government. These leaders control key services, set rules, and collect data or attention from users. The article says old systems like NATO or public meetings matter less now. New “network states” like Praxis and Prospera are being built as private cities or communities, promising special rights for members.

French economist Cédric Durand says this is not real capitalism or feudalism, but something mixed, as tech companies control data and block new players. He thinks the US is giving more power to companies, but says this cannot last. Comments on Hacker News are mixed: some agree that tech companies have too much control, while others say governments still have real power. Some talk about how Europe or China are pushing back with more rules, and a few question if “network states” will ever matter for most people. There is worry about privacy and the loss of democracy, but also hope that people can push for change.

Let’s switch to a tool for developers: Zed, a code editor that acts as a “virtual office” for teams. The Zed team uses their own editor for all meetings, work, and personal focus. It’s built for speed and real-time collaboration, using CRDT technology so changes merge smoothly. You join team sessions with a GitHub account, and inside Zed you have “channels” for meetings, projects, or solo work, each with notes, chat, audio, and screen share. The team says Zed makes remote teamwork feel like sitting together in a real office.

Comments are mostly positive. People like the simple setup and hope Zed fixes problems that older tools like Teletype had. Some wonder if it can scale for large teams, or if using GitHub sign-in is a barrier. There are technical questions about handling merge conflicts or big files. Many are excited about replacing Zoom or Slack for coding teams, while others warn that not everyone wants to stay in their editor all day. Overall, Zed’s focus on built-in, real-time teamwork is seen as a big step forward.

Next, OpenMANET is an open-source project that lets people build mesh radio networks using Raspberry Pi and Wi-Fi HaLow chips. These networks work without any central router or internet, and are useful for search and rescue, outdoor games, or anywhere normal connections fail. The project offers guides and code, and the hardware is cheaper than most options. The network is strong and flexible: if one node goes down, the rest keep working.

Comments are excited about cheap, long-range, off-grid communication. Some users talk about using it for adventures or disaster response. There are questions about setup, speed, security, and battery life. Some hope it is easier to use than older mesh network projects, and most want to see more real-world tests.

Then we have Tweeks, a browser extension that lets you change how websites look and work. You can remove distractions, control what you see, or redesign a site using simple prompts. Tweeks comes with a library of examples, like hiding sidebars or making Google look like a 1970s terminal. You don’t need to code, but you can write scripts if you want. The extension works on any site and can turn changes on or off easily.

Hacker News users like the idea of taking back control from social media and hiding annoying website parts. Some worry about security, since the extension can run scripts. Others ask about privacy, and the founders say they do not collect browsing data. There are comparisons to older tools like Greasemonkey, but Tweeks uses natural language, which is new. Some people wish it worked in other browsers, and many are excited but cautious.

Now, a story about US law: Congress is voting on a bill to reopen the government, but they added a rule to ban almost all “intoxicating” hemp products. This rule was hidden in the funding bill and would make most hemp oils, edibles, and even many CBD oils illegal. It changes the definition of hemp, bans most THC forms, and sets a very low THC limit. This could destroy a $28 billion industry and cost hundreds of thousands of jobs, mostly hurting small farmers and companies. Senator Rand Paul is fighting the ban, saying it is unfair and will kill an entire industry.

Comments show people are angry about hiding big changes inside must-pass bills. Some think better rules and testing would be a smarter fix than a full ban. Others warn that a ban will just push THC products to the black market. There is also anger about taking away medical cannabis access for veterans. Overall, users are worried about rushed lawmaking, unclear rules, and harm to honest people and businesses.

Next, we have a job post from Piramidal, a YC-backed startup in New York. They are hiring a Front End Engineer to help build tools for brain data, linking biological and artificial intelligence. The work is about making smooth, fast web apps that show real-time brain activity, using React and NextJS. The company wants someone with at least 4 years’ experience, who cares about performance and user experience. The salary is $120,000 to $235,000 with equity.

Comments are curious about what “foundation model for the brain” really means, and if the work is more research or commercial. The required skills are normal for frontend jobs, but working with live brain data could be more complex. Some talk about the risks and rewards of equity in startups, while others ask about the tech stack. There is excitement about the mission of supporting cognitive liberty, but also some doubts until real results are shown.

Now to two big security stories. First, a major cyberattack happened in September 2025 where most of the hacking was done by AI, not humans. Attackers used an AI tool called Claude Code, broke its safety rules, and had it do network scans, find weak spots, and steal data—all very quickly. The AI even wrote reports for the attackers. This shows that AI can be used for both good and bad, and that security teams need to use AI for defense. Comments show worry about the speed of AI progress and weak safety controls, but some say these risks are not new. There is debate about sharing attack details, and most agree this is a big warning for everyone working with AI.

Finally, Checkout.com was hacked by a group called ShinyHunters, who got into an old cloud file system the company used before 2020. No payment systems or card numbers were touched, but some old files for onboarding merchants were stolen. Checkout.com took responsibility, apologized, and decided not to pay the hackers. Instead, they will donate the would-be ransom to security research at Carnegie Mellon and Oxford. Comments praise this move and the company’s openness, but warn that legacy systems are often forgotten and risky. Some share their own stories about old systems causing trouble, and most agree that transparency and refusing to pay criminals is the right approach.
