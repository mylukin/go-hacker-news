First, let’s talk about ntsc-rs, a new open-source tool that copies the look of old analog TVs and VHS tapes. Unlike most video filters, ntsc-rs uses real math to show TV and tape glitches, making the effect much more real. It is based on real NTSC and VHS signal processing, using ideas from other open projects. The tool is written in Rust, so it runs very fast, even with high-quality videos. You can use ntsc-rs as a standalone app, a web app, or plugins for popular video editors like After Effects and Premiere. Many people on Hacker News are happy to see such a real-looking VHS effect. Some remember watching VHS tapes as kids and say this tool brings back strong memories. Others are surprised at how real the glitches look, even on modern, clear videos. Some developers talk about the technical side, saying Rust is a good choice for fast video tools. There are questions about using ntsc-rs on different computers and how easy it is to add to video editing setups. Some users enjoy learning about old video technology thanks to the open-source code. There is some debate about whether we need more VHS effects, but most agree that ntsc-rs is fun, detailed, and great for experimenting.

Now, let’s look at Meta’s recent trouble with Instagram account hacks. Thousands of Instagram accounts were hacked because hackers tricked Meta’s AI chatbot for account recovery. The chatbot sent password reset links to emails given by the hacker, even if that email was not linked to the real account. This allowed hackers to take over accounts and see private messages. Meta says over 20,000 users were hit, and most did not have two-factor authentication turned on. Meta has now fixed the bug and turned off the chatbot for now. They are telling people to reset passwords and use safer ways to log in. On Hacker News, people say this shows why two-factor authentication is important. Many are shocked the chatbot could be fooled so easily. Some blame Meta for not testing the chatbot enough, and some worry about other AI tools that might have similar problems. There is talk about staff cuts at Meta leading to more mistakes, and some joke about how simple the hack was. Many feel that AI is not ready for important jobs like account recovery if it can be tricked this way.

Moving on, zeroserve is a new web server that lets you serve a website from a single file and add scripts using eBPF. The main idea is you put your site in a tar file, and zeroserve serves it directly, with no unpacking and no downtime when updating. All server logic is in eBPF scripts, written in C, and runs on every request. Zeroserve uses modern features like HTTP/2, TLS 1.3, and io_uring for speed. It is single-threaded per process, but you can run many processes for scaling. The eBPF scripts are sandboxed and preemptible for safety. Benchmarks show zeroserve is faster than nginx and Caddy for small and medium files, though nginx is still better for very large files. Memory use is low. The scripting model is different: all logic is in code, not split between config files and plugins. In the comments, people like the simple packaging and atomic deploys. Some love the “program-as-configuration” idea, but a few worry about needing to write C for scripts. There are questions about security, and some note that for large files, nginx is still better. Many like the hot-reload feature for smooth updates. Some ask about Rust or other safer languages for scripting in the future. Overall, people find zeroserve interesting and fast, especially for developers who want more control.

Next, we have Universal Memory Protocol, or UMP, a new standard for letting AI agents share and use memory across apps, vendors, and sessions. Right now, every AI tool keeps its own memory, so you can’t move notes or knowledge easily. UMP creates a portable way for memory, using simple actions like recall, remember, revise, and forget. It works with local files, databases, or even markdown notes, and you don’t have to change where you keep your data. The protocol keeps data signed and trusted, and uses a “supersede-never-delete” model, so old facts are never erased, just linked to updates. Security and consent are built in. On Hacker News, many people like the idea, saying it could stop vendor lock-in and make switching tools easier. Some worry that standardizing memory is hard because different tools have different needs. Others are excited to use UMP with personal knowledge bases like Obsidian, but some think the protocol sounds complex. There are questions about security and how to keep private data safe. Some like that UMP doesn’t force a special database. A few worry that “supersede-never-delete” could make data messy, but others say it helps with tracking and trust. Most agree UMP is promising, but they want to see real-world use before judging.

In chip news, Nvidia announced a new CPU system for Windows PCs with 128 GB of shared memory and up to 6,144 CUDA cores. The CPU part has 10 performance cores and 10 efficiency cores, using Cortex-X925. The main change is the 128 GB of unified memory, which means the CPU and GPU share the same pool. This idea started with Apple Silicon and is now spreading. Shared memory is not as fast as special GPU memory, but it is cheaper and good for many jobs, like running AI models at home. Gamers might also like the new system. The article wonders how Intel and AMD will answer Nvidia’s move, especially since AMD’s chips already support strong math instructions. In the comments, some people are excited, while others say 128 GB is not needed for most people. There is debate about AI models at home, and whether this is only for a small group. Some compare SVE2 and AVX-512, with most agreeing AVX-512 is stronger but harder to use. People also wonder if shared memory will slow down some games or if Windows can really use this hardware well. Many say price will be important, and some just want better battery life.

Next, Poincake is a new note-taking app that lets you make notes on an “infinite canvas” using a special kind of geometry called the Poincaré disk. In Poincake, you can put as many notes as you want, and they all fit on the screen. Notes far away look tiny, but you can still see them. You don’t have to fix the layout, and you can use portals to jump to different places. The app is just a demo now, and the controls may feel strange.
