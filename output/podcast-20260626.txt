First, OpenAI has announced a new AI model series called GPT-5.6, with versions named Sol, Terra, and Luna. Next, we look at a technical library that is getting a lot of attention: hopscotch-map, a C++ library for fast hash maps and hash sets using hopscotch hashing. This method is very cache-friendly and efficient, giving better speed and memory use than the standard unordered_map. The library offers normal and “prime growth” versions, plus “bhopscotch” for extra security against attacks that create many hash collisions. It is header-only, works with move-only types, and supports heterogeneous lookups. The API is close to std::unordered_map, but has some differences, like using it.value() instead of ->second.

Growth policies can be power-of-two, prime, or custom. The library is easy to install—just add the include folder or link with CMake. Secure versions use a binary search tree to defend against denial-of-service attacks. It works with any C++17 compiler and is open-source under MIT.

On Hacker News, many users praise its speed and clear documentation. Some share stories of big speed-ups compared to std::unordered_map, especially with lots of items. Others note the API can be surprising at first, but easy to learn. Some worry about the complexity and debugging of modern hash tables, and others discuss the need for secure versions. There is debate on whether this library should be the default, or if unordered_map is still fine for simple cases. People like the low memory use and easy installation. A few have issues with older compilers, but most find it works well. In general, developers are happy to have more choices for fast hash maps.

Switching gears, AWS has just launched Lambda MicroVMs—a new way to run code in small, secure virtual machines. These MicroVMs use Firecracker technology for speed, giving each user or session a fast, secure, and stateful environment. You just package your app, upload to S3, create a MicroVM image, and launch. The VM starts from a snapshot, making it almost instant. It can suspend when idle and resume quickly, saving costs. Each MicroVM is fully isolated, so running untrusted code is safer. You can use up to 16 CPUs, 32GB memory, and 32GB disk per MicroVM, and only pay for what you use. Lambda MicroVMs are now available in many AWS regions.

The community is excited, as this makes it easier to run untrusted code, AI, or games securely, without needing deep knowledge of VMs or containers. It could be a big step for serverless computing. But some worry about cost, vendor lock-in, and performance for large apps. Others want to know about debugging and monitoring inside each MicroVM. Some compare it to Google Cloud products, and discuss use cases in testing, research, or education. There are questions about data privacy inside MicroVMs. Overall, people see it as promising, but want to see real-world results and prices before moving all workloads.

In developer news, there is a new tool called “router” by Workweave. It helps you pick the best AI model for each prompt, such as Claude, Codex, or open-source models. The router acts as a middleman between your app and AI providers, making smart decisions about which model to use based on your prompt. It is fast—under 50 milliseconds—and easy to set up, with support for many APIs and tools. You can run it locally or use a hosted version, and your provider keys are kept safe. Switching to science, a new method now allows doctors and researchers to take detailed 3D images of the blood vessels inside a living human brain using ultrasound, and without surgery or big machines like MRI. This works by injecting microbubbles into the blood, which help ultrasound “see” tiny details. The method is much clearer than CT scans, and the team has shared their data and software as open source. This could help find early signs of strokes or Alzheimer’s. The bubbles are safe, and the team hopes to go even further—perhaps imaging without bubbles using better hardware and machine learning.

Hacker News users are excited, saying this is a big step for brain research and health. Finally, let’s talk about nomograms. These are special charts that let you solve math problems quickly, just by drawing a line across scales for each variable. Invented in 1880, they were very important before calculators, used in engineering, medicine, and many sciences. Nomograms are simple, cheap, and can handle very complex equations. The article introduces PyNomo, a tool that lets you design nomograms on your computer and print them. It works for many types of formulas, and you can customize the look.

In the comments, people share stories of using nomograms in engineering or medicine, and are surprised they are not taught more often. Some see them as beautiful and useful, especially where computers are not available. Others say making nomograms helps you understand math better. There are technical talks about design, and people say PyNomo makes this easier. Some argue nomograms should make a comeback, since they are fun, visual, and practical for teaching or field work.
