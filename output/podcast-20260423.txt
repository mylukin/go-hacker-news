Next, we have a serious story about security. Bitwarden CLI, a tool for the Bitwarden password manager, was hacked as part of a bigger supply chain attack linked to Checkmarx. The attack happened in the npm package version @bitwarden/cli2026.4.0, using a file called bw1.js with dangerous code. Attackers got into Bitwarden’s build process by using a broken GitHub Action. The bad code tried to steal secrets like GitHub tokens and AWS keys, sending the data to a server called audit.checkmarx.cx. It even made fake GitHub repositories with Dune-themed names. The malware used tricks to stay hidden, like not running if the computer’s language was Russian, making sure only one copy was running, and changing the user’s shell files to run itself again later.

If you use Bitwarden CLI, experts say you should check your systems, remove the bad package, change all your secrets, and look for odd files or GitHub activity. Watch for new repositories with Dune-related names, strange workflows, or lock files, and check if your tokens or cloud keys were used in odd ways.

Hacker News users were shocked that a popular tool like Bitwarden could be hacked, saying it shows open-source supply chains are risky. Some pointed out that attackers now target CI/CD pipelines because they hold many secrets. Others were upset about how late the warning came, and some shared ideas for safer workflows, like using short-lived tokens and more secure GitHub Actions. People discussed the challenge of trusting npm packages and agreed there is no easy fix except to always be careful. Many agreed the attack was serious and showed how creative hackers are today. Some worried about what else might be compromised and called for better tools to catch these problems early.

Now, a lighter story. A 10-year-old girl named Evie found a rare Mexican axolotl under a bridge in Wales. This is the first recorded case of an axolotl found in the wild in the UK. There are only 50 to 1,000 left in the world. Axolotls are salamanders that never leave the water as adults, keep their gills, and can even regrow body parts. They are popular pets, but hard to look after. The axolotl Evie found was hurt, so her family took it home, named it Dippy, and asked experts how to care for it. Experts think someone released Dippy into the wild because they could not care for it. It is illegal and dangerous to release non-native animals into nature. Axolotls are endangered in the wild, but common in tanks.

The article says if you find an endangered animal, tell the right group or the government. Evie’s quick action likely saved Dippy’s life, because axolotls cannot survive long in cold UK rivers. Her story made her popular at school, and her family has no regrets about helping Dippy.

Now, let’s talk about data privacy. Sensitive health data from the UK Biobank keeps ending up on public GitHub repositories, even though strict rules say this data must not be shared. The UK Biobank has health and genetic information from about half a million people and only gives access to approved researchers. Many researchers have accidentally uploaded private data to GitHub while sharing code or research. When this happens, UK Biobank tries to remove the data by sending takedown notices to GitHub, using copyright laws. These notices often target files like Jupyter notebooks, genetic data, or spreadsheets. Sometimes, only a small amount of data is shared, but even a little can let someone be identified.

There have been at least 110 takedown requests, covering 197 repositories by 170 developers in at least 14 countries. The process for finding and removing data relies on searching GitHub’s DMCA notices and matching them with UK Biobank cases. The UK does not have strong legal rules for fast data privacy takedowns, so UK Biobank uses copyright laws, which are usually for pirated software, not privacy.

In the comments, many people are worried about how easy it is to accidentally leak sensitive data. Some say researchers need better training about data privacy. Others think the rules might be too strict or confusing. A few point out that using copyright laws for privacy is not a good solution and shows that data protection laws are weak. Some users say even a small amount of data can be dangerous. Others argue that open science and sharing code are important, but it must be done safely. Some feel UK Biobank should help researchers avoid mistakes, maybe by giving better tools or advice. Finally, a few think it is very hard to control data once it leaves the original owner.

Next, astronomers have found where the Milky Way’s star-forming disk ends—about 40,000 light-years from the center. They did this by studying over 100,000 giant stars and using computer simulations and data from space missions like Gaia. In the inner part of the galaxy, stars are young because new stars are forming. As you move out, stars get younger until about 35,000 to 40,000 light-years from the center. After that, the stars get older again. This creates a “U” shape when you look at star ages. The edge marks where star formation drops sharply. The study used data from LAMOST, APOGEE, and Gaia, focusing on red giant stars because their ages are easier to measure.

On Hacker News, some readers were excited to see a clear answer to where our galaxy ends. Others pointed out that “edge” is tricky, since stars and gas are even farther out. Some said this discovery helps us understand how galaxies change over time and how stars move inside them. Others wondered how this new “edge” fits with the known halo of old stars around the Milky Way. Many liked how new data and better models are giving us sharper pictures of our galaxy, and some joked that the real “edge” is wherever our imagination stops.
