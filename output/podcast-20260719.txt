First, a developer replaced a $120,000 bowling management system with a new setup using ESP32 microcontrollers for only $1,600. The old system controlled scoreboards, lanes, and lights, but it was hard to maintain. Now, each bowling lane has its own ESP32 chip — these are cheap, easy to program, and connect by Wi-Fi. The developer wrote custom software for scoring, light and sound control, and used touchscreens and regular TVs for displays. If one lane breaks, the others keep working. All data goes to a central server, and the system can be controlled from a phone or tablet. The code is open source, so anyone can use or change it. The bowling center saved a lot of money, and now it’s easier to fix or update the system.

People in the comments were impressed by the low cost and the smart engineering. Many said it shows how new tech can replace expensive old systems. Some pointed out this project needed a lot of custom work, so it might not fit every bowling alley. A few worried about long-term support, since big companies offer service contracts that small setups can’t match. Some liked that the code is open, so others can help improve it. Others wondered about safety and reliability, since a commercial system must work all the time. Some readers shared their own stories of replacing expensive tech with DIY projects. Many found this project inspiring and a great example of what’s possible with cheap hardware and programming skills.

Moving on to programming, there’s an article about parallel programming and how it relates to teamwork and personal growth. The writer explains that just adding more processors does not always make computers faster. You need to split the problem into smaller parts, let each processor work, and make sure they can communicate and share the work well. If not, the system slows down. The real challenge is good coordination, not just more power.

The article then compares this to people. We have intelligence, energy, feelings, and creativity, but if these parts do not work together, we feel tired and confused. Old feelings can “use up” our energy, like unfinished computer tasks. The Zen idea of “wholehearted activity” is to do something fully, then let it go, instead of holding onto old experiences. Honest communication helps us synchronize our feelings, thoughts, and actions. If we hide things, we become anxious or burnt out. Both parallel programming and Zen ask how many parts can work together as one, without losing their own special qualities.

In the comments, some people liked the comparison and said it helped them think about teamwork and self-improvement. Others warned not to stretch the analogy too far, since people and computers are different. Some mentioned that real parallel programming is even harder than the article says, with more bugs and slowdowns. A few shared stories about trying to balance their own “internal processors.” Many liked the focus on honest communication, both in code and in life. Some thought the article was a bit philosophical, but useful. Overall, readers enjoyed the mix of technical and personal ideas, and it made many reflect on how they work with others and themselves.

For home network fans, there’s a detailed guide on setting up a MikroTik router as your main home router. The article explains how to set up the router for different kinds of internet connections — IPoE or PPPoE — and how to connect it to your fiber box. It covers how to handle VLANs, clone MAC addresses, and set up good WiFi using MikroTik’s CAPsMAN system. The guide shows how to solve a common problem called bufferbloat, which causes bad latency when the network is busy. It explains how to use FQ-CoDel, a smart queue system, and how to turn off fasttrack, which can break this feature.

In the comments, many people said MikroTik offers pro features at a good price, but the setup can be hard for beginners. Some compared MikroTik to other brands like Ubiquiti or OpenWRT. Others said MikroTik’s firewall and routing are powerful, but WiFi hardware could be better. Several people said FQ-CoDel really helps with gaming. Some mentioned ISPs can make things tricky, like using CGNAT or locking to a single MAC. A few shared that after using MikroTik, they learned a lot about networking, which helped them in their jobs.

Minecraft: Java Edition made a big technical change in its latest snapshot, switching its window and input system from GLFW to SDL3. This update changes how the game handles windows, keyboard, and mouse. Now, mouse input always uses relative mode, and key bindings use physical keys, fixing old issues with different keyboard layouts. The game now defaults to borderless fullscreen. Exclusive fullscreen is gone on macOS, and Linux prefers Wayland if available. The minimum window size is now 320x240. The debug overlay now shows player speed and refresh rate. There are also new features for spectator mode, creative inventory, and technical changes for data packs, loot tables, and world generation, plus many bug fixes.

In the comments, people are happy about SDL3 because it should help Linux and Wayland users. Those with non-US keyboards like the new key binding system. Some worry about bugs in fullscreen modes, especially on Windows and Wayland. Mac users are disappointed to lose exclusive fullscreen. Modders have mixed feelings — some like the new flexible options, others are frustrated they must update mods for new formats. Linux users are excited about Wayland support. Most people see this as a good technical step, though they expect some bumps as everyone adjusts.

Blender 5.2 LTS is out, giving the free 3D software long-term support until July 2028. This is a big release with many new tools, especially for Geometry Nodes. Now you can make animations that move with sound, add details with new bevel controls, and use new nodes for cloth and hair. Cycles and Eevee, Blender’s render engines, have new features for memory use and more realistic thin objects. The Asset Library is bigger, but the download size is not. The compositor is faster, and Grease Pencil is improved. Animation, modeling, and video editing all get new options. The user interface has many small changes, and VR support is better. The Python API changed, so some scripts and add-ons need updates. Hundreds of bugs are fixed.

In the comments, people are excited, especially artists and studios who want stable tools. Many like the new node-based physics and Asset Library. Some are happy about long-term support, saying it will help teams and big projects. Others warn that the new Python API will break old scripts. Some praise the faster compositor and better memory use. A few wish for more core modeling features. Some users feel lost with all the new features and want better documentation. Still, most are happy and say Blender 5.2 LTS keeps Blender a top choice for 3D work.

Another story is about the Jamcorder MIDI recorder. The author built and sold 2,500 units, and found that making hardware was much easier than writing the software. The Jamcorder records everything played on a piano, with a simple design using only 25 unique parts. The author made the first 500 units by hand and had no major problems. The hardest part was the software, which took over three years and 200,000 lines of code. The author gives advice: keep the parts list simple, avoid complex assembly, use Chinese suppliers, aim for good profit margins, avoid hiring too many people, check quality yourself, and use small packaging.

In the comments, many agree that hardware can be simple if you keep the design simple and control the process. Some warn things get hard if you try to scale up or add features. A few point out that making simple hardware is easier today because of better tools and suppliers. Some ask about customer support and returns. Others share their own hardware project stories. Some are surprised nothing went wrong, while others say luck and careful planning help. Overall, most like the practical tips and feel encouraged to try building hardware themselves.

And finally, bananas have grown in a garden in Rayleigh, Essex, after 15 years of waiting. Peter and Emma Stav built special walls to keep their banana plants warm in cold UK winters. This shows how warmer weather is changing what can grow in British gardens. Experts say fruits like olives, figs, and apricots are now easier to grow, while old favorites like gooseberries are struggling. The banana variety, Musa Basjoo, can survive cold but usually needs help to protect its flowers from frost. Peter stopped wrapping the plants because they got too tall, and a recent heatwave helped the plants finally grow fruit. Another gardener, Stephen Hind, also saw good results. TV botanist James Wong says bananas grow fast with enough heat and water, but this type is not really for eating — it has lots of hard seeds.

In the comments, people were happy and surprised about bananas growing in the UK. Some worried that warmer weather means climate change is real and could hurt other plants. A few shared tips about making microclimates or wrapping plants in winter. Some asked if more tropical fruits will start to grow in the UK, while others missed old British fruits. People debated if these changes are good for gardens or a sign of bigger problems. One person joked about starting a banana farm in England, while another wondered if the fruit would taste as good as shop bananas. There was a lot of curiosity and some concern about what British gardens will look like in the future.
